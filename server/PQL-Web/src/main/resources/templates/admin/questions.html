  {
    "questionUuid":   "xxxxxxxx-...",           // 있으면 기존 문제 업데이트, 없거나 null이면 신규 등록
    "topicCode":      "sql_basic_select",       // 위 목록에서 선택 (필수)

    "executionMode":  "EXECUTABLE",             // EXECUTABLE | CONCEPT_ONLY (필수)
    "stem":           "지문 전체 텍스트",          // 필수
    "hint":           "힌트 한 줄",               // 선택
    "schemaDdl":      "CREATE TABLE ...",       // EXECUTABLE 필수, CONCEPT_ONLY면 null
    "schemaSampleData": "INSERT INTO ...",      // 선택
    "schemaIntent":   "테이블 의도 한 줄",         // 선택
    "answerSql":      "SELECT ..."              // EXECUTABLE 필수, CONCEPT_ONLY면 null
  },
  { ... }
]

주요 규칙

  • JSON 파일은 배열 형태여야 합니다 [{...}, {...}]
  • 한 번에 최대 100건까지 가져올 수 있습니다
  • 내보내기는 최대 500건까지 지원합니다
  • questionUuid가 있으면 기존 문제를 업데이트, 없으면 신규 등록
  • questionUuid를 제거하면 복제/이관 용도로 활용 가능
  • EXECUTABLE 문제는 가져오기 시 Sandbox 자동 검증 (DDL + 샘플데이터 적용 후 answerSql 실행 테스트)
  • CONCEPT_ONLY 문제는 Sandbox 검증을 건너뜁니다 (SKIP)
  • 검증 후 "전체 등록" 또는 "성공한 것만 등록"을 선택할 수 있습니다
