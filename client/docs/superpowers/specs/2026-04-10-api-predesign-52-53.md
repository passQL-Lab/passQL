## 이슈 #52: 합격 준비도 (Readiness)

### 타입 추가

```typescript
// src/types/api.ts

export type ToneKey =
  | "NO_EXAM"
  | "ONBOARDING"
  | "POST_EXAM"
  | "TODAY"
  | "SPRINT"
  | "PUSH"
  | "STEADY"
  | "EARLY";

export type ScoreBand = "low" | "mid" | "high";

export interface ReadinessResponse {
  readonly score: number;          // 0.0~1.0 최종 합격 준비도
  readonly accuracy: number;       // 최근 50시도 정답률
  readonly coverage: number;       // 14일 내 활성 토픽 커버리지
  readonly recency: number;        // 마지막 학습일 기반 감쇠
  readonly coveredTopicCount: number;
  readonly activeTopicCount: number;
  readonly recentAttemptCount: number;
  readonly toneKey: ToneKey;
  readonly scoreBand: ScoreBand;
  readonly daysUntilExam: number | null;
  readonly lastSubmittedAt: string | null;
}

// ProgressResponse 확장 (기존 필드 유지)
export interface ProgressResponse {
  readonly solvedCount: number;
  readonly correctRate: number;
  readonly streakDays: number;
  readonly readiness: ReadinessResponse | null;  // 신규
}
```

### toneKey x scoreBand 카피 매트릭스

| toneKey | low (0~39%) | mid (40~69%) | high (70~100%) |
|---------|-------------|--------------|----------------|
| NO_EXAM | 시험 일정을 설정하면 맞춤 전략을 세울 수 있어요 | 꾸준히 하고 있네요. 시험 일정을 설정해보세요 | 실력이 탄탄해요. 시험 도전해볼 때예요 |
| ONBOARDING | 첫 걸음을 뗐어요. 하루 5문제부터 시작해봐요 | 좋은 출발이에요. 이 페이스 유지해봐요 | 빠르게 적응하고 있어요 |
| POST_EXAM | 다음 시험을 준비하면서 약한 영역을 보강해봐요 | 복습하면서 감을 유지하세요 | 다음 시험도 충분히 합격할 수 있어요 |
| TODAY | 오늘이 시험이에요. 긴장 풀고 실력 발휘하세요 | 충분히 준비했어요. 자신감을 가지세요 | 합격 준비 완료. 실력대로만 하면 돼요 |
| SPRINT | 시험이 코앞이에요. 오늘 10문제만 풀어봐요 | 막판 스퍼트! 약한 영역 집중 공략하세요 | 시험 준비 거의 끝. 마지막 점검만 남았어요 |
| PUSH | 2주 안에 시험이에요. 매일 조금씩 풀어봐요 | 잘 하고 있어요. 남은 기간 꾸준히 하면 돼요 | 이 페이스면 합격 충분해요 |
| STEADY | 아직 시간 있어요. 하루 한 문제씩 습관을 만들어봐요 | 안정적으로 준비하고 있어요 | 벌써 이 정도면 여유 있게 준비할 수 있어요 |
| EARLY | 일찍 시작했어요. 기초부터 탄탄히 쌓아가봐요 | 여유롭게 준비하고 있네요. 좋은 흐름이에요 | 미리 준비하는 만큼 합격이 가까워져요 |

프론트에서 관리: `src/constants/readinessCopy.ts`

### Home.tsx 변경 설계

현재 (line 112-124):
```
[푼 문제: N] [정답률: N% + bar]
```

변경 후:
```
[합격 준비도: N% + circular/bar gauge]
[푼 문제: N] [연속 학습: N일]
```

- `readiness.score`를 메인 게이지로 표시
- `toneKey` + `scoreBand`로 카피 매트릭스에서 메시지 선택
- `readiness === null`이면 기존 정답률 표시 (하위 호환)
- 3요소 드릴다운: accuracy/coverage/recency를 탭하면 상세 보기 (P2, 초기에는 숨김)

## 이슈 #53: 환영 메시지 스키마 개선

### 타입 변경

```typescript
// src/types/api.ts

export type GreetingContext = "GENERAL" | "EXAM_DAY" | "URGENT" | "COUNTDOWN";

export interface GreetingResponse {
  readonly nickname: string;          // 신규: 회원 닉네임 (fallback: "회원")
  readonly message: string;           // 변경: {nickname} 플레이스홀더 포함
  readonly context: GreetingContext;   // 신규: 상황 컨텍스트
}
```

### Home.tsx 변경 설계

현재 (line 47-50):
```tsx
<h1>{greeting?.message ?? `안녕하세요, ${displayName}`}</h1>
<p>{displayName}</p>
```

변경 후:
```tsx
// {nickname} 치환 + 닉네임 하이라이트
const renderedMessage = greeting
  ? greeting.message.replace("{nickname}", greeting.nickname)
  : `안녕하세요, ${displayName}`;

<h1>
  // nickname 부분만 볼드/컬러 처리 가능
  {renderedMessage}
</h1>
```

- `context`에 따라 아이콘 분기 가능 (EXAM_DAY면 경고 아이콘, GENERAL이면 없음 등)
- 하위 호환: `greeting.nickname`이 없으면(구 API) 기존 로직 유지
