## 제목
✨ [기능개선][UI/UX] 설정 화면 UI 개선 — 콘텐츠 보강 및 시각적 완성도 향상

## 본문

📝 현재 문제점
---

- 설정 화면이 MVP 수준 이상으로 개선되지 않은 채 방치됨
- 카드 1개(디바이스 ID / 닉네임 / 버전) + 로고 + 카피라이트가 전부 → 화면이 지나치게 빈약함
- 앱을 대표하는 화면임에도 브랜드 아이덴티티나 UX 배려가 부족
- 시험 일정 선택, 앱 소개 등 설정에 자연스럽게 붙을 수 있는 콘텐츠가 없음
- TeamModal이 헤더 버튼에 숨어 있어 노출도가 낮음

🛠️ 해결 방안 / 제안 기능
---

- 섹션 구분 명확화: 계정(닉네임/디바이스 ID), 앱 정보(버전/팀 소개), 기타로 구분
- 시험 일정 선택 섹션 추가 — `/exam-schedules` API 연동
- 팀 소개(Made by 콜레라) 설정 화면 내 직접 노출 (모달 접근성 개선)
- 앱 브랜딩 영역 강화 — 상단 또는 하단에 passQL 로고 + 슬로건
- 전체적인 시각적 완성도 향상 (섹션 헤더, 아이콘, 간격 등)

⚙️ 작업 내용
---

- `src/pages/Settings.tsx`: 섹션 구조 개편 및 콘텐츠 추가
- `src/api/examSchedules.ts`: 시험 일정 API 연동 (기존 함수 재사용)
- `src/components/TeamModal.tsx`: 설정 화면 내 팀 소개 카드로 이동 or 연결 검토
- 필요 시 신규 컴포넌트 분리 (`SettingsRow.tsx` 등)
