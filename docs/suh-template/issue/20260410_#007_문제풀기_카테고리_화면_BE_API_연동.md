# ⚙️[기능추가][FE] 문제 풀기 카테고리 선택 화면 BE Topic API 연동

**라벨**: `작업전`
**담당자**: 

---

📝현재 문제점
---

- 문제 탭(`/questions`) 진입 시 카테고리 선택 화면이 **BE `/api/meta/topics` API를 사용하지 않고** Mock 데이터 또는 하드코딩으로 표시되고 있다
- FE Mock topic 코드(`JOIN`, `SUBQUERY`, `GROUP_BY`, `DDL`, `CONSTRAINT`)와 BE DB topic 코드(`sql_join`, `sql_subquery`, `sql_group_aggregate`, `sql_ddl_dml_tcl` 등)가 **불일치**한다
- BE에는 9개 topic이 SQLD 시험 기준으로 잘 분류되어 있으나, FE Mock에는 5개만 존재하고 코드 체계도 다르다

### BE Topic 목록 (DB 시드 데이터 — 변경 없음)

| sort_order | code | displayName |
|:---:|---|---|
| 1 | `data_modeling` | 데이터 모델링의 이해 |
| 2 | `sql_basic_select` | SELECT 기본 |
| 3 | `sql_ddl_dml_tcl` | DDL / DML / TCL |
| 4 | `sql_function` | SQL 함수 (문자/숫자/날짜/NULL) |
| 5 | `sql_join` | 조인 (JOIN) |
| 6 | `sql_subquery` | 서브쿼리 |
| 7 | `sql_group_aggregate` | 그룹함수 / 집계 |
| 8 | `sql_window` | 윈도우 함수 |
| 9 | `sql_hierarchy_pivot` | 계층 쿼리 / PIVOT |

### BE API 응답 형태

**GET `/api/meta/topics`** 응답 (인증 불필요):
```json
[
  {
    "topicUuid": "uuid-string",
    "code": "sql_join",
    "displayName": "조인 (JOIN)",
    "sortOrder": 5,
    "isActive": true,
    "subtopics": [
      {
        "code": "inner_join",
        "displayName": "INNER JOIN",
        "sortOrder": 1,
        "isActive": true
      }
    ]
  }
]
```

### FE vs BE 불일치 상세

| FE Mock code | BE DB code | 비고 |
|---|---|---|
| `JOIN` | `sql_join` | code 불일치 |
| `SUBQUERY` | `sql_subquery` | code 불일치 |
| `GROUP_BY` | `sql_group_aggregate` | code 불일치 |
| `DDL` | `sql_ddl_dml_tcl` | code 불일치 |
| `CONSTRAINT` | **(BE에 없음)** | BE에는 독립 topic 아님 |
| **(FE에 없음)** | `data_modeling` | FE 누락 |
| **(FE에 없음)** | `sql_basic_select` | FE 누락 |
| **(FE에 없음)** | `sql_function` | FE 누락 |
| **(FE에 없음)** | `sql_window` | FE 누락 |
| **(FE에 없음)** | `sql_hierarchy_pivot` | FE 누락 |

### FE TopicTree 타입 vs BE 응답 차이

현재 FE 타입 (`src/types/api.ts`):
```typescript
interface TopicTree {
  readonly code: string;
  readonly displayName: string;
  readonly subtopics: readonly SubtopicItem[];
}
interface SubtopicItem {
  readonly code: string;
  readonly displayName: string;
}
```

BE 실제 응답에는 `topicUuid`, `sortOrder`, `isActive` 필드가 추가로 포함됨. FE 타입에 반영 필요.

---

🛠️해결 방안 / 제안 기능
---

### 1. FE `TopicTree` 타입을 BE 응답에 맞게 확장

- `topicUuid`, `sortOrder`, `isActive` 필드 추가
- `SubtopicItem`에도 `sortOrder`, `isActive` 필드 추가
- 파일: `src/types/api.ts`

### 2. Mock 데이터를 BE topic code 체계로 동기화

- `MOCK_TOPICS`의 code를 BE와 일치시킨다 (`JOIN` -> `sql_join` 등)
- 9개 전체 topic을 Mock에 반영한다
- `MOCK_QUESTIONS`의 `topicName`도 BE displayName과 일치시킨다
- 파일: `src/api/mock-data.ts`

### 3. 문제 풀기 카테고리 그리드 화면 구현 및 API 연동

- `/questions` 진입 시 BE `/api/meta/topics` API 호출하여 카테고리 그리드를 동적으로 렌더링
- 기존 `useTopics()` 훅이 이미 구현되어 있으므로 그대로 활용
- 카테고리 클릭 시 해당 topic의 문제 목록으로 이동 (topic code를 쿼리 파라미터로 전달)
- 파일: `src/pages/Questions.tsx`

### 4. 문제 목록 필터의 topic code 동기화

- 문제 목록 드롭다운 필터에서 사용하는 topic code가 BE code 체계와 일치하는지 확인
- `fetchQuestions(params)` 호출 시 `topic` 파라미터가 BE code 체계(`sql_join` 등)를 사용하도록 확인
