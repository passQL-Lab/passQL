📝 현재 문제점
---

- `client.ts`의 `log()` 함수가 DEV 환경에서 API 요청/응답을 콘솔에 출력하고 있으나, 브라우저 개발자 도구를 열어야만 확인 가능함
- 개발자 모드(Easter Egg) 제거로 인해 인앱 디버그 진입점이 없어짐
- API 응답 시간, AI 디버그 정보, Zustand 스토어 상태 등을 화면에서 직접 확인할 수단이 없음
- 디버그 관련 코드가 프로덕션 코드와 혼재될 가능성이 있음

🛠️ 해결 방안 / 제안 기능
---

- 설정 화면에 **개발자 모드 토글** 추가 — 활성화 시 sessionStorage에 `devMode` 플래그 저장
- 개발자 모드 활성화 시 화면에 **플로팅 FAB(Floating Action Button)** 표시 — daisyUI Speed Dial 활용
- FAB에서 Speed Dial 방식으로 디버그 패널 진입 (기능 확장에 유리)
- DEV 빌드(`import.meta.env.DEV`)에서만 렌더링되도록 제한

**FAB Speed Dial 패널 구성 (1차):**
1. **API 로그 패널** — `apiFetch` 호출 이력 (method, path, status, 소요시간, req/res body 펼치기)
2. **AI 응답 디버거** — `explainError` / `diffExplain` 응답시간, `promptVersion` 등
3. **Zustand 스토어 뷰어** — `authStore`, `practiceStore`, `memberStore` 실시간 스냅샷

⚙️ 작업 내용
---

- `src/dev/` 폴더로 개발자 기능 완전 분리
  - `DevProvider.tsx` — context + 활성화 상태 관리
  - `DevFab.tsx` — daisyUI Speed Dial FAB 컴포넌트
  - `panels/ApiLogPanel.tsx` — API 호출 이력 패널
  - `panels/AiDebugPanel.tsx` — AI 응답 디버거 패널
  - `panels/StorePanel.tsx` — Zustand 스토어 뷰어 패널
  - `hooks/useApiLog.ts` — `apiFetch` 후킹 로직
  - `index.ts` — 외부 re-export (App.tsx는 이것만 import)
- `Settings.tsx`에 개발자 모드 토글 추가 (sessionStorage `devMode` 플래그)
- `client.ts`의 `log()` 콘솔 출력을 HUD 패널로 리디렉션
- 프로덕션 빌드에서는 `src/dev/` 전체가 렌더링되지 않도록 처리
