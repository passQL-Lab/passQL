👋 [기능추가][Server] 홈 화면 인사말 API (Greeting) 구현

📝 현재 문제점
---

- 홈 화면 상단의 "안녕하세요, {닉네임}님" 영역을 단순 하드코딩하면 시간대/시험 D-day 등 컨텍스트를 반영할 수 없습니다.
- "프론트 비즈니스 로직 제로 원칙"에 따라 인사말 문장 자체를 백엔드가 완성해 내려줘야 합니다.
- 시험 임박도(D-7, D-30 등), 시간대(아침/낮/저녁/심야), 자격증 종류에 따라 다양한 친근한 메시지를 노출해야 합니다.

🛠️ 해결 방안 / 제안 기능
---

- `GET /api/home/greeting` 엔드포인트를 신설하여 완성된 한 문장(`message`)을 응답합니다.
- 백엔드는 `ExamScheduleRepository.findSelected()`로 D-day를 계산하고, KST 현재 시각의 시간대를 분기 기준으로 사용합니다.
- D-day 임박/시험 당일/시험 후/일반의 4가지 분기와, 시간대별 메시지 풀에서 랜덤 선택을 조합합니다.
- 자격증명(`SQLD`, `SQLP`)을 메시지에 그대로 노출합니다 (한글화는 추후).

### 📌 API

- [ ] `GET /api/home/greeting?memberUuid={uuid}` 엔드포인트 추가
- [ ] Response DTO: `GreetingResponse { message }`
- [ ] `memberUuid` 미존재 시 404

### 📌 분기 로직

- [ ] `GreetingService.build(UUID memberUuid) → String`
  - 입력: `nickname` (Member 조회), `now` (KST), `selectedExam` (ExamScheduleRepository.findByIsSelectedTrue)
  - `dDay = selectedExam != null ? ChronoUnit.DAYS.between(today, examDate) : null`
- [ ] 분기 우선순위
  - `dDay == 0` → `"오늘이 {certType} 시험 날이에요, {nickname}님 💪 화이팅!"`
  - `1 <= dDay <= 7` → `"{certType} D-{dDay}, 마지막 스퍼트예요 {nickname}님 🔥"`
  - `8 <= dDay <= 30` → 시간대 풀 + `"{certType} 시험 {dDay}일 전이에요, {nickname}님"` 가중치 포함 랜덤
  - 그 외(`dDay == null` ∨ `dDay > 30` ∨ `dDay < 0`) → 시간대 풀에서만 랜덤

### 📌 시간대 메시지 풀 (`GreetingMessagePool` 상수)

- [ ] 05~10 (아침): "좋은 아침이에요", "공부하기 좋은 날이에요", "오늘도 한 문제씩"
- [ ] 10~17 (낮): "꾸준함이 답이에요", "한 문제 풀고 갈까요", "오늘도 화이팅"
- [ ] 17~22 (저녁): "좋은 저녁이에요", "하루 마무리, 한 문제 어때요", "수고 많았어요"
- [ ] 22~05 (심야): "늦은 시간까지 고생이에요", "무리하지 마세요"
- [ ] suffix `, {nickname}님` 자동 부착 헬퍼

### ✅ 단위 테스트

- [ ] D-day 분기 6케이스: `dDay = 0 / 3 / 15 / 45 / -3 / null`
- [ ] certType: SQLD / SQLP / null
- [ ] 시간대 4종 분기 (Clock 주입으로 결정적 테스트)
- [ ] 닉네임 suffix 부착 검증
