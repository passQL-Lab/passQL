# ❗[버그][문제풀기] 선택지별 쿼리 실행 버튼이 독립적으로 동작하지 않는 버그

**라벨**: `작업전`
**담당자**: 

---

🗒️설명
---

- 문제 풀기 화면(제출 후 실행 모드)에서 선택지 A의 "쿼리 실행"을 눌러 **실행 중...** 상태가 된 뒤, 선택지 B의 "쿼리 실행"을 누르면 A의 상태가 **"쿼리 실행"으로 초기화**되고 B만 "실행 중..."으로 표시되는 버그
- 각 선택지의 쿼리 실행은 서로 독립적으로 병렬 실행되어야 하나, 단일 mutation 인스턴스의 상태를 공유해 마지막 실행만 "실행 중..." 으로 반영됨

🔄재현 방법
---

1. 문제 풀기 화면에서 답안을 제출한다 (EXECUTABLE 타입 문제)
2. 선택지 A의 "쿼리 실행" 버튼을 클릭한다 → **실행 중...** 상태 확인
3. 실행이 완료되기 전, 선택지 B의 "쿼리 실행" 버튼을 클릭한다
4. 선택지 A가 "실행 중..." → "쿼리 실행"으로 초기화되는 것을 확인한다

📸참고 자료
---

- 관련 파일: `client/src/pages/QuestionDetail.tsx`
- 관련 훅: `client/src/hooks/useQuestionDetail.ts` — `useExecuteChoice`

✅예상 동작
---

- 선택지 A가 "실행 중..."인 상태에서 선택지 B를 실행하면 **A는 "실행 중..." 유지**, B도 독립적으로 "실행 중..." 표시
- 각 선택지의 실행 상태는 서로 영향을 주지 않아야 함
