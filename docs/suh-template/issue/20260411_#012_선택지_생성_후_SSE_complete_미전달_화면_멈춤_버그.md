# ❗[버그][문제풀기] 선택지 생성 성공 후 SSE complete 미전달로 화면 멈춤

**라벨**: `작업전`
**담당자**: 

---

🗒️설명
---

- 두 번째 이후 문제에서 선택지 생성 요청 시 서버에서 선택지 생성은 성공하지만 클라이언트 화면이 "선택지 생성 중..." 상태에서 멈추는 버그
- 첫 번째 문제는 정상 동작하나 이후 문제부터 SSE `complete` 이벤트가 클라이언트에 전달되지 않음
- 서버 로그에서 HikariPool이 Pool-3, 4, 5, 6...으로 지속 생성 및 즉시 종료되는 비정상 패턴 확인

🔄재현 방법
---

1. 문제 풀기 화면에서 첫 번째 문제 선택지 생성 및 제출 (정상 동작)
2. 다음 문제로 이동
3. 두 번째 문제의 선택지 생성 SSE 연결 시 "선택지 생성 중..." 에서 화면 멈춤
4. 서버 로그에서 `[choice-gen] success` 로그는 출력되나 SSE complete 전달 안 됨

📸참고 자료
---

- `ChoiceSetGenerationService.java` — `saveSuccess()`, `saveFailed()`, `saveConceptSuccess()`가 `protected` + `@Transactional` 조합으로 self-invocation 문제 발생
- `ChoiceSetResolver.java` — `resolveForUser()`에 `@Transactional` 선언으로 AI·샌드박스 호출 동안 트랜잭션 유지
- 서버 로그: HikariPool-3 ~ HikariPool-10 연속 생성/종료 패턴

✅예상 동작
---

- 선택지 생성 성공 시 SSE `complete` 이벤트가 정상적으로 클라이언트에 전달되어야 함
- HikariPool 커넥션이 정상적으로 반환되어 Pool 번호가 증가하지 않아야 함
