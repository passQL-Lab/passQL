# ➕[기능추가][Admin] 관리자 대시보드 화면 구현

**라벨**: `작업전`
**담당자**: 

---

📝현재 문제점
---

- `/admin` 진입 시 라우팅할 `AdminDashboardController`가 존재하지 않아 404 발생
- `dashboard.html` 템플릿은 존재하나 데이터를 공급하는 서비스·컨트롤러가 없어 사실상 빈 화면
- 핵심 지표(전체 문제 수, 오늘 제출 수, 에러율 등) 및 회원 현황을 한눈에 파악할 수 있는 화면 부재
- 관리자가 주요 메뉴로 빠르게 이동할 수 있는 진입점 없음

🛠️해결 방안 / 제안 기능
---

- `AdminDashboardController` 생성 — `/admin`, `/admin/` 라우팅 처리, dashboard.html 렌더링
- `AdminDashboardService` 생성 (Application 레이어) — 여러 도메인 Repository를 조합한 통계 집계
  - 활성 문제 수, 토픽별 문제 분포
  - 전체·활성·정지 회원 수 (테스트 계정 제외)
  - 오늘 제출 건수, 최근 24h 에러율
- `DashboardStats` DTO (record) 추가 — 집계 결과를 Thymeleaf에 전달
- `SubmissionRepository`에 `countBySubmittedAtAfter()` 추가 — 오늘 제출 건수 효율 집계
- `dashboard.html` 보강
  - 핵심 지표 카드 4개 (문제 수 / 제출 수 / AI 호출 / 에러율)
  - 회원 통계 카드 3개 (전체 / 활성 / 정지)
  - 토픽별 문제 분포 — 비율 막대 바 차트
  - 빠른 링크 — 주요 관리 페이지 6개 바로가기
  - 최근 실행 로그 테이블 (20건)

⚙️작업 내용
---

- `PQL-Application/.../dto/DashboardStats.java` 신규
- `PQL-Application/.../service/AdminDashboardService.java` 신규
- `PQL-Web/.../controller/admin/AdminDashboardController.java` 신규
- `PQL-Domain-Submission/.../repository/SubmissionRepository.java` — `countBySubmittedAtAfter()` 추가
- `PQL-Web/.../templates/admin/dashboard.html` 수정
