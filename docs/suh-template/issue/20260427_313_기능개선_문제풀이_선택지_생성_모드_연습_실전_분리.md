📝 현재 문제점
---

- 현재 선택지 생성 로직(`ChoiceSetResolver`)은 본인(`generatedForMemberUuid = 현재사용자`) 대상 프리페치 선택지만 캐시 히트로 인정한다.
- 프리페치 미스 시 DB에 다른 사용자용으로 검증 완료(`status=OK`)된 선택지가 이미 존재하더라도 무조건 AI 실시간 생성(`AI_RUNTIME`)을 수행한다.
- 실시간 생성은 AI 호출 + 샌드박스 SQL 검증 최대 3회 반복으로 사용자가 최대 60초 가까이 대기해야 하는 UX 문제가 있다.
- 선택지가 문제 특성상 개인화가 불필요함에도 재사용 가능한 선택지를 버리고 있다.

🛠️ 해결 방안 / 제안 기능
---

- 설정 페이지에 **문제풀이 선택지 생성 모드** 옵션을 추가한다.
- 모드는 두 가지로 구성한다.

  | 모드 | 동작 |
  |------|------|
  | **연습 모드** | DB에 `status=OK`인 선택지가 있으면 기존 것을 재사용. 없을 때만 AI 실시간 생성 |
  | **실전 모드** | 기존 선택지 유무와 무관하게 항상 AI 실시간 생성 |

- **연습 모드** 세부 동작:
  1. 현재 사용자의 프리페치 선택지(`source=AI_PREFETCH`, `generatedForMemberUuid=현재사용자`, `consumed_at IS NULL`) 조회 → HIT 시 사용
  2. 미스 시 다른 사용자 포함 전체 `status=OK` 선택지 중 최신 1건 조회 → HIT 시 사용
  3. 전체 미스 시 AI 실시간 생성

- **실전 모드** 세부 동작:
  1. 프리페치·기존 선택지 조회 없이 바로 AI 실시간 생성 (`source=AI_RUNTIME`)

⚙️ 작업 내용
---

- 사용자 설정에 `choiceGenerationMode` 필드 추가 (`PRACTICE` / `REAL`)
- `ChoiceSetResolver`에 모드 분기 로직 추가
- 연습 모드 전용 DB 조회 쿼리 구현 (다른 사용자 선택지 포함 `status=OK` 최신 조회)
- 설정 페이지(`Settings.tsx`) UI에 모드 선택 토글/라디오 추가
- 클라이언트에서 모드 값을 선택지 생성 API 요청 시 함께 전달
