# ❗[버그][프론트엔드] 선택지 생성 SSE 에러 미수신 시 UI 무한 로딩 버그

**라벨**: `작업전`
**담당자**: 

---

🗒️설명
---

- 선택지 생성(`/generate-choices`) SSE 요청에서 백엔드가 에러를 반환하거나 스트림이 조용히 종료될 경우, 프론트엔드 UI가 "선택지 생성 중..." 상태로 영구 멈춘다.
- 사용자는 실패 여부를 알 수 없고, 새로고침 외에 탈출 방법이 없다.
- SSE는 HTTP 200으로 스트림이 시작된 후 비동기로 에러가 발생하므로, 백엔드 에러가 `onError` SSE 이벤트로 클라이언트에 전달되지 않으면 프론트엔드는 상태를 변경하지 못한다.

🔄재현 방법
---

1. `choiceSetPolicy=AI_ONLY` 문제 풀이 페이지 접근 (`/daily-challenge` 또는 `/questions/:id`)
2. 선택지 생성 요청 발생 (`POST /api/questions/{questionUuid}/generate-choices`)
3. 백엔드에서 선택지 생성 3회 모두 실패 (`CHOICE_SET_VALIDATION_MULTIPLE_CORRECT` 등)
4. UI가 "선택지 생성 중..." 스피너 상태로 영구 멈춤
5. 에러 메시지나 재시도 버튼이 나타나지 않음

📸참고 자료
---

**SSE 응답 흐름:**
```
HTTP 200 (스트림 시작)
→ event:status
  data:{"phase":"generating","message":"선택지 생성 중..."}
→ (백엔드 가상 스레드에서 에러 발생)
→ onError 이벤트 미전달 또는 스트림 조용히 종료
→ 프론트엔드 로딩 상태 영구 유지
```

**재현 페이지:** `http://localhost:5174/daily-challenge`

✅예상 동작
---

- SSE `onError` 이벤트 수신 시 에러 메시지와 재시도 버튼이 표시되어야 한다.
- 일정 시간(예: 60초) 내에 `complete` 또는 `error` 이벤트가 오지 않으면 클라이언트 타임아웃을 발동하여 에러 상태로 전환해야 한다.
- 스트림이 조용히 닫혔을 때도 에러 상태로 전환하여 사용자에게 실패를 알려야 한다.
