📝현재 문제점
---

- 현재 README는 기능 나열 중심으로 구성되어 있어 **AI 교육 공모전 제출물로서의 어필력이 부족**하다.
- 공모전 심사위원이 한눈에 파악해야 하는 **사업성·시장성·기술 차별성**이 구조적으로 드러나지 않는다.
- 기존 문구에 노출된 내부 구현 세부사항이 **공모전 포지셔닝과 어긋나는 부분**이 있다.
  - `MariaDB` 노출 → 실제 SQLD 시험은 Oracle 기반이므로 외부 노출 시 `Oracle DB`로 통일 필요 (MariaDB는 테스트 환경 한정)
  - `Gemini`라는 벤더 명칭 노출 → `AI Agent`라는 추상화된 표현이 기술적 독립성·확장성 측면에서 더 강한 인상
  - `bge-m3` → 현존 최고 수준 임베딩인 `Gemma Embedding`으로 변경 및 어필 포인트화 필요
- 시장 규모와 사업성을 뒷받침하는 **정량 데이터**가 부재하다. (SQLD 연간 응시자 수, 기존 수험서/인강 가격, 반복 결제 구조)
- 공모전의 핵심인 **"왜 AI인가"**에 대한 스토리텔링 섹션이 약하다.

🛠️해결 방안 / 제안 기능
---

### 1. 시장 규모·사업성 섹션 신설 및 강화

- **SQLD 연간 약 10만 명 응시** 규모를 명시하여 타깃 시장 크기를 정량화
- 기존 수험서(3~4만 원), 인강(10~30만 원)과의 **가격 비교표** 추가
- **"1%만 월 구독해도 연 수억 원대 SaaS"** 형태의 반복 결제 수익 모델 제시
- B2B 라이선스(부트캠프/학원/기업 교육), ADsP·빅데이터분석기사 등 **인접 자격증 확장 가능성** 명시

### 2. 기술 표현의 추상화·격상

- `Gemini` → `AI Agent` 전면 치환
  - Agent가 "문제 생성 → 샌드박스 실행 검증 → 애매하면 재생성" 루프를 스스로 수행한다는 점을 부각
- `MariaDB` → `Oracle DB` 전면 치환
  - "SQLD 실제 시험 환경과 동일한 엔진에서 검증"이라는 **차별화 포인트로 전환**
  - `ORA-00904` 등 실전 Oracle 에러를 미리 경험할 수 있다는 학습 가치 강조
- `bge-m3` → `Gemma Embedding`
  - "현존 최고 수준 오픈 임베딩 모델" 문구로 기술 우위 어필

### 3. "왜 지금, AI Agent인가" 서사 강화

- 기존 문제집의 본질적 한계(고정 문제 + 답 외우기)를 먼저 제시
- passQL이 AI Agent로 이 구조를 깨뜨리는 방식을 4~5개 핵심 bullet로 정리
  - AI Agent가 선택지 4개를 세트로 생성 (문법 함정 정교화)
  - 샌드박스 실행으로 정답 자체 검증 (AI가 틀려도 실행 결과가 진실)
  - 애매하면 재생성 루프 (자체 피드백)
  - 구조화 JSON 기반 확장 가능한 파이프라인

### 4. 타깃 사용자 명시

- SQLD 응시자 / SQLP 응시자 / 비전공자 / 현업 개발자 / **부트캠프·기업 교육 담당자(B2B)** 까지 확장

### 5. 링크 섹션 정비

- README 상단에 **서비스 메인 / 관리자 화면 / Swagger UI / API Docs(JSON)** 링크를 한 블록으로 노출
- 공모전 심사위원이 별도 탐색 없이 **즉시 동작하는 서비스·API 문서**로 진입 가능하도록

### 6. 내부 구현 노출 금지 원칙 문서화

- README는 외부 공개 문서이므로, 테스트·개발 환경 스택(MariaDB 등)은 **노출하지 않는다**는 원칙을 이슈에 명시
- 내부 기술 스택 문서는 별도 위치(`docs/`)에서 관리

🧠고찰: 공모전 어필 포인트 정리
---

### A. 기술적 어필

1. **AI Agent 자율 피드백 루프**
   단순 생성이 아니라 "생성 → 실행 검증 → 실패 시 재생성"이라는 **Agent 다운 자율성**. 공모전 심사에서 "AI를 어떻게 썼는가"에 강력한 답변.

2. **샌드박스 실행 기반 진실성(Ground Truth)**
   AI가 틀려도 실행 결과가 진실이라는 **신뢰 가능한 AI 시스템 설계 원칙**. AI 환각 문제에 대한 실질적 해법.

3. **Gemma Embedding을 활용한 개념 기반 학습 경로**
   키워드가 아닌 **의미 기반 약점 추적**. 개인 맞춤 학습의 기술적 기반.

### B. 사업적 어필

1. **이미 존재하는 10만 명 규모 시장**
   "만들고 나서 시장을 찾는" 프로젝트가 아니라 **기존 수요에 더 나은 대안을 주는** 프로젝트. 심사위원이 사업성 판단 시 가장 중요시하는 요소.

2. **가격 구조의 명확한 우위**
   책(3~4만 원, 1회성) / 인강(수십만 원) 대비 **저렴한 월 구독 + 무제한 문제**라는 단순하고 강력한 경쟁력.

3. **확장 가능한 SaaS 구조**
   SQLD → SQLP → ADsP → 빅데이터분석기사로 **템플릿 복제** 가능. PL/SQL·NoSQL 등 영역 확장도 동일 구조.

4. **B2B 채널 존재**
   부트캠프·기업 교육 담당자라는 **단가 높은 B2B 시장**이 동시에 열려 있음.

### C. 교육적 어필 (공모전이 AI 교육 분야인 경우 핵심)

1. **"읽기"에서 "실행하기"로의 학습 방식 전환**
   SQL 학습의 본질적 문제(실행 없는 암기)를 AI + 샌드박스로 해결.

2. **무제한 반복 연습의 학습 효과**
   고정 문제집의 한계(답 외우기)를 AI 자동 생성으로 돌파.

3. **실전과 동일한 환경(Oracle DB)에서의 실습**
   시험장에서의 적응 비용 최소화.
