# ⚙️[기능개선][UI] 제출·액션 버튼 fixed bottom 전 화면 통일

**라벨**: `작업전`
**담당자**: 

---

🖌️요청 내용
---

- 현재 `QuestionDetail`의 제출 버튼이 스크롤 영역 내부에 위치하여, 제출 후 `PracticeFeedbackBar`(fixed bottom)와 동시에 존재하는 UI 불일치 발생
- `AnswerFeedback` 페이지는 이미 fixed bottom 버튼을 사용하고 있으나, 나머지 화면과 스타일(z-index, 배경, 테두리)이 통일되어 있지 않음
- 모든 화면에서 액션 버튼을 `fixed bottom-0 inset-x-0`으로 통일하여 일관된 UX 제공

**현재 상태 (불일치)**

| 화면 | 액션 버튼 위치 | 방식 |
|---|---|---|
| QuestionDetail (단독 + practiceMode) | 스크롤 영역 내부 | 플로우 안 |
| DailyChallenge | 스크롤 영역 내부 (QuestionDetail 상속) | 플로우 안 |
| PracticeSet | 스크롤 영역 내부 (QuestionDetail 상속) | 플로우 안 |
| AnswerFeedback | fixed bottom-0 | 고정 하단 |
| PracticeFeedbackBar | fixed bottom-0 z-30 | 고정 하단 |

🎯기대 결과
---

- 모든 화면에서 액션 버튼이 `fixed bottom-0 inset-x-0`에 위치
- 스크롤 영역은 버튼 높이만큼 `pb-24` 패딩 확보
- z-index 계층 통일: 제출 버튼 `z-20`, PracticeFeedbackBar `z-30` (피드백 시 자연스럽게 덮음)
- 버튼 컨테이너 스타일 통일: `bg-surface-page border-t border-border`

📋참고 자료
---

- 관련 파일
  - `client/src/pages/QuestionDetail.tsx` — 제출 버튼 fixed 전환 대상
  - `client/src/pages/AnswerFeedback.tsx` — 스타일 통일 대상
  - `client/src/components/PracticeFeedbackBar.tsx` — z-30 기준 레이어 (변경 없음)
  - `client/src/pages/DailyChallenge.tsx` — 변경 없음 (QuestionDetail 수정으로 해결)
  - `client/src/pages/PracticeSet.tsx` — 변경 없음 (QuestionDetail 수정으로 해결)

💡추가 요청 사항
---

- `QuestionDetail`의 최상위 컨테이너 `pb-6` → `pb-24` 변경으로 fixed 버튼과 콘텐츠 겹침 방지
- `AnswerFeedback`의 기존 fixed 버튼 z-index를 `z-20`으로 명시 통일
