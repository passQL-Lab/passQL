# ❗[버그][문제풀이] RESULT_MATCH 선택지 테이블 클릭 불가 및 선택 시 텍스트 미반전 버그

**라벨**: `작업전`
**담당자**: 

---

🗒️설명
---

- RESULT_MATCH 유형 선택지(`choiceSetPolicy === "RESULT_MATCH"`)에서 두 가지 버그 발생
- **버그 1**: 테이블 영역을 클릭해도 선택이 되지 않음. 카드 바깥 테두리 영역만 클릭 가능
- **버그 2**: 선택지를 선택해도(파란 배경으로 전환) 테이블 텍스트가 반전되지 않아 파란 배경 위에 어두운 텍스트가 겹쳐 가독성 0

🔄재현 방법
---

1. `choiceSetPolicy === "RESULT_MATCH"` 유형 문제 풀이 화면 진입
   - 예: `/practice/:sessionId` 또는 `/recommendation/:questionUuid`
2. 선택지 카드의 **테이블 셀 영역**을 직접 클릭
3. 선택이 되지 않음 (카드 테두리 영역 클릭 시에만 선택됨)
4. 카드 테두리 클릭으로 선택 후 파란 배경으로 전환된 상태 확인
5. 테이블 헤더·셀 텍스트가 기본 색상(어두운 텍스트) 그대로 유지되어 가독성 없음

📸참고 자료
---

✅예상 동작
---

- 테이블 포함 카드 전체 영역 클릭 시 선택 상태로 전환되어야 함
- 선택(파란 배경) 상태에서 테이블 헤더·셀 텍스트가 흰색 계열로 반전되어야 함 (`data-table--inverted` 적용)
