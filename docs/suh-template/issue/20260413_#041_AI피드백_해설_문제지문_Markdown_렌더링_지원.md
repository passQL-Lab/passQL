---

📝현재 문제점
---

- `PracticeFeedbackBar`의 해설(rationale)이 plain text로 표시되어 굵게·코드 등 Markdown 서식이 깨져 보임
- `PracticeResult`의 AI 리포트 텍스트도 plain text 렌더링 — AI가 Markdown을 사용해도 그대로 노출
- 문제 지문(stem)은 `QuestionDetail.tsx`에서 이미 `ReactMarkdown`으로 처리 중이나, 별도 커스텀 컴포넌트 없이 인라인으로 작성되어 재사용 불가
- AI 리포트의 단어별 fade-in 애니메이션(`useAiText`)이 `innerHTML` 직접 조작 방식이라 `ReactMarkdown`과 충돌 — 동시 적용 불가

🛠️해결 방안 / 제안 기능
---

- `src/components/MarkdownText.tsx` 공용 컴포넌트 신규 생성
  - `animated` prop으로 단어별 fade-in 켜고 끄기
  - `ReactMarkdown`의 `components` prop으로 passQL 디자인 시스템 클래스 주입
- `src/hooks/useWordFadeIn.ts` 훅 신규 생성 — 기존 `useAiText` 대체
  - React Context로 단어 인덱스 관리, CSS `--i` 변수로 animation-delay 계산
  - DOM 직접 조작 없이 순수 React 방식
- 사용처 교체
  - `PracticeResult.tsx` : `<MarkdownText text={aiComment} animated />`
  - `PracticeFeedbackBar.tsx` : `<MarkdownText text={result.rationale} />`
  - `QuestionDetail.tsx` : `<MarkdownText text={question.stem} />`
- `useAiText` 훅 삭제 (MarkdownText로 완전 대체)
