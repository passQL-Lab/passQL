# ⚙️[기능개선][문제풀기] 실행 UI 통일·샘플 데이터 파싱 버그 수정·결과 테이블 가독성 개선

**라벨**: `작업전`
**담당자**: 

---

📝현재 문제점
---

- 실행 버튼 스타일이 `ChoiceCard`, `SqlPlayground`, `AnswerFeedback` 세 곳에서 각각 달라 시각적 일관성이 없음 (`btn btn-xs btn-outline`, `btn-compact`, `btn-run-all` 혼재)
- `INSERT INTO T1 VALUES (1), (2), (3)` 형식처럼 한 INSERT 문에 여러 행을 한꺼번에 삽입하는 샘플 데이터를 파싱하지 못해 스키마 샘플 데이터가 깨져 표시됨
- `SqlPlayground`에서 네트워크 오류·5xx 발생 시 `catch`가 없어 에러가 조용히 사라지고 사용자에게 피드백이 없음
- 선택된 ChoiceCard(파란 배경) 위 `ResultTable`을 흰 박스로 감싸 오히려 더 튀어 보임
- `SqlPlayground`에 사용하지 않는 `questionUuid` prop이 잔존
- `SqlPlayground` 헤더 텍스트 "SQL 실행기"가 다른 섹션 레이블 스타일과 달라 시각적으로 튐
- `textarea`에 인라인 `style` 속성 5개 사용으로 프로젝트 인라인 style 금지 규칙 위반

🛠️해결 방안 / 제안 기능
---

- 실행 버튼을 `btn-run-all` 단일 토큰으로 통일, 선택된 카드 위 반전 variant(`btn-run-all--inverted`) 분리
- `parseSampleData` 정규식을 개선하여 `VALUES (1),(2),(3)` 다중 행 INSERT 형식 파싱 지원
- `SqlPlayground.handleExecute`에 `catch` 블록 추가, 네트워크 에러 시 `error-card`로 표시
- 선택된 카드의 `ResultTable`에 `inverted` prop 도입, 흰 wrapper 제거 후 반투명 배경+흰색 텍스트로 처리
- 미사용 `questionUuid` prop 제거
- `SqlPlayground` 헤더를 `Terminal` 아이콘 + "직접 실행해보기" 레이블로 교체
- textarea 인라인 스타일 → `sql-playground-textarea` CSS 클래스로 분리

⚙️작업 내용
---

- `client/src/components/ChoiceCard.tsx`: 실행 버튼 `btn-run-all` 통일, `inverted` prop 전달
- `client/src/components/SqlPlayground.tsx`: 헤더 개선, textarea 클래스화, catch 추가, `questionUuid` prop 제거
- `client/src/components/ResultTable.tsx`: `inverted` prop 추가, 선택 카드 위 반전 스타일 적용
- `client/src/components/SchemaViewer.tsx`: `parseSampleData` 다중 행 INSERT 파싱 수정
- `client/src/pages/QuestionDetail.tsx`: `questionUuid` prop 전달 제거
- `client/src/styles/components.css`: `btn-run-all--inverted`, `sql-playground-textarea`, `data-table--inverted`, `result-table-border--inverted` 등 신규 클래스 추가
