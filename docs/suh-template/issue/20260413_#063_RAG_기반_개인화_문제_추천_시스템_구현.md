# ➕[기능추가][AI/추천] RAG 기반 개인화 문제 추천 시스템 구현

**라벨**: `작업전`
**담당자**: 

---

📝현재 문제점
---

- 현재 `GET /api/questions/recommendations` API는 `ORDER BY RANDOM()`으로 완전 랜덤 추천만 제공
- 사용자의 풀이 기록(오답, 취약 토픽)이 추천에 전혀 반영되지 않음
- Qdrant 벡터 DB, Ollama bge-m3 임베딩 클라이언트 등 RAG 인프라가 이미 준비되어 있으나 미연결 상태
- Python AI 서버의 `similar()` 메서드가 `raise CustomError("아직 구현되지 않았습니다")`로 방치

🛠️해결 방안 / 제안 기능
---

- 문제(Question)를 `stem + 토픽 + 난이도 + 개념태그` 조합으로 bge-m3 임베딩하여 Qdrant `passql_questions` 컬렉션에 적재
- 사용자의 Submission 기록에서 최근 오답, 취약 토픽을 집계하여 쿼리 벡터를 동적 합성
- Qdrant 벡터 유사도 검색으로 개인화된 문제 추천 (이미 푼 문제 제외 필터 적용)
- Python AI 서버 장애 시 기존 RANDOM() 방식으로 자동 fallback

⚙️작업 내용
---

**Python AI 서버 (`ai/`)**
- `ai_request.py`: `IndexQuestionRequest`, `IndexQuestionsBulkRequest`, `RecommendRequest` 모델 추가
- `ai_response.py`: `IndexQuestionResponse`, `RecommendResponse` 모델 추가
- `qdrant_client.py`: `create_collection()`, filter 파라미터 지원 추가
- `ai_service.py`: `index_question()`, `index_questions_bulk()`, `recommend()` 메서드 구현
- `ai_router.py`: `POST /api/ai/index-question`, `POST /api/ai/index-questions-bulk`, `POST /api/ai/recommend` 엔드포인트 추가

**Java 서버 (`server/`)**
- `AiGatewayClient.java`: `indexQuestion()`, `indexQuestionsBulk()`, `recommend()` 메서드 추가
- `QuestionService.java`: `getRecommendations()` — memberUuid 수신 시 AI 추천, 없으면 RANDOM fallback
- `QuestionController.java`: recommendations에 memberUuid 파라미터 연동
- 신규 DTO: `IndexQuestionRequest`, `RecommendRequest`, `RecommendResult` 등
- `QuestionGenerateService.java`: 문제 등록 시 자동으로 `indexQuestion()` 호출

**설계 원칙**
- 사용자 신호(오답 UUID 목록, 취약 토픽 UUID 목록)는 Java가 DB에서 집계 후 Python에 전달
- Python은 임베딩 생성 및 Qdrant 검색만 담당 (DB 직접 접근 없음)
- Qdrant 컬렉션명: `passql_questions` (기존 `passql_concepts`와 분리)
