# ➕[기능추가][문제풀이] 문제 풀이 중 이탈 시 경고 모달 구현

**라벨**: `작업전`
**담당자**: 

---

📝현재 문제점
---

- 연습 세션(`PracticeSet`), 데일리 챌린지(`DailyChallenge`), 단독 문제 풀이(`QuestionDetail`) 중 다른 화면으로 이동 시 아무런 경고 없이 이동됨
- `PracticeSet`의 경우 마지막 문제까지 완료해야 세션 기록이 저장되는 구조인데, 중간 이탈 시 해당 세션의 풀이 기록이 유실됨
- 사이드바 메뉴, 상단 Home 버튼, 브라우저 뒤로가기 등 모든 이탈 경로에 경고 없음

🛠️해결 방안 / 제안 기능
---

- 공통 `ConfirmModal` 컴포넌트 신규 구현 (`src/components/ConfirmModal.tsx`)
  - props: `isOpen`, `title`, `description`, `confirmLabel`, `cancelLabel`, `onConfirm`, `onCancel`
  - 디자인: 모바일 바텀시트 / 데스크톱 중앙 모달 (기존 Design System 준수)
- React Router DOM v7의 `useBlocker` 훅을 사용해 앱 내 네비게이션 이동 시 차단
- 아래 3개 화면에 이탈 방지 적용

| 화면 | 파일 | 차단 조건 | 해제 시점 |
|---|---|---|---|
| 연습 세션 | `PracticeSet.tsx` | 세션 진입 후 | 결과 페이지 이동 직전 |
| 데일리 챌린지 | `DailyChallenge.tsx` | 진입 후 | `submitAnswer` 성공 후 |
| 단독 문제 풀이 | `QuestionDetail.tsx` | 진입 후 (`practiceMode=false`일 때만) | 제출 후 |

- 모달 문구
  - 제목: `"풀이를 그만할까요?"`
  - 설명: `"지금 나가면 현재 풀이 기록이 저장되지 않아요."`
  - 확인: `"나가기"` (Secondary 버튼)
  - 취소: `"계속 풀기"` (Primary 버튼 — 파괴적 액션 비강조)

⚙️작업 내용
---

- `src/components/ConfirmModal.tsx` 신규 생성
- `src/pages/PracticeSet.tsx` — `useBlocker` + `ConfirmModal` 연결
- `src/pages/DailyChallenge.tsx` — `useBlocker` + `ConfirmModal` 연결
- `src/pages/QuestionDetail.tsx` — `practiceMode=false` 조건부 `useBlocker` + `ConfirmModal` 연결
