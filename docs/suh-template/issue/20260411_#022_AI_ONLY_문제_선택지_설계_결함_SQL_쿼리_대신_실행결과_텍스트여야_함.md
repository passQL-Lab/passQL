# ❗[버그][문제설계] AI_ONLY 문제 선택지가 SQL 쿼리로 생성되어 문제 유형과 불일치

**라벨**: `작업전`
**담당자**: 

---

🗒️설명
---

- "다음 SQL의 실행 결과로 올바른 것은?" 유형(`AI_ONLY`)은 주어진 SQL의 **실행 결과(테이블/값)** 를 맞추는 문제
- 그런데 AI가 생성하는 선택지 `body`가 실행 결과 텍스트가 아닌 **SQL 쿼리** 형태로 생성됨
- 사용자는 A, B, C, D 각각의 실행 버튼을 눌러 결과를 확인한 뒤 정답을 골라야 하는 구조가 됨
- 이는 퀴즈가 아니라 SQL 실행기 사용법 테스트에 불과 — 문제의 학습 목적과 완전히 어긋남
- 결국 `AI_ONLY` + SQL 선택지 조합은 어떤 문제 유형도 제대로 충족하지 못하는 어중간한 상태

🔄재현 방법
---

1. `executionMode: EXECUTABLE`, `choiceSetPolicy: AI_ONLY` 문제 풀이 페이지 진입
2. "다음 SQL의 실행 결과로 올바른 것은?" 지문 확인
3. 선택지 A~D가 모두 SQL 쿼리 텍스트로 표시됨
4. 각 선택지 실행 버튼을 눌러야만 결과를 볼 수 있어 사실상 정답을 직접 확인하는 구조

📸참고 자료
---

**현재 잘못된 구조**:
```
문제: "다음 SQL의 실행 결과로 올바른 것은?"
선택지 A: SELECT A.NAME, B.DEPT_NAME FROM EMP A JOIN DEPT B ON ...  ← SQL 쿼리
선택지 B: SELECT A.NAME, B.DEPT_NAME FROM EMP A, DEPT B WHERE ... OR ...  ← SQL 쿼리
```

**올바른 구조 (두 가지 방향)**:

| 방향 | 문제 유형 | 선택지 형태 |
|------|---------|-----------|
| 실행 결과 예측 | "이 SQL의 실행 결과로 올바른 것은?" | 결과 테이블 텍스트 (NAME \| DEPT_NAME\n홍길동 \| 개발팀 ...) |
| 동일 결과 SQL 찾기 | "같은 결과를 내는 SQL은?" | SQL 쿼리 (현재 방식) — 단 문제 지문이 달라야 함 |

**영향 범위**:
- `server/PQL-Web/src/main/resources/db/migration/` — `generate_choice_set` 프롬프트 시드
- `server/PQL-Web/src/main/resources/static/docs/question-bulk-guide.md`
- `server/PQL-Web/src/main/resources/static/docs/question-register-guide.md`
- `client/src/pages/QuestionDetail.tsx` — 선택지 렌더링 방식

✅예상 동작
---

- `AI_ONLY` + "실행 결과로 올바른 것은?" 유형에서 선택지는 **실행 결과 텍스트** 형태여야 함
- 선택지에 SQL 쿼리가 나오는 유형은 `ODD_ONE_OUT`("결과가 다른 것은?") 또는 별도 정책으로 분리
- 가이드라인과 AI 프롬프트에 선택지 `body` 형태 규칙 명확히 정의 필요
