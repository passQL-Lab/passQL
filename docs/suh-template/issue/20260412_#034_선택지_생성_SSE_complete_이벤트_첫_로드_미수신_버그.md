# ❗[버그][SSE] 선택지 생성 첫 로드 시 complete 이벤트 미수신으로 화면에 선택지 미표시

**라벨**: `작업전`
**담당자**:

---

🗒️설명
---

- 문제 상세 페이지 최초 진입 시 AI 선택지 생성이 완료됨에도 선택지가 화면에 표시되지 않음
- 페이지를 리로드하면 DB에 저장된 결과를 정상 표시함
- 백엔드 로그 상 `SSE complete 전송: itemCount=4` 및 DB 저장까지 정상 완료됨

🔄재현 방법
---

1. 선택지가 아직 생성되지 않은 문제 상세 페이지 최초 진입
2. 백엔드 로그에서 `[choice-gen] SSE complete 전송` 확인
3. 프론트 화면에 선택지가 표시되지 않고 로딩 상태로 멈춤
4. 페이지 리로드 시 선택지가 정상 표시됨

📸참고 자료
---

**백엔드 정상 로그 (선택지 생성 완료 확인):**
```
[choice-gen] Sandbox 검증 완료: attempt=3, correctCount=1
[choice-gen] 검증 통과, 저장 시작
[choice-gen] success: attempts=3, setUuid=6e078f38-dc06-4a41-900b-323958eac28a
[generate-choices] SSE complete 전송: itemCount=4
```

**원인 분석:**

Spring `SseEmitter.complete()`가 호출되면 `complete` 이벤트와 스트림 종료(`done: true`)가 동일한 TCP 패킷에 묶여 도착할 수 있음.

`client/src/api/questions.ts`의 SSE 파싱 루프에서 `done: true` 분기 진입 시 함께 수신된 `value`(Uint8Array)를 decode하지 않고 버림:

```typescript
// 기존 코드 — done: true 시 value 유실
if (done) {
  if (buffer.trim()) {
    buffer += "\n\n";
    processBuffer();  // buffer에 complete 이벤트가 없어 처리 못함
  }
  break;
}
```

`complete` 이벤트가 `done: true`와 함께 오는 마지막 `value` 안에 있으면, buffer에 추가되지 않아 `processBuffer()`가 이벤트를 찾지 못하고 `onComplete` 콜백이 호출되지 않음.

**관련 파일:**
- `client/src/api/questions.ts` — SSE ReadableStream 파싱 루프 (`done: true` 분기)
- `server/PQL-Web/src/main/java/com/passql/web/controller/QuestionController.java` — `generateChoices` SSE 엔드포인트

✅예상 동작
---

- 문제 상세 페이지 최초 진입 후 AI 선택지 생성이 완료되면 선택지 4개가 즉시 화면에 표시되어야 함
