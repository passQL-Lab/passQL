# ⚙️[기능개선][Admin] 문제 등록 가이드라인 및 AI 변환 프롬프트 고도화

**라벨**: `작업전`
**담당자**: 

---

📝현재 문제점
---

- 단건 등록(`question-register-guide.md`) 및 벌크 등록(`question-bulk-guide.md`) 가이드라인의 AI 변환 프롬프트가 최소 수준에 머물러 있음
- 외부에서 AI를 활용해 JSON을 생성할 때 품질이 일정하지 않음 (stem 품질 저하, schemaDdl 누락, answerSql 오류 등)
- `schemaDisplay` 필드가 JSON 스키마에 누락되어 있어 AI 생성 시 빠뜨림
- ODD_ONE_OUT 유형, CONCEPT_ONLY 유형에 대한 안내가 부족하여 잘못된 JSON이 자주 생성됨
- JSON 예시가 없어 AI가 올바른 형식을 파악하기 어려움
- 프롬프트에 MariaDB/Oracle 호환 SQL 작성 기준이 불명확

🛠️해결 방안 / 제안 기능
---

- 단건/벌크 가이드 모두에 `schemaDisplay` 필드 추가 및 설명 보강
- AI 변환 프롬프트를 확장하여 아래 항목 포함:
  - EXECUTABLE vs CONCEPT_ONLY 판별 기준 명확화
  - ODD_ONE_OUT 유형 식별 방법 및 answerSql 작성 기준
  - schemaDdl / schemaSampleData 생성 규칙 (NULL 허용 여부, FK 명시, MariaDB 호환)
  - stem 작성 품질 기준 (문제 지문만, 보기 A/B/C/D 제외)
  - 완성된 JSON 예시 포함 (EXECUTABLE 1건 + CONCEPT_ONLY 1건 + ODD_ONE_OUT 1건)
- 벌크 프롬프트에 배열 형식 강제 안내 강화
- 가이드라인 다운로드 MD 파일도 동일하게 최신화
