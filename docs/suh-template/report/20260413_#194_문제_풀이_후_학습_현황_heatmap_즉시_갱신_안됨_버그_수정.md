# ❗[버그][홈] 문제 풀이 후 학습 현황(heatmap) 즉시 갱신 안 됨 #194

## 개요

문제 제출 완료 후 홈 화면 복귀 시 학습 현황 캘린더(heatmap)와 streak이 갱신되지 않는 버그를 수정했다. 원인은 `DailyChallenge`, `QuestionDetail`, `RecommendationPractice`, `PracticeSet` 4개 페이지 모두에서 `submitAnswer` 성공 시 `heatmap` / `progress` 쿼리 무효화(`invalidateQueries`)가 누락된 것으로, React Query의 `staleTime: 5분` 동안 캐시된 오래된 데이터가 그대로 표시되는 문제였다.

## 변경 사항

### 쿼리 무효화 추가 (프론트엔드)

- `client/src/pages/DailyChallenge.tsx`: 정답 제출 성공 시 `heatmap`, `progress` 쿼리 무효화 추가
- `client/src/pages/QuestionDetail.tsx`: 제출 완료 시 `heatmap`, `progress` 쿼리 무효화 추가
- `client/src/pages/RecommendationPractice.tsx`: 제출 완료 시 `heatmap`, `progress` 쿼리 무효화 추가
- `client/src/pages/PracticeSet.tsx`: `useQueryClient` import 추가, 마지막 문제 제출 완료 시(`currentIndex + 1 >= totalQuestions`)에만 `heatmap`, `progress` 쿼리 무효화 추가

## 주요 구현 내용

각 제출 지점에서 `submitAnswer` API 성공 직후 아래 두 쿼리를 무효화하여 홈 복귀 시 즉시 리페치되도록 처리했다.

```ts
queryClient.invalidateQueries({ queryKey: ["heatmap"] });
queryClient.invalidateQueries({ queryKey: ["progress"] });
```

`PracticeSet`은 연속 문제 풀기 세션 구조로, 매 문제마다 무효화할 경우 불필요한 API 호출이 반복되므로 마지막 문제 제출 시에만 무효화하도록 결정했다. 중간 이탈 시에는 갱신하지 않는 것이 의도된 동작이다.

## 주의사항

- `PracticeSet`의 무효화 조건은 `currentIndex + 1 >= totalQuestions`로 판단하며, `submitAndAdvance` store 액션 호출 이후에 위치하므로 index 증가 전 값을 기준으로 비교한다.
- `heatmap` 쿼리 키는 `["heatmap", uuid, from, today]` 형태이나 prefix 매칭으로 무효화되므로 `["heatmap"]`만으로 충분하다.
