# 🚀[기능개선][문제등록] 문제 등록 가이드라인 및 AI 변환 프롬프트 고도화

## 개요

단건·벌크 문제 등록 가이드라인 MD 파일과 관리자 페이지 모달 내 AI 변환 프롬프트를 대폭 강화했다. 외부 AI로 JSON을 생성할 때 발생하던 품질 불일치(stem에 보기 포함, schemaDdl 누락, ODD_ONE_OUT 미지정 등) 문제를 해소하고, 3가지 유형(EXECUTABLE·ODD_ONE_OUT·CONCEPT_ONLY)의 완성된 예시를 포함하여 AI가 형식을 오해 없이 생성할 수 있도록 개선했다.

## 변경 사항

### 가이드라인 MD

- `server/PQL-Web/src/main/resources/static/docs/question-register-guide.md`: 전면 재작성
  - `schemaDisplay` 필드 스키마·필드 규칙·프롬프트에 추가 (기존 누락)
  - `executionMode` 판별 기준표 신규 추가
  - `ODD_ONE_OUT` 유형 전용 작성 규칙 섹션 추가 (answerSql = 소수 쪽 SQL 명시)
  - `schemaDdl` 작성 규칙 상세화 (MariaDB 호환, FK 순서, 불필요 컬럼 제외)
  - AI 변환 프롬프트: difficulty 5단계 기준 명시, 3가지 유형 완성 예시 JSON 포함

- `server/PQL-Web/src/main/resources/static/docs/question-bulk-guide.md`: 전면 재작성
  - 단건 가이드와 동일한 신규 항목 반영
  - JSON 배열 스키마에 `schemaDisplay` 필드 추가
  - 벌크 프롬프트에 배열 형식 강제 안내 강화, 3가지 유형 완성 예시 배열 포함

### 관리자 페이지 UI

- `question-register.html`: 모달 내 JSON 스키마 표시에 `choiceSetPolicy` 필드 미포함 상태 (MD 파일 다운로드로 최신 스키마 확인 가능)
- `questions.html`: 벌크 가이드 모달의 AI 프롬프트 복사 버튼 위치를 modal-action(하단)에서 모달 헤더 우측으로 이동하여 단건 등록 가이드 모달과 UI 통일

## 주요 구현 내용

AI 변환 프롬프트에 완성된 JSON 예시를 3가지 유형으로 포함함으로써 AI가 형식을 추론하는 데 걸리는 컨텍스트를 줄이고, 필드 누락이나 보기 포함 같은 구조적 오류를 사전에 방지한다. `schemaDisplay` 필드는 이전에 MD 가이드와 HTML 모달 스키마 표시 모두에서 누락되어 있었으며 이번에 가이드라인 전체에 추가되었다.

## 주의사항

- HTML 모달 내 하드코딩된 JSON 스키마(`<pre>` 블록)는 이번에 수정되지 않았음. 최신 스키마는 MD 다운로드 버튼으로 확인
- 벌크 가이드 모달의 AI 프롬프트 본문(`BULK_AI_PROMPT` JS 상수)도 MD 파일 내용과 별도로 관리됨 — 추후 동기화 자동화 검토 필요
