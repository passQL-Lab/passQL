# ❗[버그][문제신고] 문제 신고 API 코드 품질 버그 수정

## 개요

문제 신고 API(`#222`) 구현 이후 발견된 4가지 버그를 수정하였다. 관리자 목록 조회의 N+1 쿼리 문제, Controller에 위치한 검증 로직, nativeQuery Timestamp 캐스팅 안전성, `question_report_category` 중복 삽입 방지 누락, 그리고 `QuestionReport` 엔티티의 `@JoinColumn(name)` 명시로 인한 서버 기동 불가 오류까지 해결하였다.

---

## 변경 사항

### Service 레이어 — QuestionReportService

- `server/PQL-Application/.../service/QuestionReportService.java`
  - **N+1 제거**: `getReportSummaries()`에서 신고 건수만큼 반복 조회하던 stem 조회를 `questionRepository.findAllById(questionUuids)`로 일괄 배치 조회로 전환 (쿼리 1회)
  - **검증 로직 이동**: `submissionUuid` null 체크, `categories` 비어있음 체크, `ETC` 카테고리 `detail` 필수 검증, 중복 신고 체크를 Controller에서 Service로 이동 (CLAUDE.md 규칙: Controller 비즈니스 로직 금지)
  - **Timestamp 안전 처리**: nativeQuery 결과의 `row[3]`을 `instanceof Timestamp`, `instanceof LocalDateTime` 분기로 캐스팅 — DB 드라이버 종류에 무관하게 안전 처리
  - **resolveReport() 검증 강화**: `correctionScope == null` 방어, `RESOLVED` 상태 중복 처리 차단, `CHOICE_SET_ONLY` 시 `choiceSetUuid` 존재 여부 검증 추가
  - **`ReportSummaryRow` record**: Controller가 Web 모듈 DTO 없이 Application 레이어 결과를 바로 소비할 수 있도록 내부 record 타입 도입

### Controller 레이어 — QuestionReportController

- `server/PQL-Web/.../controller/QuestionReportController.java`
  - 검증 로직 전체 제거 — Service 위임으로 단순화, 요청/응답 변환 역할만 수행

### 엔티티 — QuestionReport

- `server/PQL-Domain-Question/.../entity/QuestionReport.java`
  - `@ManyToOne` + `@JoinColumn(name = "question_uuid")`, `@JoinColumn(name = "choice_set_uuid")` 제거
  - **원인**: `@JoinColumn(name = "...")` 명시 시 Hibernate가 동일 물리 컬럼(`choice_set_uuid`)을 논리 이름(`choiceSetUuid`)과 중복 매핑으로 인식 → `DuplicateMappingException`으로 서버 기동 불가
  - **해결**: 연관 객체(`question`, `choiceSet`)를 실제 사용하는 코드가 없으므로 `@ManyToOne` 관계 자체를 제거, UUID 필드만 유지

### DB 마이그레이션

- `server/PQL-Web/.../db/migration/V0_0_126__add_pk_to_question_report_category.sql`
  - `question_report_category` 테이블에 복합 PK `(question_report_uuid, category)` 추가 — V0_0_120에서 누락
  - `IF NOT EXISTS` 조건으로 멱등성 보장 (`DO $$ ... $$` 블록)

## 주요 구현 내용

**N+1 → 배치 조회 전환**

기존에는 신고 집계 결과 행마다 `questionRepository.findById(uuid)`를 개별 호출하여 N+1이 발생했다. `findAllById(uuidList)`로 한 번에 조회 후 `Map<UUID, String>`으로 변환하여 O(1) 탐색으로 처리하도록 변경하였다.

**Timestamp 다형 처리**

PostgreSQL 드라이버 버전 및 Hibernate 설정에 따라 nativeQuery 결과의 날짜 컬럼이 `java.sql.Timestamp` 또는 `java.time.LocalDateTime`으로 반환될 수 있다. `instanceof` 패턴 매칭으로 두 타입을 모두 안전하게 처리한다.

**DuplicateMappingException 근본 원인**

`CLAUDE.md`의 `@Column(name = ...)` 금지 규칙과 동일한 맥락으로, `@JoinColumn`에도 `name` 속성을 명시하면 Hibernate PhysicalNamingStrategy와 충돌이 발생한다. 해당 엔티티는 UUID 필드(`questionUuid`, `choiceSetUuid`)만으로 충분히 참조 가능하므로 `@ManyToOne` 관계 자체를 제거하였다.

- `question`, `choiceSet` 연관 객체가 향후 필요해지면 `@JoinColumn`에 `name` 없이 `referencedColumnName`만으로 재도입 검토 필요
- `question_report_category` 복합 PK는 V0_0_126 마이그레이션 성공 이후부터 DB 레벨 중복 차단이 적용됨 (이전 중복 데이터는 직접 확인 필요)
