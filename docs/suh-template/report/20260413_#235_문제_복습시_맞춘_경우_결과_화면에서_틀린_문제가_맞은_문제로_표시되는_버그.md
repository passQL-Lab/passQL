# ❗[버그][문제풀이] 문제 복습시 맞춘 경우 결과 화면에서 틀린 문제가 맞은 문제로 표시되는 버그

## 개요

연습문제 세트 완료 후 결과 화면에서 틀린 문제를 "다시 풀기"로 복습해 정답을 맞추면, Zustand store의 결과값이 덮어써져 결과 화면으로 돌아왔을 때 오답이 초록색(정답)으로 표시되는 버그를 수정했다. 최초 제출 시점의 결과는 불변이어야 하며, 복습은 추가 학습으로만 처리되어야 한다는 원칙에 따라 `markCorrect` 액션 자체를 제거했다.

## 변경 사항

### Store
- `client/src/stores/practiceStore.ts`: `markCorrect` 액션 및 `PracticeActions` 인터페이스 정의 제거

### 페이지
- `client/src/pages/RecommendationPractice.tsx`: `markCorrect` import 및 호출 코드 제거, `useCallback` 의존성 배열에서 `markCorrect` 제거

## 주요 구현 내용

**버그 원인**: `RecommendationPractice`에서 복습 문제를 정답으로 제출하면 `markCorrect(questionUuid)`를 호출해 store의 `results` 배열에서 해당 문제의 `isCorrect`를 `true`로 덮어썼다. `PracticeResult`는 이 store 값을 기반으로 정오답 색상을 렌더링하므로, 결과 화면 복귀 시 원래 틀렸던 문제가 초록색으로 표시됐다.

**수정 방향**: 세션 결과(`results`)는 `submitAndAdvance` 호출 시점에 확정되며 이후 변경되지 않는다. 복습 제출은 `heatmap`, `progress`, `recommendations` 캐시 무효화만 수행하고 세션 결과에는 영향을 주지 않는다.

## 주의사항

- 복습 완료 여부를 결과 화면에 별도로 표시하는 기능(예: "복습 완료" 뱃지)은 현재 미구현 상태로, 추후 별도 이슈로 추가 예정
- `markCorrect` 제거로 인해 해당 액션을 참조하는 다른 코드가 없음을 확인함 (전체 탐색 완료)
