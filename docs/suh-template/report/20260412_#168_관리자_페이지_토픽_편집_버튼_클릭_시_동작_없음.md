# [버그][관리자] 관리자 페이지 토픽 편집 버튼 클릭 시 동작 없음 #168

## 개요

관리자 페이지 토픽/태그 관리 화면에서 편집(연필) 버튼 클릭 시 `alert(code)` 호출만 하고 실제 편집 UI가 열리지 않는 미구현 상태였음. 토픽 편집 모달과 `PUT /admin/topics/{code}` API를 신규 구현하고, 입력값 서버 측 검증 및 비활성 토픽의 관리자 목록 노출 문제를 함께 수정함.

## 변경 사항

### 백엔드 — 편집 API

- `TopicUpdateRequest.java` (신규): 토픽 편집 요청 DTO. `displayName`, `sortOrder`, `isActive` 필드에 `@NotBlank`, `@Min(1)`, `@NotNull` 검증 추가
- `MetaService.java`: `updateTopic(String code, TopicUpdateRequest)` 메서드 추가. 더티 체킹으로 명시적 `save()` 없이 수정 처리
- `AdminTopicController.java`: `PUT /admin/topics/{code}` 엔드포인트 추가. `@Valid`로 요청 검증
- `TopicRepository.java`: `findAllByOrderBySortOrderAsc()` 메서드 추가 (비활성 토픽 포함 전체 조회)

### 백엔드 — 목록 조회 수정

- `MetaService.getTopicTree()`: 기존 활성 토픽만 조회(`findByIsActiveTrueOrderBySortOrderAsc`)에서 전체 조회(`findAllByOrderBySortOrderAsc`)로 변경. 관리자 화면에서 비활성화 후 되돌릴 수 없는 문제 해결

### 프론트엔드

- `topics.html`: 토픽 편집 모달 HTML 추가 (이름, 정렬 순서, 활성 여부 체크박스)
- `topics.html`: `editTopic()` 함수 — 편집 버튼 클릭 시 현재 토픽 값으로 모달 필드를 채우고 `showModal()` 호출
- `topics.html`: `saveTopicEdit()` 함수 — `PUT /admin/topics/{code}` JSON 요청 후 성공 시 페이지 새로고침, 네트워크 오류 시 `.catch()` 처리

## 주요 구현 내용

편집 버튼의 `data-*` 속성(`data-code`, `data-name`, `data-sort`, `data-active`)에 Thymeleaf로 토픽 현재 값을 주입하고, JS에서 이를 읽어 모달 폼에 바인딩한다. 저장은 REST API(`PUT /admin/topics/{code}`) 호출로 처리하며, 서버에서 `@Valid` + Bean Validation으로 입력값을 검증한다. `isActive`는 HTML `data-*` 속성이 문자열이므로 `=== 'true'` 비교로 정확히 처리한다.

## 주의사항

- 관리자 목록에 비활성 토픽도 노출됨에 따라 일반 사용자 대상 API(`/api/topics` 등)에서 활성 필터가 유지되는지 별도 확인 필요
- 토픽 추가(`POST /admin/topics`) 엔드포인트는 HTML 폼에 존재하나 Controller에 미구현 상태 — 별도 이슈로 처리 필요
