# 홈화면 오늘의 문제 문제풀이 UX 및 결과 화면 UI 개선

이슈: passQL-Lab/passQL#106

---

### 📌 작업 개요

결과 화면의 정답 표시 오류(A/B/C/D 라벨 혼용), 데일리 챌린지 플로우의 버튼 텍스트 부적절, 오늘의 문제 완료 카드 UI 미반영 문제를 수정. 정답 후 자동 리다이렉트 버그도 함께 수정.

---

### 🔍 문제 분석

1. **정답 표시 오류** — `AnswerFeedback`의 `SqlCompareBlock` / `TextCompareBlock`에서 선택지 키(A/B/C/D) 배지를 별도 렌더링하고 있었음. 이 배지가 실제 정답 키가 아닌 고정 값으로 표시되어 오표시 발생
2. **버튼 텍스트 부적절** — 데일리 챌린지 완료 후 "다음 문제" 버튼이 표시됨. `isDailyChallenge` 컨텍스트 분기 없이 일괄 `navigate("/questions")` 처리
3. **홈 완료 카드 미반영** — `alreadySolvedToday=true`여도 일반 카드 스타일 그대로 표시. 완료 여부를 시각적으로 구분하지 않음
4. **정답 후 자동 리다이렉트 버그** — `queryClient.invalidateQueries`를 `await`해 refetch가 완료되면서 `alreadySolvedToday=true`가 즉시 반영, 피드백 바가 표시되기 전에 홈으로 이동

---

### ✅ 구현 내용

#### 1. 결과 화면 선택지 키 라벨 제거

- **파일**: `src/pages/AnswerFeedback.tsx`
- **변경 내용**: `SqlCompareBlock` / `TextCompareBlock`에서 `choiceKey` prop 제거. 키 배지 대신 `label` 텍스트("정답 SQL", "내가 선택한 SQL")를 시맨틱 컬러(초록/빨강)로 표시
- **이유**: A/B/C/D 알파벳 라벨이 정답 내용 파악에 불필요하고 오표시 원인이었음

#### 2. 데일리 챌린지 버튼 분기

- **파일**: `src/pages/AnswerFeedback.tsx`
- **변경 내용**: `FeedbackState`에 `isDailyChallenge?: boolean` 추가. 하단 버튼: 데일리 챌린지 + 정답 → "홈으로 가기"(`/`), 오답 → "다시 풀기"(`/daily-challenge`), 일반 → "문제 목록으로"(`/questions`)
- **이유**: 1문제 플로우인 데일리 챌린지에서 "다음 문제"는 맥락에 맞지 않음

#### 3. 홈 오늘의 문제 완료 카드 UI

- **파일**: `src/pages/Home.tsx`
- **변경 내용**: `alreadySolvedToday`가 `true`이면 성공 카드 스타일(초록 left border + 연초록 배경 + 체크 아이콘) 적용. 클릭 불가(Link 제거, `cursor-default`). 미완료 상태는 `/daily-challenge`로 이동
- **이유**: 완료 여부를 시각적으로 즉시 인지 가능하도록 개선

#### 4. DailyChallenge 오답 처리 및 리다이렉트 버그 수정

- **파일**: `src/pages/DailyChallenge.tsx`
- **변경 내용**:
  - 오답 시 `submitAnswer` 호출 안 함 → `alreadySolvedToday=false` 유지 → 다시 풀기 가능
  - `queryClient.invalidateQueries` `await` 제거 → 피드백 바 표시 후 백그라운드에서 캐시 무효화
  - `alreadySolvedToday` 리다이렉트 조건에 `!feedback` 추가 → 피드백 표시 중 리다이렉트 방지
- **이유**: 정답 제출 즉시 `invalidateQueries` 결과가 반영되면서 피드백이 뜨기 전에 홈으로 이동하는 버그 차단

### 🔧 주요 변경사항 상세

#### AnswerFeedback — 비교 블록 구조

키 배지 제거 전/후:

| 이전 | 이후 |
|------|------|
| `label + key 배지(A/B)` | `label만 시맨틱 컬러 텍스트로` |
| `choiceKey` prop 필요 | prop 제거 |

`state`에서 `correctKey`, `selectedKey`를 비교 블록에 전달하지 않으므로 UI에서 키 오표시 자체가 불가능해짐.

#### DailyChallenge — 오답 처리 로직

오답 시 `submitAnswer`를 호출하지 않아 서버의 `alreadySolvedToday`가 `false`로 유지됨. 로컬 `SubmitResult`를 생성해 피드백만 표시하고, 오답 버튼 클릭 시 `setFeedback(null)`로 초기화해 같은 문제를 다시 풀 수 있음.

#### DailyChallenge — 리다이렉트 버그

```
기존 흐름 (버그):
정답 제출 → await invalidateQueries → alreadySolvedToday=true 즉시 반영 → Navigate 실행 → 피드백 미표시

수정 후:
정답 제출 → setFeedback(result) → 피드백 바 표시
            ↘ invalidateQueries (await 없음, 백그라운드)
```

`!feedback` 조건으로 피드백 바가 열려 있는 동안은 `alreadySolvedToday` 리다이렉트를 건너뜀.

### 🧪 테스트 및 검증

- 정답 후 홈 복귀 시 완료 카드(초록) 즉시 반영 확인
- 일반 결과 화면 "문제 목록으로" 버튼 정상 동작
- 결과 화면에서 A/B/C/D 라벨 미표시 확인

### 📌 참고사항

- 오답 시 `submitAnswer`를 호출하지 않아 서버 제출 기록이 남지 않음 — 의도된 동작 (오답은 완료로 처리하지 않음)
- 일반 문제풀기(`/questions/{uuid}`)에서 들어온 경우 `isDailyChallenge`가 `undefined`이므로 기존 동작 유지
