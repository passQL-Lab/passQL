# 🚀 [기능개선][관리자] 문제 관리 페이지 선택 일괄삭제 기능 추가 및 툴바 UI 개선

## 개요

관리자 문제 관리 페이지(`/admin/questions`)에서 체크박스로 선택한 문제를 한 번에 삭제할 수 있는 일괄삭제 기능을 추가하고, 버튼이 과도하게 나열되던 툴바를 드롭다운 메뉴로 정리하여 UI 가독성을 개선했다.

## 변경 사항

### 백엔드

- `AdminQuestionController.java`: `DELETE /admin/questions/bulk` API 추가. UUID 목록을 JSON body로 받아 `bulkDeleteQuestions()`를 호출하고 결과(`deleted`, `skipped`, `errors`)를 JSON으로 반환한다.

- `AdminQuestionDeleteService.java`: 일괄삭제 로직 3개 추가
  - `bulkDeleteQuestions(List<UUID>)`: 순차 반복하며 건별 삭제 시도. `Exception` 전체를 catch하여 JPA 예외 포함 모든 실패를 skipped 처리하고 나머지 삭제는 계속 진행
  - `deleteSingleInNewTransaction(UUID)`: `@Transactional(propagation = REQUIRES_NEW)` 적용. 문제별 독립 트랜잭션을 보장하여 한 건 실패가 다른 건 롤백에 영향을 주지 않음
  - `BulkDeleteResult` record: `deleted`, `skipped`, `errors` 필드를 담는 응답 DTO

### 프론트엔드

- `question-import-export.js`
  - `updateExportButtonState()` 확장: 선택 내보내기 버튼뿐 아니라 선택 삭제 버튼(`deleteSelectedBtn`) 상태도 함께 갱신
  - `deleteSelected()` 추가: 선택 개수를 확인 모달에 표시 후 `bulkDeleteModal` 열기
  - `confirmBulkDelete()` 추가: `DELETE /admin/questions/bulk` 호출 → `resp.ok` 체크를 `resp.json()` 파싱 전에 수행하여 HTML 에러 응답 시 파싱 오류 방지. 실패·예외 시 확인 버튼 상태 복구

- `questions.html`
  - 툴바 재구성: JSON 가져오기·필터 전체 내보내기·선택 내보내기·가이드를 `dropdown` 컴포넌트 하나로 통합
  - 선택 삭제 버튼(`btn-error btn-outline`) 추가 — 체크박스 미선택 시 `disabled`
  - 일괄삭제 확인 모달(`bulkDeleteModal`) 추가 — 삭제 개수 표시, 활성 세션 사용 중 문제는 건너뜀 안내

## 주요 구현 내용

### 트랜잭션 분리 전략

단일 `@Transactional` 안에서 반복 삭제하면 한 건이 예외를 던졌을 때 전체가 롤백된다. 이를 방지하기 위해 `bulkDeleteQuestions`는 트랜잭션 없이 반복하고, 내부의 `deleteSingleInNewTransaction`이 `REQUIRES_NEW`로 매 건마다 독립 커밋을 수행한다.

### Cascade 삭제 순서 (기존 `deleteQuestionCascade` 재사용)

활성 QuizSession 안전장치 확인 → ExecutionLog → Submission → DailyChallenge → QuestionConceptTag → QuestionChoiceSetItem → QuestionChoiceSet → QuestionChoice → Question 순으로 삭제하여 FK 제약을 준수한다.

### 에러 핸들링 개선

`confirmBulkDelete`에서 `resp.ok` 판단을 JSON 파싱 전에 수행한다. 서버가 HTML 에러 페이지를 반환하는 경우에도 파싱 예외 없이 오류 메시지를 표시하고 버튼 상태를 복구한다.

## 주의사항

- `BulkDeleteResult` record가 `AdminQuestionDeleteService` inner type으로 선언되어 있어 Controller에서 `AdminQuestionDeleteService.BulkDeleteResult`로 참조한다. 추후 dto 패키지로 분리하면 가독성이 개선된다.
- 완료/취소된 QuizSession의 `questionOrderJson`에 삭제된 UUID가 JSON으로 남는다. 이미 종료된 세션 이력이므로 기능 영향은 없으나, 세션 조회 시 fallback 처리 여부를 확인해 두는 것이 좋다.
