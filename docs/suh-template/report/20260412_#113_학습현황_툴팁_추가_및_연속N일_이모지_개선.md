# 🚀 [기능개선][홈] 학습현황 툴팁 추가 및 연속 N일 이모지 개선 #113

## 개요

홈화면 학습현황 섹션의 히트맵 각 셀에 데스크톱 hover / 모바일 long press 시 날짜·풀이 수·정답률을 표시하는 커스텀 툴팁을 추가했다. 연속 학습일 배지의 불꽃 아이콘에 `fill="currentColor"`를 적용해 속이 채워진 형태로 시각 강조를 개선했다.

## 변경 사항

### 홈화면 (이모지 개선)
- `client/src/pages/Home.tsx`: 연속 N일 배지의 `<Flame>` 아이콘에 `fill="currentColor"` 추가 — 외곽선만 있던 아이콘을 경고 텍스트 색상으로 꽉 채운 형태로 변경

### 히트맵 컴포넌트 (툴팁 구현)
- `client/src/components/HeatmapCalendar.tsx`: 툴팁 기능 전체 추가 및 성능·버그 개선
  - `TooltipState` 인터페이스 추가 (`date`, `solvedCount`, `correctCount`, `cellRect`)
  - `onMouseEnter/Leave` — 데스크톱 hover 즉시 표시/숨김
  - `onTouchStart/End/Move` — 모바일 500ms long press 판정, 스크롤 시 타이머 취소
  - 툴팁 위치: 셀의 `getBoundingClientRect()` 기준으로 컨테이너 내 절대 위치 계산, 셀 상단 중앙에 표시
  - 툴팁 하단 꼬리 삼각형으로 대상 셀 명확화
  - `buildDayMap` 반환 타입 `Map<string, number>` → `Map<string, HeatmapEntry>`로 변경해 `correctCount` 접근 가능
  - `formatDateKo`: `new Date()` 파싱 대신 문자열 슬라이싱으로 타임존 버그 수정
  - `getLast30Days`: `toISOString()` 대신 로컬 타임존 기준 날짜 생성으로 수정
  - `dayLabel` 추출도 문자열 슬라이싱(`date.slice(8, 10)`)으로 통일
  - `buildDayMap`, `getLast30Days`, `tooltipStyle` 모두 `useMemo` 적용

## 주요 구현 내용

**툴팁 위치 계산**: 툴팁을 `position: absolute`로 렌더링하고, 셀과 컨테이너의 `getBoundingClientRect()` 차이로 컨테이너 기준 상대 좌표를 계산한다. `transform: translateX(-50%) translateY(-100%)`으로 셀 상단 중앙에 정렬한다. `tooltip` state가 변경될 때만 `useMemo`로 재계산해 불필요한 DOM 접근을 최소화한다.

**모바일 long press**: `onTouchStart`에서 500ms 타이머를 시작하고, `onTouchEnd`와 `onTouchMove` 모두에서 타이머를 취소한다. 스크롤 의도를 long press로 오인하지 않도록 `onTouchMove` 취소가 핵심이다.

**타임존 버그 수정**: `new Date("YYYY-MM-DD")`는 UTC 자정으로 파싱되어 UTC+9 환경에서 하루 밀림이 발생한다. 날짜 생성은 로컬 타임존 기준 수동 포맷으로, 날짜 표시는 문자열 슬라이싱으로 전환해 일관성을 확보했다.

## 주의사항

- 툴팁은 `pointer-events-none`이라 마우스 이벤트를 가로채지 않는다 — `onMouseLeave` 오작동 없음
- 풀이가 없는 날(`solvedCount === 0`)은 "풀이 없음" 텍스트로 표시해 0 나누기 방지
- 히트맵 데이터는 API(`/progress/heatmap`) 응답 기준이므로 `correctCount`가 `solvedCount`를 초과하는 경우는 발생하지 않음
