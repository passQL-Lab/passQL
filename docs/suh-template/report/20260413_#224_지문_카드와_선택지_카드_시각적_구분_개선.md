# 지문 카드와 선택지 카드 시각적 구분 개선 #224

## 📌 작업 개요

문제 풀기 화면(`QuestionDetail`)에서 지문 카드가 선택지 카드와 동일한 시각적 스타일을 가져 구분이 어려운 UX 문제 해결. 지문 카드에 항상 노출되는 라벨과 C1 디자인(인디고 배경/보더)을 적용하여 시각적 위계를 명확히 구분. 연관 컴포넌트(SchemaViewer, SqlPlayground)도 함께 개선.

---

## 🔍 문제 분석

지문이 펼쳐진 상태에서 지문 카드와 선택지 카드가 동일한 흰색 카드 스타일(`bg-surface-card border border-border`)로 표시되어 지문이 첫 번째 선택지처럼 보이는 문제 발생.

기존에는 접힌 상태에서만 `BookOpen` 아이콘이 표시되고, 펼치면 아이콘이 사라지는 구조였음. 결과적으로 펼친 상태에서는 지문임을 식별할 수 있는 시각적 단서가 전혀 없었음.

---

## ✅ 구현 내용

### 1. 지문 카드 라벨 상시 노출

- **파일**: `client/src/pages/QuestionDetail.tsx`
- **변경 내용**: `BookOpen` 아이콘을 접힌 상태에서만 조건부 렌더링하던 구조를 제거하고, 접힘/펼침 무관하게 항상 표시되는 `문제` 라벨 블록으로 교체
- **결과**: 12px `BookOpen` 아이콘 + "문제" 텍스트(xs, semibold, uppercase, tracking-wide)로 구성된 상단 라벨이 항상 노출됨

```tsx
{/* Before: 접힌 상태에서만 표시 */}
{!stemOpen && (
  <BookOpen size={16} className="text-brand mt-0.5 shrink-0" />
)}

{/* After: 항상 표시되는 라벨 */}
<div className="flex items-center gap-1.5 mb-2 text-brand text-xs font-semibold uppercase tracking-wide">
  <BookOpen size={12} className="shrink-0" />
  문제
</div>
```

### 2. 지문 카드 C1 디자인 적용

- **파일**: `client/src/pages/QuestionDetail.tsx`
- **변경 내용**: 지문 카드 배경을 `bg-surface-card`(흰색)에서 `bg-brand-light`(#EEF2FF, 인디고 연배경)로, 보더를 `border-border`(회색)에서 `border-brand-medium`(인디고 계열)으로 변경
- **결과**: 선택지 카드(흰색/회색 보더)와 지문 카드(인디고 배경/보더)가 색상으로 명확히 구분됨

### 3. 스키마 뷰어 샘플 데이터 카드 헤더 추가

- **파일**: `client/src/components/SchemaViewer.tsx`
- **변경 내용**: 샘플 데이터 카드 상단에 `Table2` 아이콘 + 테이블명을 표시하는 헤더 추가. 스키마 구조 카드와 동일한 스타일(`bg-accent-light`, `text-brand`, `font-mono`) 적용
- **이유**: 스키마 카드와 샘플 데이터 카드의 시각적 일관성 확보. 팀에서 제안한 "Table 아이콘 + 테이블명 헤더" 패턴 적용

### 4. SQL 플레이그라운드 헤더 클릭 영역 개선

- **파일**: `client/src/components/SqlPlayground.tsx`
- **변경 내용**: 컨테이너 `div`에 걸려있던 패딩을 헤더 `button` 안으로 이동. 헤더 버튼 전체 영역에 `hover:bg-surface transition-colors` 추가
- **결과**: 헤더 텍스트뿐만 아니라 헤더 전체 행이 클릭 가능 영역으로 확장됨. 아코디언 열림/닫힘 클릭 UX 개선

---

## 🧪 테스트 및 검증

- 지문 카드가 접힌/펼친 두 상태 모두에서 `BookOpen` + "문제" 라벨 표시 확인
- 지문 카드(인디고 배경)와 선택지 카드(흰색 배경) 시각적 구분 확인
- 스키마 뷰어에서 샘플 데이터 헤더 렌더링 확인
- SQL 플레이그라운드 헤더 전체 영역 클릭 동작 확인

## 📌 참고사항

- `bg-brand-light`, `border-brand-medium` 클래스는 프로젝트 디자인 토큰(`#EEF2FF`, `#818CF8`)으로 정의된 커스텀 클래스
- 선택지 카드(`ChoiceCard`)는 별도 수정 없이 기존 흰색 카드 스타일 유지 — 지문 카드와의 대비가 자연스럽게 형성됨
