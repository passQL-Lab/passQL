# 🎨 AI 리포트 화면 패딩 이상 및 버튼 스타일 공통화

**이슈**: [#205](https://github.com/passQL-Lab/passQL/issues/205)

---

### 📌 작업 개요

AI 리포트(`PracticeResult`) 화면에서 발생하던 좌우 여백 이중 적용 문제와 버튼 정렬 불일치, 버튼 스타일 공통화 누락을 일괄 수정.
패딩 중복 구조를 해소하고 `btn-primary` 공통 클래스를 적용하여 앱 전체 버튼 일관성 확보.

---

### 🔍 문제 분석

**이중 패딩 원인**

`PracticeResult`의 컨테이너가 `px-4`(16px)를 가지고, 내부 `StepNavigator`의 step 패널도 `px-4`(16px)를 중복 적용 → 실제 좌우 여백 32px.
추가로 `sm:px-0` 분기가 존재해 sm+ 환경에서는 컨테이너 패딩이 0이 되고 step 패널의 px-4만 살아있는 구조 — 반응형 의도와 패딩 책임이 두 레이어에 분산.

**버튼 위치 이상 원인**

`StepNavigator` 하단 버튼 래퍼에 `px-6`(24px)이 적용되어, 컨테이너 px-4까지 합산하면 버튼이 콘텐츠보다 40px 안쪽에 위치.

**버튼 스타일 공통화 누락**

`components.css`의 `.btn-primary`(height 44px / radius 8px)가 정의되어 있음에도 `StepNavigator` 다음 버튼과 `PracticeFeedbackBar` 계속하기/확인 버튼이 각각 인라인 Tailwind(`h-12 rounded-xl` = 48px / 12px)로 구현되어 시각적 불일치 발생.

---

### ✅ 구현 내용

#### PracticeResult.tsx — 컨테이너 패딩 단일화

- **파일**: `src/pages/PracticeResult.tsx`
- **변경 내용**: `px-4 sm:px-0` → `px-4`
- **이유**: step 패널 패딩을 제거하면서 sm+ 환경에서 패딩 책임을 컨테이너 단일 레이어로 통합. `sm:px-0`은 step 패널의 `px-4`와 세트로 동작하는 구조였으므로 함께 제거.

#### StepNavigator.tsx — 패딩 제거 및 버튼 공통화

- **파일**: `src/components/StepNavigator.tsx`
- **변경 내용**:
  - step 패널 `px-4` 제거 — 좌우 패딩 책임을 부모 컨테이너로 위임
  - 버튼 래퍼 `px-6` 제거 — 버튼 정렬선을 콘텐츠와 일치
  - 다음/마지막 버튼 클래스 `w-full h-12 bg-brand text-white font-bold rounded-xl` → `btn-primary w-full`
- **이유**: `StepNavigator`는 범용 컴포넌트이므로 내부에서 패딩을 강제하지 않고 사용처 컨테이너에 패딩 제어권을 위임하는 방향이 적합.

#### PracticeFeedbackBar.tsx — 버튼 스타일 통일

- **파일**: `src/components/PracticeFeedbackBar.tsx`
- **변경 내용**:
  - 다시 풀기 버튼: `flex-1 h-12 rounded-xl font-bold text-base text-white bg-brand` → `btn-primary flex-1`
  - 계속하기/확인 버튼: `h-12 rounded-xl` → `h-11 rounded-lg` (정답/오답 색상 로직은 유지)
- **이유**: 다시 풀기 버튼은 `bg-brand` 단색으로 `btn-primary`와 완전히 동일하여 직접 교체. 계속하기/확인 버튼은 정답/오답에 따라 초록·빨강 배경을 사용하므로 색상 클래스는 유지하되 높이(48→44px)와 radius(12→8px)만 `btn-primary` 기준으로 맞춤.

### 🔧 주요 변경사항 상세

#### 패딩 책임 분리 원칙

수정 전에는 컨테이너(`PracticeResult`)와 콘텐츠 래퍼(`StepNavigator` step 패널) 양쪽에 패딩이 분산되어 있었음. 수정 후 컨테이너 `px-4` 단일 레이어로 책임 통합 — 어느 환경에서도 좌우 16px 보장.

#### FeedbackBar 다음 버튼 처리

`btn-primary`는 `background: var(--color-brand)`로 고정되어 있어 정답(초록)/오답(빨강) 상태 버튼에 직접 적용 불가. 색상 시맨틱을 유지하면서 크기 스펙만 `btn-primary` 기준으로 정렬(`h-11 rounded-lg`).
