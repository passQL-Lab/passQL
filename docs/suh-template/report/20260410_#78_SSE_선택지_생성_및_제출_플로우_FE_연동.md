### 📌 작업 개요

BE #69에서 구현한 SSE 선택지 생성 엔드포인트(`POST /questions/{uuid}/generate-choices`)와 `choiceSetId` 기반 제출 플로우를 FE에 연동한다. 기존 FE는 `choiceSets[]`에서 직접 선택지를 사용하고 `submitAnswer`가 2인자였으나, 이번 작업으로 SSE 스트림 파싱 → `choiceSetId` 획득 → 3인자 제출 흐름으로 전환한다.

**이슈**: passQL-Lab/passQL#78

---

### 🎯 구현 목표

- `generateChoices` SSE 함수 추가 — `EventSource` 대신 `fetch + ReadableStream` 방식 (커스텀 헤더 전송 필요)
- `submitAnswer` 시그니처 변경 — `(questionUuid, selectedChoiceKey)` → `(questionUuid, choiceSetId, selectedChoiceKey)`
- `SubmitResult` 타입에 `selectedResult`, `correctResult`, `correctSql`, `selectedSql` 필드 추가 (CONCEPT_ONLY 시 null)
- SSE 이벤트 타입(`SseStatusEvent`, `SseErrorEvent`, `ChoiceSetGenerateResponse`) 신규 정의
- 호출부(훅, 페이지, PracticeSet) 전체를 새 시그니처에 맞게 업데이트
- 계약 테스트로 스펙 고정

### 🔧 주요 변경사항 상세

#### generateChoices — fetch + ReadableStream SSE 구현

`\n\n`(빈 줄)으로 이벤트 청크를 구분하고, `event:` / `data:` 라인을 각각 파싱한다.
`complete` 이벤트 수신 시 `choiceSetId`와 `choices[]`를 콜백으로 전달. `AbortController`를 반환해 컴포넌트 unmount 시 스트림 중단 가능.

**특이사항**:
- `AbortError`는 정상 cleanup이므로 콜백 호출 없이 무시
- 네트워크 에러나 JSON 파싱 실패 시 `onError` 콜백에 `{ code: "STREAM_ERROR", retryable: true }` 전달
- BE 타임아웃 60초 — 화면에서 로딩 상태 피드백 필수

#### SubmitResult nullable 필드 처리

`selectedResult`, `correctResult`, `selectedSql`, `correctSql`은 CONCEPT_ONLY 문제에서 모두 `null`. 결과 화면에서 SQL 비교 UI를 렌더링할 때 반드시 null 체크 후 조건부 렌더링 처리.

```typescript
// EXECUTABLE: non-null
// CONCEPT_ONLY: 모두 null
readonly selectedResult: ExecuteResult | null;
readonly correctResult: ExecuteResult | null;
readonly correctSql: string | null;
readonly selectedSql: string | null;
```

#### choiceSetId 취득 경로

SSE를 사용하는 경우: `generateChoices` → `onComplete` 콜백의 `choiceSetId`를 state에 보관 → `submitAnswer` 호출 시 전달.

SSE 없이 기존 `choiceSets[]`를 사용하는 경우: `QuestionDetail.choiceSets[0].choiceSetUuid`를 `choiceSetId`로 전달 (ADMIN_SEED 선택지 기반 폴백 경로).

### 📌 참고사항

- BE SSE 타임아웃은 60초. 사용자에게 "선택지 생성 중" 로딩 피드백 없으면 UX 문제 발생 가능.
- `choiceSets[]` 구조는 여전히 구 `QuestionChoice` 테이블 기반. ADMIN_SEED 폴백 시 `choiceSets[0].choiceSetUuid`를 `choiceSetId`로 사용.
- `SubmitResult.correctSql` / `selectedSql`이 BE에서 직접 내려오므로, `navigate state`로 SQL을 전달하던 기존 방식은 더 이상 불필요.
