### 📌 작업 개요
설정 탭에 풀(full) UI로 펼쳐져 있던 건의사항 폼(FeedbackForm + FeedbackList)을 `/settings/feedback` 독립 서브페이지로 분리. 설정 탭은 순수 row 목록 구조로 정리하고, 건의사항 섹션은 ChevronRight 탐색 row로 교체.

### 🔍 문제 분석
설정 탭이 "기능 페이지" 역할을 겸하고 있어 항목이 늘어날수록 탭이 무거워지는 구조적 문제. iOS 설정 앱, 토스 등 표준 모바일 패턴과 달리 설정 탭이 입력 폼을 직접 포함하고 있었음.

- **이유**: DevPage와 SettingsFeedback 두 서브페이지가 동일한 헤더 패턴 공유 → DRY 원칙 적용. DevPage도 SubpageLayout으로 전환

### 🔧 주요 변경사항 상세

#### 레이아웃 콜랩스 버그 수정
초기 구현에서 콘텐츠 영역에 `max-w-lg mx-auto`를 적용했으나 명시적 `width` 없이 블록이 중앙 수축되어 텍스트가 세로로 좁은 띠에서 렌더링되는 버그 발생. DevPage의 `py-6 px-4` 패턴과 동일하게 통일하여 해결.

**특이사항**:
- #208 브랜치가 작업 시작 전 main에 이미 머지 → `git stash && git rebase origin/main`으로 최신화 후 App.tsx 충돌 수동 해소 (DevGuard + SettingsFeedback 양쪽 변경 병합)

#### SettingsRow 접근성 개선 (코드리뷰 반영)
- **파일**: `src/components/SettingsRow.tsx`
- **변경 내용**: `onClick` prop 유무에 따라 조건부 렌더링 — `onClick` 있으면 `<button type="button">`, 없으면 `<div>` 유지
- **이유**: 기존 `<div onClick>` 구조는 키보드/스크린리더 접근 불가. `<button>`으로 변경 시 Enter/Space 키 동작과 포커스가 자동 보장됨. `action`에 실제 `<button>`이 있는 행(닉네임, 디바이스ID)은 `onClick` 없는 패턴이므로 button-in-button 중첩 방지

#### MobileHeader 아이콘 원형 배경 수정
- **파일**: `src/components/MobileHeader.tsx`
- **변경 내용**: `bg-accent-light` → `bg-brand-light`, `hover:bg-indigo-100` → `hover:bg-brand-medium/20`
- **이유**: `bg-accent-light`는 tokens.css에 미정의 클래스 — 배경이 투명하게 렌더링되어 원형이 보이지 않던 문제. `bg-brand-light`(`#EEF2FF`)가 올바른 디자인 토큰
