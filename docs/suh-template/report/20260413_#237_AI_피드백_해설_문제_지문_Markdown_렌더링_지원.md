# 🚀 [기능개선][마크다운] AI 피드백, 해설·문제 지문에 대한 Markdown 형식 렌더링 지원

## 개요

AI 피드백(PracticeResult), 해설(PracticeFeedbackBar), AI 한마디(StatsAnalysisCard), 문제 지문(QuestionDetail) 4곳에 Markdown 렌더링을 통일 지원한다. 기존 `useAiText` 훅은 `innerHTML` 직접 조작 방식이라 `ReactMarkdown`과 충돌하는 구조적 문제가 있었으며, 이를 React Context + CSS custom property 기반의 `MarkdownText` 공용 컴포넌트로 대체하여 단어별 fade-in 애니메이션을 유지하면서 Markdown 렌더링을 동시에 지원한다.

## 변경 사항

### 신규 생성

- `client/src/components/MarkdownText.tsx`: 공용 Markdown 렌더링 컴포넌트. `animated` prop으로 단어별 fade-in 켜고 끄기. `ReactMarkdown`의 `components` prop으로 passQL 디자인 시스템 클래스(`bg-surface-code`, `font-mono`, `font-bold` 등) 주입.

### 스타일

- `client/src/styles/components.css`: `.md-word` 클래스 추가. CSS custom property `--i`를 이용해 `animation-delay: calc(var(--i, 0) * 55ms + 200ms)` 계산. 기존 `ai-word-in` 키프레임 재사용.

### 사용처 교체

- `client/src/pages/PracticeResult.tsx`: `useAiText` 훅 및 `aiTextRef` 제거 → `<MarkdownText text={aiComment as string} animated />` 적용
- `client/src/components/PracticeFeedbackBar.tsx`: `<p>{result.rationale}</p>` → `<MarkdownText text={result.rationale} />` 교체 (애니메이션 없이 정적 렌더링)
- `client/src/components/StatsAnalysisCard.tsx`: `useAiText` 훅 제거 → `<MarkdownText text={comment} animated />` 적용
- `client/src/pages/QuestionDetail.tsx`: 인라인 `ReactMarkdown` + 커스텀 components 제거 → `<MarkdownText text={question.stem} />` 통일

## 주요 구현 내용

**충돌 해결 방식**: 기존 `useAiText`는 `container.innerHTML = ""`로 DOM을 직접 초기화한 뒤 단어별 `<span>`을 추가하는 방식이었다. 이 방식은 React가 관리하는 DOM 외부에서 조작이 이루어져 `ReactMarkdown`과 동시에 사용할 수 없었다.

**새 방식**: `ReactMarkdown`의 `components` prop으로 `p`, `strong`, `em`, `code`, `pre`, `ul`, `ol`, `li` 각 요소를 커스텀 컴포넌트로 교체한다. `animated=true`이면 `WordIndexContext`(React Context)로 전역 단어 인덱스 카운터를 공유하고, 각 커스텀 컴포넌트가 텍스트 노드를 `<span className="md-word" style={{ "--i": idx }}>` 단위로 분해한다. CSS가 `--i` 값을 읽어 `animation-delay`를 자동 계산하므로 DOM 직접 조작 없이 동일한 fade-in 효과를 구현한다.

**성능 고려**: `MD_COMPONENTS`를 모듈 스코프 상수로 선언하여 매 렌더마다 새 객체가 생성되어 ReactMarkdown이 불필요한 리렌더를 발생시키는 문제를 방지한다. 인라인 코드(`<code>`)는 단어 분해 없이 통째로 렌더링하여 코드 토큰 분리에 따른 의미 손실을 방지한다.

## 주의사항

- `style={{ "--i": idx }}` 인라인 style 사용: CLAUDE.md의 인라인 style 금지 규칙의 예외 케이스. CSS custom property 동적 주입을 위한 유일한 수단으로 `MarkdownText` 컴포넌트에 한해 허용.
- `useAiText.ts` 파일이 아직 남아 있음 — 모든 참조가 제거된 데드코드 상태. 팀 확인 후 삭제 예정.
- AI가 실제로 Markdown 문법을 얼마나 사용하는지 미확인 — 기본 문법(`**굵게**`, `` `코드` ``, 줄바꿈)은 지원하며, 헤딩·표 등 추가 문법은 필요 시 `MD_COMPONENTS`에 요소 추가로 확장 가능.
