### 📌 작업 개요

데일리 챌린지 및 홈 추천 문제 플로우에 #144에서 개선한 UI/UX 패턴을 통일 적용.
`PracticeFeedbackBar`, `LoadingOverlay`, `showExecution` 패턴을 두 화면 모두에 일관되게 반영하고,
홈 추천 문제 진입을 전담하는 `RecommendationPractice` 페이지를 신규 생성.

---

### 🎯 구현 목표

- PracticeResult에서 오답 재풀기 후 돌아가기 시 step3 복귀 지원
- 연타·피드백바 탭-스루로 인한 문제 스킵 버그 방어

### 🔧 주요 변경사항 상세

#### 피드백바 탭-스루 버그 방어

피드백바의 "다시 풀기" 버튼 클릭 후 패널이 닫히는 300ms 사이에 QuestionDetail의 초이스카드가 즉시 눌려버리는 문제 방어.
`RecommendationPractice`에서는 `onSecondary` 클릭 시 `isProcessingRef.current = true`를 먼저 세팅하고 300ms 쿨다운 후 해제.
`DailyChallenge`에서는 `retryCount` 증가 → QuestionDetail 리마운트로 ref 초기화.

#### PracticeResult 복귀 시 step3 유지

PracticeResult(연습 결과 화면)에서 오답 재풀기 → 정답 처리 후 돌아가기 시 step3(결과 화면)으로 복귀해야 하는 요구사항.
`location.state`로 `initialStep`을 전달했으나 React Router v7에서 state 전달이 불안정한 이슈 발생.
`practiceStore`에 `returnStep` + `setReturnStep` + `clearReturnStep`을 추가하고, 복귀 직전 store에 step 정보를 기록하는 방식으로 우회.

#### PracticeFeedbackBar 이중 클릭 방지

`hasActedRef`를 컴포넌트 내부에 두어 버튼 언마운트 전 이중 클릭 방지.
부모가 언마운트하면 ref가 자동 초기화되므로 별도 cleanup 불필요.

### 📌 참고사항

- `DailyChallenge`는 오답 시 `submitAnswer`를 호출하지 않음 — 미완료 상태 유지로 당일 재풀기 가능
- `RecommendationPractice`는 정답/오답 무관 `submitAnswer` 항상 호출 — 연습 이력 기록 목적
- `practiceStore.returnStep`은 복귀 후 `PracticeResult` 진입 시 소비되어 초기화됨
