# 🚀 [기능개선][통계] 통계 페이지 UI 전면 개선 필요

## 개요

통계 페이지의 상단 요약 카드 아이콘 부재, AI 분석 카드 레이아웃 어색함, 바 차트 가독성 문제를 전면 개선했다. recharts 기반 바 차트를 제거하고 커스텀 토픽 리스트로 교체했으며, 합격 준비도·토픽별 학습 현황에 팝오버 설명 버튼을 추가했다.

## 변경 사항

### 페이지

- `src/pages/Stats.tsx`: 상단 요약 카드 3개(푼 문제·합격 준비도·연속 학습)에 lucide-react 아이콘 추가. `StatsBarChart` → `StatsTopicList` 교체. 합격 준비도 라벨 옆 `?` 버튼 및 `ReadinessPopover` 컴포넌트 구현 (createPortal 기반, 정답률·범위·꾸준함 세부 미니 바 포함). 팝오버 위치 계산 시 `fixed` 포지셔닝 기준으로 `scrollX/Y` 제거.

### 컴포넌트

- `src/components/StatsAnalysisCard.tsx`: 아이콘을 `Lightbulb` → `Sparkles`로 교체. 아이콘+제목 한 줄 인라인 레이아웃으로 변경. 단어 단위 fade-in 애니메이션(`animateWords`) 구현 — 텍스트를 `span.ai-word`로 분리 후 55ms stagger로 순차 표시. 제목 `AI 한마디`로 확정.
- `src/components/StatsTopicList.tsx` (신규): recharts 바 차트 대체 컴포넌트. 막대 길이는 푼 문제 수 기준, 색상 뱃지는 정답률 구간별(80%↑ 초록 / 60%↑ 인디고 / 40%↑ 노랑 / 40%↓ 빨강). 미학습 토픽 회색 처리 및 "미학습" 라벨 표시. 하단 범례 추가. 제목 옆 `?` 버튼 및 `TopicListPopover` 구현 (createPortal 기반).
- `src/components/StatsRadarChart.tsx`: 카드 제목 `영역별 분석` → `토픽별 분석`으로 수정, 폰트 `text-lg` → `text-base`로 통일.

### 스타일

- `src/styles/components.css`: 팝오버·AI 카드용 애니메이션 추가 — `ai-word-in`(단어 fade-in), `ai-header-slide`(헤더 슬라이드인), `sparkle-pop`(아이콘 팝업), `popover-in`(팝오버 페이드인).

## 주요 구현 내용

**createPortal 팝오버 패턴**: `animate-stagger`에 `animation` 속성이 걸리면 새로운 stacking context가 생성되어 내부 z-index가 격리된다. 이를 해결하기 위해 두 팝오버 모두 `createPortal`로 `document.body`에 렌더링하고, `getBoundingClientRect()`로 버튼 위치를 계산해 `fixed` 포지셔닝으로 배치했다. 화살표는 `arrowLeft = btnCenterX - finalLeft - 6`으로 동적 계산해 버튼 중앙을 항상 가리키도록 했다.

**단어 단위 AI 애니메이션**: `animateWords()` 함수가 텍스트를 공백 기준으로 토큰 분리 후 각 단어를 `span.ai-word`로 감싼다. `useEffect`로 comment 변경 시마다 재실행하며, 단어 인덱스 × 55ms delay로 왼쪽부터 순차 출현한다.
