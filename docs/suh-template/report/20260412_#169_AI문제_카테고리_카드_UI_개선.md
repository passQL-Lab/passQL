# 🎨[디자인][AI문제] AI문제 풀기 카테고리 카드 UI 개선

## 개요

AI문제 풀기 화면(`/questions`)의 토픽 카테고리 카드가 단순하고 밋밋하다는 피드백에 따라 카드 UI를 전면 개선했다. 아이콘 배경 박스를 명확하게 표시하고, 웹/모바일 양 환경에서 인터랙션 피드백을 추가했다. 또한 카드 크기에 맞지 않게 긴 토픽 이름 4개를 Flyway로 축약했다.

## 변경 사항

### 프론트엔드

- `client/src/pages/CategoryCards.tsx`
  - 카드 버튼 클래스를 `card-base` + 인라인 Tailwind 조합에서 `card-topic` 단일 커스텀 클래스로 교체
  - 아이콘 감싸는 div를 인라인 Tailwind에서 `card-topic-icon` 커스텀 클래스로 교체
  - 아이콘 크기 22px → 26px 확대

- `client/src/styles/components.css`
  - `.card-topic` 클래스 신규 추가: `border-radius: 16px`, `padding: 20px 12px 16px`, flex 정렬, `transition: all 250ms`
  - `.card-topic:hover`: `translateY(-3px)` + 인디고 틴트 그림자(`rgba(79,70,229,0.12)`) + 테두리 색상 `#818CF8`
  - `.card-topic:active`: 모바일 터치 피드백 — `scale(0.98)` + `translateY(-1px)` + 그림자
  - `.card-topic-icon` 클래스 신규 추가: 52×52px, `border-radius: 14px`, `background-color: var(--color-brand-light)` (Tailwind `bg-accent-light` 미정의 문제 해결)

### 백엔드 (DB)

- `server/PQL-Web/src/main/resources/db/migration/V0_0_94__shorten_topic_display_names.sql`
  - 카드 UI에 맞게 너무 긴 토픽 이름 4개 축약:
    - `데이터 모델링의 이해` → `데이터 모델링`
    - `SQL 함수 (문자/숫자/날짜/NULL)` → `SQL 함수`
    - `조인 (JOIN)` → `JOIN`
    - `계층 쿼리 / PIVOT` → `계층 / PIVOT`

## 주요 구현 내용

기존에 `bg-accent-light`라는 Tailwind 클래스를 사용하고 있었으나 이 클래스가 프로젝트에 정의되지 않아 아이콘 배경 박스가 렌더링되지 않는 문제가 있었다. 이를 CSS 커스텀 클래스(`.card-topic-icon`)로 분리하고 `var(--color-brand-light)` CSS 변수를 직접 참조하도록 수정해 해결했다.

모바일 환경 대응은 `:active` pseudo-class를 활용했다. 터치 디바이스에서는 `:hover`가 동작하지 않거나 지연되는 경우가 있어 `:active`로 즉각적인 시각 피드백을 제공한다.

## 주의사항

- 토픽 이름 변경은 Flyway 마이그레이션(`V0_0_94`)으로 처리되므로 서버 재시작 시 자동 적용된다
- 관리자 페이지 토픽 편집 버튼이 미구현 상태(`alert`만 호출)임을 별도 이슈(#170)로 등록했다
