# Vercel 배포 워크플로우 빌드 검증 단계 추가

## 개요

GitHub Actions에서 Vercel 배포 시 빌드 검증 없이 배포가 진행되어 TypeScript/Vite 빌드 오류가 있어도 Success로 표시되던 문제를 해결했다. 배포 전 빌드 검증 단계를 추가하고, 워크플로우 파일명을 프로젝트 컨벤션에 맞게 변경했으며, main 브랜치 push 시 React 빌드 체크 워크플로우를 별도로 추가했다.

## 변경 사항

### CI/CD 워크플로우

- `.github/workflows/Vercel-deploy.yml` → 삭제 예정 (파일명 컨벤션 미준수)
- `.github/workflows/PROJECT-COMMON-VERCEL-DEPLOY.yml`: 기존 Vercel 배포 워크플로우를 컨벤션에 맞게 재작성. `npm ci` → `npm run build` 빌드 검증 단계 추가. 빌드 실패 시 Vercel 배포 단계 실행 차단
- `.github/workflows/PROJECT-COMMON-REACT-BUILD-CHECK.yml`: 신규 추가. `main` 브랜치 push/PR 시 `client/**` 경로 변경 감지 → 빌드 체크 자동 실행

## 주요 구현 내용

**빌드 검증 단계 추가 (VERCEL-DEPLOY)**

`deploy` 브랜치 push 시 기존에는 `vercel deploy` 명령을 바로 실행했으나, 이제 아래 순서로 실행된다:
1. `actions/setup-node@v4` (Node 20, npm 캐시)
2. `npm ci` — lock 파일 기반 의존성 설치
3. `npm run build` — `tsc && vite build` 실행. 이 단계 실패 시 이후 단계 중단
4. Vercel CLI 설치 및 배포

**main 빌드 체크 (REACT-BUILD-CHECK)**

`client/**` 하위 파일 변경이 감지된 경우에만 트리거되어 불필요한 Actions 소비를 방지한다. `push`와 `pull_request` 이벤트 모두 대응하여 PR 단계에서도 사전 검증이 가능하다.

**파일명 컨벤션 통일**

기존 `Vercel-deploy.yml`을 `PROJECT-COMMON-VERCEL-DEPLOY.yml`로 변경하여 다른 워크플로우 파일(`PROJECT-COMMON-SUH-ISSUE-HELPER-MODULE.yml`, `PROJECT-COMMON-TEMPLATE-UTIL-VERSION-SYNC.yml`)과 네이밍 컨벤션을 통일했다.

## 주의사항

- 기존 `Vercel-deploy.yml`은 `git rm`으로 삭제 후 커밋 필요 (현재 워킹 디렉토리에 잔존)
- `PROJECT-COMMON-VERCEL-DEPLOY.yml`의 Deployment Summary에서 `job.status` 조건이 Summary 스텝 실행 시점 기준으로 평가되므로, 빌드 스텝 outcome을 직접 참조하는 방식으로 추후 개선 여지 있음
