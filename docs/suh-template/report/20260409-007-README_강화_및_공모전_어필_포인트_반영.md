# README 강화 및 AI 공모전 어필 포인트 반영

## 개요

AI 교육 공모전 제출을 목표로 `README.md`를 **기능 나열 중심**에서 **사업성·포지셔닝 중심의 설득형 문서**로 전면 재작성했다. "시장이 크다"는 단편적 어필 대신, **왜 돈이 되는 구조인가**를 공식 통계·가격 비교·매출 시나리오로 입증하는 구성으로 재편했다. 더불어 내부 구현 용어(MariaDB, Gemini, bge-m3)를 외부 공개에 적합한 상위 개념(Oracle DB, AI Agent, Gemma Embedding)으로 통일했다.

## 변경 사항

### 상단 구조

- `README.md` (상단): 헤드라인을 **"읽는 SQL 공부는 끝. 직접 실행하며 합격하세요"** 로 교체. 부제에 "AI Agent가 문제 생성 + 실제 Oracle DB 실행" 핵심 가치 명시.
- `README.md` (링크 섹션): 버전 블록 하단에 **서비스 메인 / 관리자 화면 / Swagger UI / API Docs(JSON)** 4개 링크를 한 블록으로 배치. 심사위원이 즉시 동작 서비스·API 문서로 진입 가능.

### 제품 소개 파트

- `README.md` (§passQL은 무엇인가): 5초 안에 파악되는 **한 문장 소개 + bullet 5개** 로 압축. 클로저로 **"문제은행이 아니라, 실행형 훈련기입니다"** 포지셔닝 한 줄 배치.
- `README.md` (§기존 공부와 뭐가 다른가): 비교표 1개로 "고정 문제 → 매번 생성 / 읽기 → 실행 / 해설 부재 → AI 해설 / 교재 여러 권 → 월 5,000원" 4축 정리.
- `README.md` (§핵심 기능): 6개 기능을 **아이콘 + 한 줄 설명** 형태로 간결화. 장황한 부연 설명 제거.

### 사업성 파트 (핵심 재작성)

- `README.md` (§왜 사업성이 있는가): 기존 1개 섹션을 **8개 하위 섹션**으로 재구조화.
  1. **이미 검증된 시장** — 2025년 PQI 공식 통계 (SQLD 접수 88,440 / 응시 62,664, SQLP 접수 1,656 / 응시 1,086, 합계 90,096 / 63,750). 2020→2025 SQLD 접수 540% 성장 수치 추가.
  2. **반복적으로 열리는 시장** — SQLD 연 4회 × SQLP 연 2회 시행 구조를 구독 모델 당위성으로 연결.
  3. **교재보다 싼 가격** — 전자책 1.7만~2.1만 / 종이책 1.9만~2.6만 / 강의 8.9만~13.9만 vs **월 5,000원** 병기.
  4. **매출 시나리오** — 전환율 1%/3%/5%별 월·연 매출 테이블. 기존 "1%만 구독해도 연 수억" 과장 문구를 **1% 전환 → 월 450만 원 / 연 5,405만 원**으로 사실 기반 수치로 교체.
  5. **낮은 콘텐츠 원가** — 반자동 문제 생산 구조(출제 가이드 + AI 생성 + 샌드박스 검증)로 기존 교육 서비스의 인건비 원가 구조와 차별화.
  6. **B2B 확장 가능** — 동일 기능 그대로 학원·부트캠프·기업 교육 라이선스로 전환 가능.
  7. **단계적 확장 로드맵** — SQLD → SQLP → PL/SQL → ADsP/빅분기 순서를 **기술 재활용** 관점으로 설명.
  8. **Why Now** — "시장 크다 + 기존 방식 정적 + AI로 묶을 수 있음" 3줄로 타이밍 정당화.

## 주요 구현 내용

### 삼단 구조 재편

공모전 심사위원은 크게 두 그룹 — 기술 심사역과 사업 심사역 — 으로 나뉜다. 양쪽 모두를 설득하기 위해 README를 **상단(제품 데모) → 중단(사업성) → 하단(기술 스택)** 삼단으로 재편했다. 각 섹션이 독립적으로 읽혀도 핵심이 전달되도록 구성.

### 사업성 논증의 4단계 프레임

심사 기준에 맞춰 사업성 섹션을 **① 진짜 결제하는 시장인가 → ② 반복 결제 이유가 있는가 → ③ 가격 우위가 있는가 → ④ 확장 가능한가** 순서로 논리 흐름을 짰다. 각 질문에 대해 공식 통계 또는 구조적 설명으로 답하는 형태.

### 수치 기반 설득

모든 시장·가격 수치는 **공식 출처(PQI)와 시중 판매가**에서 교차 검증한 값으로 교체. 과장 표현("연 수억")을 제거하고 **"1% 전환 시 월 450만 원"** 처럼 계산 가능한 수치로 대체해 심사위원의 반박 여지를 줄임.

### 포지셔닝 전환

README 전체에서 가장 결정적인 변경은 **"문제은행이 아니라 실행형 훈련기"** 한 줄. 경쟁 카테고리 자체를 "책 vs AI 책"에서 "정적 학습 vs 실행형 학습"으로 바꿔, 심사위원의 머릿속 비교 대상 집합을 재구성한다.

### 내부 구현 노출 최소화

MariaDB 등 테스트 환경 세부사항은 외부 공개 문서에서 제거. Oracle DB로 통일해 **실제 시험 환경과 동일한 엔진**이라는 학습 가치로 재프레이밍. AI 모델명(Gemini)도 "AI Agent"로 추상화해 기술적 독립성·확장성 인상을 강화.
