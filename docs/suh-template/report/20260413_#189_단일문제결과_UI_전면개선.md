# 단일 문제 결과 화면 UI 전면 개선 #189

## 개요

홈화면의 AI 추천 문제·오늘의 문제에서 진입하는 단일 문제 결과 화면(`AnswerFeedback.tsx`)을 전면 재설계했다. EXECUTABLE 모드에서의 정보 과부하, 문제 지문 확인 수단 부재, SQL 비교 섹션의 접근성 문제를 해소하고, 실제 SQL을 실행하는 학습 도구임을 UI에서 명확히 드러내도록 개선했다.

## 주요 구현 내용

**문제 보기 토글** — `stem`이 navigate state로 전달된 경우에만 카드를 렌더링한다. 접힌 상태에서는 지문 첫 줄 미리보기를 표시하고, 펼치면 ReactMarkdown으로 마크다운 테이블·코드 블록을 렌더링한다. `schemaDisplay`가 null인 경우 `schemaDdl`로 폴백하여 SchemaViewer를 표시한다.

**SQL 아코디언** — `choices` 배열을 `정답(0) → 내 선택(1) → 나머지(2)` 순으로 정렬하여 렌더링한다. 각 카드는 `getChoiceCardClass()`로 상태별 CSS·뱃지를 결정한다. `executingKeyRef`와 `resultCacheRef`를 guard로 활용해 병렬 실행 중 중복 호출을 방지한다. 제출 응답에서 받은 `selectedResult`/`correctResult`는 초기 캐시로 즉시 주입된다.

**ReactMarkdown 호환성** — `className` prop이 TS 오류를 유발해 `components` prop으로 커스텀 렌더러를 직접 지정했다. 단일 `\n`도 줄바꿈으로 표시되도록 `p` 컴포넌트에 `whitespace-pre-line`을 적용했다.

## 주의사항

- `isDailyChallenge` 플래그가 있는 오늘의 문제는 오답 시 "다시 풀기"로 같은 문제를 반복한다 (추천 문제와 동작 방식 상이 — 백엔드 의도적 설계).
- `/api/ai/similar` 엔드포인트가 500을 반환할 경우 유사 문제 섹션은 graceful하게 숨겨진다.
- `schemaDisplay`가 API에서 null로 내려오는 문제가 있으므로, 항상 `schemaDdl` 폴백 경로를 유지해야 한다.
