# [기능추가] 사용자 문제 풀기 — 선택지 생성(SSE) 및 제출 플로우 완성 (#69) FE 연동 가이드

## 개요

BE에서 문제 풀기 핵심 플로우 두 가지를 완성했다.

1. **SSE 선택지 생성**: `POST /api/questions/{uuid}/generate-choices` — AI가 실시간으로 4지선다 선택지를 생성하며 SSE 스트림으로 내려줌
2. **제출 플로우 마이그레이션**: `POST /api/questions/{uuid}/submit` — `choiceSetId` 기반으로 정답 판별 + EXECUTABLE 문제는 선택 SQL과 정답 SQL 실행 결과 비교까지 반환

---

## FE ↔ BE 현재 불일치 상황 (연동 전 필독)

> FE contract test (`api-contract.test.ts`)를 확인한 결과, FE는 현재 **SSE 없이 기존 구조(ADMIN_SEED choiceSets)** 를 사용하는 방향으로 리팩토링된 상태다.

| 항목 | BE #69 구현 | FE 현재 상태 |
|------|------------|-------------|
| 선택지 취득 방식 | SSE 엔드포인트로 AI 생성 후 `choiceSetId` 획득 | `GET /questions/{uuid}` 의 `choiceSets[]` 에서 직접 사용 |
| submit body | `{ choiceSetId: UUID, selectedChoiceKey: string }` | `{ selectedChoiceKey: string }` (choiceSetId 없음) |
| `generateChoices` 함수 | 사용 예정 | `questions.ts`에서 제거됨 |

**FE에서 이슈 #69 BE와 연동하려면 아래 변경이 필요하다.**

## E2E 전체 플로우

```
[1] 화면 진입
    └─ GET /api/questions/{questionUuid}
       → QuestionDetail.choiceSets[] 에 ADMIN_SEED 선택지 포함
          (executionMode: EXECUTABLE | CONCEPT_ONLY)

[2] 선택지 생성 (SSE)
    └─ POST /api/questions/{questionUuid}/generate-choices
       Headers: X-Member-UUID: {memberUuid}, Accept: text/event-stream
       
       이벤트 스트림 순서:
       event: status  → { phase: "generating",  message: "선택지 생성 중..." }
       event: status  → { phase: "validating",  message: "SQL 실행 검증 중..." }
       event: complete → { choiceSetId: "uuid", choices: [...] }  ← 이 UUID를 보관
       
       ※ 에러 시:
       event: error   → { code: "GENERATION_FAILED", message: "...", retryable: true }

[3] 선택지 표시 + SQL 자동 실행
    └─ choices[]: [{ key: "A", kind: "SQL", body: "SELECT ...", sortOrder: 0 }, ...]
       선택지 클릭 시 → POST /api/questions/{questionUuid}/execute
       Body: { sql: "SELECT ..." }
       → ExecuteResult: { status, columns, rows, rowCount, elapsedMs, errorCode, errorMessage }

[4] 제출
    └─ POST /api/questions/{questionUuid}/submit
       Headers: X-Member-UUID: {memberUuid}
       Body: { choiceSetId: "uuid-from-step2", selectedChoiceKey: "A" }
       
       Response (EXECUTABLE):
       {
         isCorrect: true,
         correctKey: "B",
         rationale: "...",
         selectedResult: { status, columns, rows, rowCount, elapsedMs, ... },
         correctResult:  { status, columns, rows, rowCount, elapsedMs, ... },
         correctSql: "SELECT ...",
         selectedSql: "SELECT ..."
       }
       
       Response (CONCEPT_ONLY):
       {
         isCorrect: false,
         correctKey: "C",
         rationale: "...",
         selectedResult: null,
         correctResult: null,
         correctSql: null,
         selectedSql: null
       }
```

### FE (필요한 변경)

#### 1. `src/api/questions.ts` — `generateChoices` SSE 함수 추가

```typescript
interface ChoiceItem {
  key: string;
  kind: string;
  body: string;
  sortOrder: number;
}

interface ChoiceSetGenerateResponse {
  choiceSetId: string;
  choices: ChoiceItem[];
}

type SseStatusPhase = "generating" | "validating";

interface SseStatusEvent {
  phase: SseStatusPhase;
  message: string;
}

interface SseErrorEvent {
  code: string;
  message: string;
  retryable: boolean;
}

/**
 * AI 선택지 세트를 SSE 스트림으로 생성한다.
 * 콜백으로 각 이벤트를 처리하고, complete 이벤트에서 choiceSetId를 받아 저장해야 한다.
 */
export function generateChoices(
  questionUuid: string,
  callbacks: {
    onStatus: (event: SseStatusEvent) => void;
    onComplete: (response: ChoiceSetGenerateResponse) => void;
    onError: (event: SseErrorEvent) => void;
  }
): () => void {
  const url = `/api/questions/${questionUuid}/generate-choices`;
  const eventSource = new EventSource(url, { withCredentials: false });

  // EventSource는 커스텀 헤더를 지원하지 않으므로
  // X-Member-UUID가 필요하면 fetch + ReadableStream 방식으로 구현 필요
  // (현재 BE는 헤더 기반 인증이므로 fetch 방식 권장)

  eventSource.addEventListener("status", (e) => {
    callbacks.onStatus(JSON.parse(e.data));
  });

  eventSource.addEventListener("complete", (e) => {
    callbacks.onComplete(JSON.parse(e.data));
    eventSource.close();
  });

  eventSource.addEventListener("error", (e) => {
    if ((e as MessageEvent).data) {
      callbacks.onError(JSON.parse((e as MessageEvent).data));
    }
    eventSource.close();
  });

  // cleanup 함수 반환 (컴포넌트 unmount 시 호출)
  return () => eventSource.close();
}
```

> **중요**: `EventSource`는 커스텀 헤더(`X-Member-UUID`)를 지원하지 않는다. BE가 `@RequestHeader(value = "X-Member-UUID")` 를 필수로 받으므로, `fetch` + `ReadableStream` 방식으로 구현이 필요하다. 아래 패턴 참고:

```typescript
export async function generateChoices(
  questionUuid: string,
  memberUuid: string,
  callbacks: {
    onStatus: (event: SseStatusEvent) => void;
    onComplete: (response: ChoiceSetGenerateResponse) => void;
    onError: (event: SseErrorEvent) => void;
  }
): Promise<void> {
  const response = await fetch(
    `/api/questions/${questionUuid}/generate-choices`,
    {
      method: "POST",
      headers: {
        "X-Member-UUID": memberUuid,
        "Accept": "text/event-stream",
      },
    }
  );

  if (!response.body) return;

  const reader = response.body.getReader();
  const decoder = new TextDecoder();
  let buffer = "";

  while (true) {
    const { done, value } = await reader.read();
    if (done) break;

    buffer += decoder.decode(value, { stream: true });
    const lines = buffer.split("\n\n");
    buffer = lines.pop() ?? "";

    for (const chunk of lines) {
      const eventNameMatch = chunk.match(/^event: (\w+)/m);
      const dataMatch = chunk.match(/^data: (.+)/m);
      if (!eventNameMatch || !dataMatch) continue;

      const eventName = eventNameMatch[1];
      const data = JSON.parse(dataMatch[1]);

      if (eventName === "status") callbacks.onStatus(data);
      else if (eventName === "complete") callbacks.onComplete(data);
      else if (eventName === "error") callbacks.onError(data);
    }
  }
}
```

## 핵심 구현 내용 (BE)

### SSE 엔드포인트 내부 동작

`POST /api/questions/{uuid}/generate-choices` 를 호출하면:

1. Virtual Thread에서 비동기 실행
2. `ChoiceSetResolver.resolveForUser()` 호출
   - **캐시 히트**: `source=AI_PREFETCH`인 미소비(`consumed_at IS NULL`) 세트가 있으면 즉시 반환
   - **캐시 미스**: AI 호출 → EXECUTABLE이면 샌드박스 SQL 실행으로 정답 자동 판별 → 최대 3회 재시도
3. DB에 `question_choice_set` + `question_choice_set_item` 저장 (메타데이터 포함)
4. `isCorrect`, `rationale`은 응답에서 제외 (사용자 정답 노출 방지)

### 제출 시 정답 판별 로직

1. `choiceSetId` → `QuestionChoiceSet` 조회
2. `selectedChoiceKey` → `QuestionChoiceSetItem.isCorrect` 로 정답 판별
3. `choiceSet.consumed_at` 최초 제출 시 기록 (재사용 방지)
4. EXECUTABLE 문제: `SandboxExecutor`로 선택 SQL + 정답 SQL 각각 실행 → `SubmitResult`에 포함

## 주의사항

- **구 `QuestionChoice` 테이블**: `GET /questions/{uuid}` 응답의 `choiceSets[]`는 여전히 구 테이블 기반. ADMIN_SEED 선택지는 이 경로로 조회된다. FE가 SSE 없이 이 선택지를 사용할 경우 `choiceSetUuid`가 필요하므로, `choiceSets[0].choiceSetUuid`를 `choiceSetId`로 전달해야 한다.
- **CONCEPT_ONLY 문제**: `selectedResult`, `correctResult`, `correctSql`, `selectedSql`이 모두 `null`. 결과 화면에서 null 체크 필수.
- **SSE 타임아웃**: BE 60초. AI 응답 지연 시 사용자에게 로딩 피드백 필수.
- **FE contract test 업데이트 필요**: `api-contract.test.ts`의 `submitAnswer` 관련 테스트가 현재 `choiceSetId` 없는 스펙 기준이므로, 변경 후 테스트도 함께 수정 필요.
