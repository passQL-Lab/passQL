# 데일리챌린지 제출 중 이탈 확인 모달 오탐 수정

**이슈**: [#146](https://github.com/passQL-Lab/passQL/issues/146)

---

### 📌 작업 개요

데일리 챌린지에서 "제출하기" 버튼 클릭 시 이탈 차단 모달("풀이를 그만할까요?")이 오탐으로 표시되는 버그 수정.
제출 API 호출 중에도 `useBlocker`가 활성 상태를 유지하여, React Router가 내부 네비게이션을 이탈로 오인하여 발생.

---

### 🔍 문제 분석

**원인**: `useBlocker` 활성 조건에 제출 진행 상태(`submitting`)가 고려되지 않음

기존 조건:
```typescript
const blocker = useBlocker(!isLoading && feedback === null);
```

- 선택지 선택 후 제출 버튼 클릭 → `submitting = true` 상태로 API 호출
- 이 시점에 `feedback === null`이 여전히 `true`이므로 blocker 활성화
- React Router 내부 처리 또는 오답 catch 블록의 `navigate("/")` 호출이 이탈로 인식
- "풀이를 그만할까요?" 모달 오탐 발생

---

### ✅ 구현 내용

#### useBlocker 활성 조건에 `submitting` 상태 추가
- **파일**: `src/pages/DailyChallenge.tsx`
- **변경 내용**: blocker 조건에 `&& !submitting` 추가

```typescript
// 수정 후
const blocker = useBlocker(!isLoading && feedback === null && !submitting);
```

- **이유**: 제출 API 호출 중에는 이탈 차단을 해제하여 내부 네비게이션이 모달 없이 통과되도록 처리
  - 오답 catch 블록의 `navigate("/", { replace: true })`도 blocker에 의해 차단되지 않아야 함
  - 제출 중에는 `LoadingOverlay`로 화면 조작을 별도 차단하므로 이탈 방지 중복 불필요

---

### 🔧 주요 변경사항 상세

#### DailyChallenge.tsx — blocker 조건 및 submitting 상태 관리

제출 흐름과 이탈 차단 상태의 관계:

| 상태 | isLoading | feedback | submitting | blocker 활성 |
|------|-----------|----------|------------|-------------|
| 로딩 중 | true | null | false | ❌ 비활성 |
| 풀이 중 | false | null | false | ✅ 활성 |
| **제출 API 호출 중** | false | null | **true** | **❌ 비활성** |
| 제출 완료(정답) | false | result | false | ❌ 비활성 |
| 제출 완료(오답) | false | result | false | ❌ 비활성 |

`submitting` 상태는 `setSubmitting(true)` → API 호출 → `finally { setSubmitting(false) }` 구조로 관리.

**특이사항**:
- 제출 중 화면 조작 차단은 `LoadingOverlay`가 담당, `useBlocker`는 실제 이탈(홈 이동, 뒤로가기)만 막는 역할로 분리
- `useBlocker`는 훅 규칙상 조건부 `return` 이전에 호출 위치 유지

---

### 🧪 테스트 및 검증

- 데일리 챌린지 진입 → 선택지 선택 → 제출 클릭: 모달 없이 채점 진행 확인
- 채점 중 홈 버튼 클릭: `LoadingOverlay`로 조작 차단 (blocker 아닌 오버레이가 방어)
- 오답 제출 후 `catch`에서 `navigate("/")` 호출: 모달 없이 홈 이동 확인
- 풀이 중(선택지 미제출 상태) 홈 버튼 클릭: "풀이를 그만할까요?" 모달 정상 표시 확인
