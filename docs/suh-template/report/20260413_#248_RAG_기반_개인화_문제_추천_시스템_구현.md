# RAG 기반 개인화 문제 추천 시스템 구현

## 개요

기존 `GET /api/questions/recommendations`는 `ORDER BY RANDOM()`으로 완전 랜덤 추천만 제공했다. 이번 구현에서는 bge-m3 임베딩 + Qdrant 벡터 검색을 활용한 RAG 파이프라인을 연결하여, 회원의 최근 오답 패턴을 기반으로 유사한 문제를 우선 추천하는 개인화 시스템을 도입했다. 오답 기록 없음·AI 서버 장애 등 예외 상황에서는 기존 RANDOM 추천으로 자동 폴백한다.

## 변경 사항

### Python AI 서버

- `ai/src/services/qdrant_client.py`: `create_collection_if_not_exists(collection, vector_size)` 메서드 추가 — 컬렉션 자동 생성 (Cosine 거리). `get_vector(collection, point_id)` 추가 — 단일 포인트 벡터 조회. `search()`에 `must_not_ids` 파라미터 추가 — 이미 푼 문제 payload 필터 제외
- `ai/src/models/ai_request.py`: `IndexQuestionRequest`, `IndexQuestionsBulkRequest`, `RecommendRequest` 모델 추가
- `ai/src/models/ai_response.py`: `IndexQuestionResponse`, `IndexQuestionsBulkResponse`, `RecommendedQuestion`, `RecommendResponse` 모델 추가
- `ai/src/services/ai_service.py`: `_build_embed_text()` — stem + 토픽 + 난이도 + 태그 구조화 텍스트 생성. `index_question()` — 단건 임베딩 적재. `index_questions_bulk()` — 다건 순차 적재 (개별 실패 스킵). `recommend()` — 오답 벡터 평균 쿼리 → Qdrant 검색 구현
- `ai/src/apis/ai_router.py`: `POST /api/ai/index-question`, `POST /api/ai/index-questions-bulk`, `POST /api/ai/recommend` 엔드포인트 추가

### Java - PQL-Domain-AI (신규 DTO)

- `IndexQuestionRequest.java`: 문제 임베딩 적재 요청 (questionUuid, topicDisplayName, subtopicDisplayName, difficulty, tagKeys, stem)
- `IndexQuestionResult.java`: 적재 결과 (questionUuid, created)
- `IndexQuestionsBulkRequest.java`: 다건 적재 요청 래퍼
- `IndexQuestionsBulkResult.java`: 다건 적재 결과 집계 (total, succeeded, failed, failedUuids)
- `RecommendRequest.java`: 추천 요청 (size, recentWrongQuestionUuids, solvedQuestionUuids)
- `RecommendResult.java`: 추천 결과 (items, querySourceCount) + 내부 레코드 `RecommendedQuestion(questionUuid, score)`

### Java - PQL-Domain-AI (기존 파일)

- `AiGatewayClient.java`: `indexQuestion()`, `indexQuestionsBulk()`, `recommend()` 메서드 추가 — 인덱싱 실패는 null 반환(non-critical), 추천 실패도 null 반환(Java에서 RANDOM fallback 처리)

### Java - PQL-Domain-Question

- `QuestionConceptTagRepository.java`: `findTagKeysByQuestionUuid(String questionUuid)` 추가 — concept_tag JOIN 단일 쿼리로 태그 키 목록 조회 (N+1 방지), `CAST(:questionUuid AS uuid)` 패턴 적용
- `QuestionGenerateService.java`: `indexQuestionAsync(Question)` 추가 — `Thread.startVirtualThread()`로 비동기 실행. `createQuestionOnly()`, `createQuestionWithSeedSet()` 저장 직후 호출하여 문제 등록 시 Qdrant 자동 색인
- `QuestionService.java`: `getRecommendationsByUuids(List<String>)` 추가 — 추천된 UUID 목록으로 QuestionSummary 변환

### Java - PQL-Domain-Submission

- `SubmissionRepository.java`: `findRecentWrongQuestionUuids(String memberUuid, int limit)` 추가 — 최근 오답 문제 UUID (DISTINCT, MAX(submitted_at) 기준 정렬, LIMIT). `findSolvedQuestionUuids(String memberUuid, int limit)` 추가 — 최근 푼 문제 UUID (LIMIT 200, Qdrant must_not 필터용)

### Java - PQL-Application (신규 서비스)

- `RecommendationService.java`: PQL-Domain-Question과 PQL-Domain-Submission 양쪽 의존성이 필요하여 Application 레이어에 신설. `recommend()` — size 클램핑(1-5), memberUuid 있을 시 AI 추천 시도 후 빈 결과면 RANDOM fallback. `tryAiRecommend()` — 오답 UUID 조회 → 풀이 UUID 조회 → AI 서버 호출 → UUID 목록으로 QuestionSummary 변환

### Java - PQL-Web

- `QuestionController.java`: `getRecommendations()`에 `@RequestParam(required = false) UUID memberUuid` 추가, `recommendationService.recommend()` 위임으로 교체
- `QuestionControllerDocs.java`: `getRecommendations` 시그니처에 `memberUuid` 파라미터 추가, `@ApiLog` 이력 항목 추가

## 주요 구현 내용

**임베딩 텍스트 구성**: 토픽 경로 + 난이도 + 개념 태그 + 문제 본문 앞 300자를 `[TOPIC]/[DIFFICULTY]/[TAGS]/[STEM]` 구조로 조합하여 검색 품질과 범주 유사성을 동시에 반영한다.

**추천 쿼리 생성**: 사용자의 최근 오답 문제(최대 10개)에 해당하는 Qdrant 벡터를 조회하고 단순 평균하여 쿼리 벡터를 합성한다. 이 벡터로 `passql_questions` 컬렉션에서 코사인 유사도 검색을 수행하며, 이미 풀어본 문제(최근 200개)는 must_not 필터로 제외한다.

**모듈 의존성 분리**: `PQL-Domain-Question`은 `PQL-Domain-Submission`에 의존할 수 없어 기존 `QuestionService`에서 처리 불가. `PQL-Application` 레이어에 `RecommendationService`를 신설하여 양쪽 도메인을 조합한다.

**비동기 색인**: 문제 등록 후 Qdrant 적재는 `VirtualThread`로 분리 실행하여 관리자 문제 등록 응답 속도에 영향을 주지 않는다. 색인 실패는 warn 로그만 남기고 등록 플로우에 영향을 주지 않는다.

**Graceful Degradation**: 오답 없음 → AI 서버 장애 → 추천 결과 없음 등 모든 예외 경로에서 기존 RANDOM 추천으로 자동 폴백하여 서비스 가용성을 보장한다.

## 주의사항

- 모든 네이티브 쿼리에서 UUID 바인딩은 `CAST(:param AS uuid)` 패턴 사용 (`::uuid` 금지 — Hibernate 파라미터 파싱 오류 선례 있음)
- `findSolvedQuestionUuids`는 전체 조회 시 페이로드 과부하 우려로 최근 200건으로 제한 (`SOLVED_QUESTION_LIMIT = 200`)
- 관리자 전체 재색인 엔드포인트(`POST /api/ai/index-questions-bulk`)는 Python 서버에 구현됐으나 Spring 관리자 컨트롤러 트리거는 미구현 — 별도 이슈로 추적 필요
