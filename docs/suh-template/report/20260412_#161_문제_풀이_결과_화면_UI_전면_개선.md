# 🎨[디자인][문제결과] 문제 풀이 결과 화면 UI 전면 개선

## 개요

Practice Result 화면(StepNavigator + PracticeResult)의 UI/UX 전면 개선 작업. 헤더 중복 요소 제거, 스텝별 콘텐츠 개선(통계 순차 애니메이션·AI 브랜딩·아코디언 복습), 선택지 본문 저장 방식 도입, Zustand 레이스 컨디션 수정까지 포함한다.

## 주요 구현 내용

**아코디언 높이 애니메이션**: JS 없이 `grid-rows-[0fr] ↔ grid-rows-[1fr]` CSS 전환만으로 부드러운 높이 애니메이션 구현. 내부 div에 `overflow-hidden` 적용.

**동적 transform 처리**: Tailwind 동적 값 제한을 우회하고자 CSS 커스텀 프로퍼티 `--step-offset`를 인라인 주입하고 `.step-slider` 클래스에서 `translateX(var(--step-offset))` 참조. 인라인 style 규칙과 충돌 없이 처리.

**순차 애니메이션**: `[0,1,2].forEach((i) => setTimeout(..., i * 150))` 패턴으로 통계 항목을 순차적으로 등장시킴. `timerIdsRef`로 ID를 보관하고 재호출 시 기존 타이머를 초기화하여 StrictMode 이중 호출 방어.

**선택지 본문 보존**: 제출 API 응답에는 선택지 본문이 포함되지 않으므로, 제출 시점에 `choices` 배열에서 `key` 기준으로 `body`를 추출해 스토어에 함께 저장. 결과 화면에서 재조회 없이 사용.

## 주의사항

- `stemPreview`가 결과 카드에서 노출하는 최대 지문 — 전체 stem은 별도 API 호출 없이는 접근 불가
- `selectedChoiceBody`는 스토어에만 존재하며 서버 전송 대상이 아님 (PracticeSubmitPayload 제외)
- 탭-투-어드밴스는 `StepNavigator`에만 적용되어 있어, 다른 페이지에서 재사용 시 동작 확인 필요
