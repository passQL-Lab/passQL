# #70 문제 상세 - 스키마 뷰어, SQL 실행기, 결과 피드백 고도화

### 📌 작업 개요

EXECUTABLE 문제 유형에 맞는 SQL 실행 환경을 문제 상세 화면에 통합하고, 제출 결과 화면을 문제 유형(EXECUTABLE / CONCEPT_ONLY)에 따라 분기하여 재설계. 실행 결과 비교 기반 학습 피드백 제공이 핵심 목표.

---

### 🎯 구현 목표

- 문제 상세에 스키마 시각화(SchemaViewer), 자유 SQL 실행기(SqlPlayground) 통합
- 공용 실행 결과 테이블(ResultTable) 추출하여 재사용
- 제출 결과 화면에서 SQL/텍스트 답안을 유형별로 비교 표시
- 오답 시 내 SQL 실행 결과 vs 정답 SQL 실행 결과 나란히 표시

---

### 🔧 주요 변경사항 상세

#### SchemaViewer 파싱 로직

`schemaDisplay` 포맷(`테이블명(컬럼 타입 PK, ...)`) 을 정규식으로 파싱해 구조화된 객체 배열로 변환. `schemaSampleData`는 INSERT INTO 구문을 정규식으로 파싱해 테이블명별 샘플 행을 Map으로 구성. 파싱 함수(`parseSchemaDisplay`, `parseSampleData`)를 컴포넌트 외부에 노출해 테스트 가능하도록 설계.

#### AnswerFeedback 분기 로직

```
executionMode === "EXECUTABLE"
  → SqlCompareBlock: SQL 코드 + ResultTable
executionMode === "CONCEPT_ONLY" (또는 undefined)
  → TextCompareBlock: 선택지 본문 텍스트만 표시
```

오답인 경우에만 내 선택지 블록을 위에 표시하고, 정답 블록은 항상 표시.

**특이사항**:
- 정답 시에는 AI 해설 버튼 미표시 (EXECUTABLE 문제도 동일). 오답 시에만 `AI에게 자세히 물어보기` 버튼 노출
- `selectedResult`, `correctResult`는 선택적 필드 — BE가 실행 결과를 내려주지 않는 경우에도 UI 깨짐 없이 처리
