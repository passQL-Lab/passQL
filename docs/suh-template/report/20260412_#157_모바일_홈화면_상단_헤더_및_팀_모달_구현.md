# ➕[기능추가][모바일] 모바일 홈화면 상단 헤더 및 팀 모달 구현

## 개요

모바일 홈화면에 상단 헤더가 없어 앱 정체성이 불분명했던 문제를 해결했다. 데스크톱은 좌측 사이드바에 로고가 있지만 모바일은 하단 탭만 존재해 상단이 비어 있었다. `MobileHeader` 컴포넌트와 팀 정보 모달(`TeamModal`)을 신규 추가하고, `AppLayout`에 모바일 분기로 삽입했다. 데스크톱 레이아웃은 변경 없음.

## 변경 사항

### 신규 컴포넌트

- `client/src/components/MobileHeader.tsx`: 모바일 전용 sticky 헤더. 좌측 로고, 우측 팀 아이콘 버튼(`Users` 아이콘). 클릭 시 팀 모달 오픈 — 상태는 부모(`AppLayout`)가 관리하고 콜백으로 전달.
- `client/src/components/TeamModal.tsx`: 팀 정보 모달. 팀명 "콜레라 (Cold Red Rice Team)", 멤버별 이름·GitHub 링크·역할 뱃지 표시. `createPortal`로 `document.body`에 직접 마운트하여 레이아웃 stacking context 문제 완전 해결.

### 기존 파일 수정

- `client/src/components/AppLayout.tsx`: 모바일 레이아웃 분기에 `MobileHeader` 삽입. 팀 모달 open 상태(`teamModalOpen`)를 `AppLayout`에서 관리하고 `TeamModal`을 조건부 렌더링.
- `client/CLAUDE.md`: 인라인 `style` 속성 절대 금지 규칙 추가 — Tailwind 유틸리티 클래스 또는 `components.css` 커스텀 클래스만 사용.
- `client/src/styles/components.css`: 중앙 고정 모달 전용 클래스 `.modal-overlay` / `.modal-card` 추가.

## 주요 구현 내용

**모달 stacking context 해결**: `fixed inset-0` 클래스가 Tailwind 기본 동작 기대와 달리 레이아웃 컨테이너 안에 갇히는 문제가 있었다. `createPortal(JSX, document.body)`로 모달을 DOM 트리 최상단에 직접 마운트하여 어떤 부모의 `overflow`, `transform`, stacking context에도 영향받지 않도록 처리했다.

**모달 CSS 클래스 분리**: Tailwind 동적 클래스 감지 이슈를 피하기 위해 `position: fixed`, `inset: 0`, `z-index: 9999` 등 고정 레이아웃 속성을 `components.css`의 `.modal-overlay` 클래스로 정의하고, 컴포넌트에서는 클래스명만 참조.

**단방향 상태 흐름**: 모달 open 상태를 `AppLayout`에서 관리하고 `MobileHeader`에 콜백(`onTeamClick`)으로 전달. 향후 다른 진입점에서도 같은 모달을 열 수 있는 구조.

## 주의사항

- 팀 정보(이름, GitHub URL)는 Mock 하드코딩 — API 연동 없음.
- `bg-blue-50 text-blue-600`, `bg-purple-50 text-purple-600` 등 역할 뱃지 색상이 프로젝트 디자인 토큰 미사용 — Mock 단계 허용, 추후 토큰화 권장.
- 이 모달 구조(`createPortal` + `.modal-overlay`)는 향후 다른 중앙 고정 모달에도 재사용 가능.
