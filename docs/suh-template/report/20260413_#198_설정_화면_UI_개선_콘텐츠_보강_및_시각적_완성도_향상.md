# 🚀 설정 화면 UI 개선 — 콘텐츠 보강 및 시각적 완성도 향상

**이슈**: [#198](https://github.com/passQL-Lab/passQL/issues/198)

---

### 📌 작업 개요

MVP 수준에 머물던 설정 화면을 섹션 기반 구조로 전면 재구성. 기존 단일 카드(닉네임/디바이스 ID/버전) 구성에서 **계정 / 건의사항 / 앱 정보 / 개발자** 4개 섹션으로 확장. 건의사항 제출·조회 기능을 선행 구현하고, 오프라인 감지·에러 처리·개발자 도구를 함께 추가.

---

### 🎯 구현 목표

- 섹션 헤더로 정보 위계 명확화
- 건의사항 제출·조회 기능 추가 (백엔드 미구현 → graceful fallback)
- 오프라인 상태 명시적 처리 (배너 + form disabled + ErrorFallback)
- 개발자 도구: localStorage 초기화 (모바일 접근성 개선)
- 전체 섹션 순차 페이드인 애니메이션 (useStagger 0~6)

### 🔧 주요 구현 상세

#### 건의사항 섹션

**FeedbackForm**: `focus-within`으로 카드 보더를 인디고로 전환. 카운터는 입력 시 숫자만 `text-brand` 강조. 제출 버튼은 0자·500자 초과 시 disabled. 제출 실패 시 입력 내용을 보존하고 인라인 에러(`bg-sem-error-light`, 왼쪽 굵은 보더 없음) + 재시도 버튼 표시.

**FeedbackList**: 로딩 시 animate-pulse 스켈레톤. 빈 상태는 카드를 숨기지 않고 Edit3 아이콘 + 안내 문구. 각 FeedbackItem은 부모에서 stagger 클래스를 주입받아 순차 페이드인.

**FeedbackStatus 표시**:

- `PENDING` → 노란 pill (대기)
- `REVIEWED` → 인디고 pill (확인됨)
- `APPLIED` → 초록 pill (반영됨)

#### 오프라인 처리

`useOnline()` 훅으로 실시간 감지. 오프라인 시:

- 건의사항 섹션 상단 노란 배너 표시
- FeedbackForm disabled (입력·제출 차단)
- FeedbackList ErrorFallback (재시도 버튼 없음)

#### 개발자 섹션

localStorage 초기화 버튼에 2단계 확인 적용. 1차 클릭 → "진짜요?" 상태 전환 (3초 타임아웃 자동 복귀), 2차 클릭 → `localStorage.clear()` + `window.location.reload()`. 모바일에서 개발자 도구 콘솔 접근 없이 앱 초기화 가능하도록 구현.

#### API 미구현 대응

`fetchMyFeedback`에서 `ApiError.status === 404` 감지 시 빈 배열로 fallback. 섹션 자체는 유지.
