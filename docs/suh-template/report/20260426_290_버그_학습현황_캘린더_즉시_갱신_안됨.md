# 학습현황 캘린더 즉시 갱신 안됨

## 개요

문제 풀이 후 홈 화면으로 복귀했을 때 학습현황 히트맵(캘린더)이 즉시 갱신되지 않는 버그를 수정했다. `invalidateQueries` 대신 `refetchQueries`를 사용해 네비게이션 타이밍과 무관하게 최신 데이터가 반영되도록 했다.

## 변경 사항

### 프론트엔드

- `client/src/pages/DailyChallenge.tsx`: 제출 후 heatmap 쿼리를 `invalidateQueries` → `refetchQueries`로 교체
- `client/src/pages/PracticeSet.tsx`: 동일 패턴 교체
- `client/src/pages/QuestionDetail.tsx`: 동일 패턴 교체
- `client/src/pages/RecommendationPractice.tsx`: 동일 패턴 교체

## 주요 구현 내용

`invalidateQueries`는 쿼리를 "stale" 상태로만 표시하고 실제 재요청은 해당 쿼리가 다음에 마운트될 때 발생한다. 홈 화면으로 navigate하는 시점과 쿼리 재조회 타이밍이 어긋나면 이전 캐시 데이터가 그대로 표시된다. `refetchQueries`는 무효화와 강제 재요청을 동시에 수행하므로 네비게이션 타이밍과 무관하게 항상 최신 데이터를 보장한다.

## 주의사항

- heatmap 외에 recommendations, progress 쿼리도 동일 패턴으로 통일되어 있어 일관성을 유지한다.
- 네트워크 요청이 즉시 발생하므로 느린 연결 환경에서는 홈 복귀 시 잠깐 로딩 상태가 보일 수 있다.
