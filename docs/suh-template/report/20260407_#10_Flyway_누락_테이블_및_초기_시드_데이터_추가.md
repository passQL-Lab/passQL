# [기능추가][Flyway] Flyway V0_0_15 누락 테이블 및 초기 시드 데이터 추가

## 개요

기존 마이그레이션(V0_0_11, V0_0_12)에 누락된 `question_concept_tag` 테이블을 추가하고, 앱 부팅에 필요한 운영 설정값(app_setting 13개), 프롬프트 템플릿(3종 v1), 토픽 마스터(9개)를 Flyway 마이그레이션으로 초기 적재하였다. 당시 version.yml 버전(0.0.21) 기준으로 `V0_0_21__add_concept_tag_table.sql` 단일 파일에 DDL과 DML을 통합하여 작성하였다.

## 변경 사항

### Flyway 마이그레이션

- `server/PQL-Web/src/main/resources/db/migration/V0_0_21__add_concept_tag_table.sql` : `question_concept_tag` 테이블 DDL + app_setting·prompt_template·topic 시드 데이터 DML 통합

### 설정 문서

- `server/CLAUDE.md` : DB 스키마 관리 규칙 보강 (version.yml 우선 확인, 버전당 파일 1개 원칙, 멱등성 보장 방식), 마이그레이션 이력 표 업데이트

### 공통

- `server/PQL-Common/src/main/java/com/passql/common/dto/Author.java` : 개발자 상수 클래스 추가

### API 계층 (병행 작업)

- `server/PQL-Web/src/main/java/com/passql/web/controller/` 하위 Controller/Docs 파일 : AI·Member·Meta·Progress·Question 컨트롤러 Swagger 문서 인터페이스 추가

## 주요 구현 내용

### 멱등성 처리 전략

테이블별 PK 특성에 따라 INSERT 전략을 분리하였다.

| 테이블 | PK | 전략 |
|---|---|---|
| `app_setting` | `setting_key` (문자열) | `ON DUPLICATE KEY UPDATE` |
| `topic` | `code` (문자열) | `ON DUPLICATE KEY UPDATE` |
| `prompt_template` | `id` (AUTO_INCREMENT) | `INSERT ... SELECT WHERE NOT EXISTS (key_name, version)` |

`prompt_template`는 PK가 자동 증가 정수라 `ON DUPLICATE KEY UPDATE`가 동작하지 않으므로 `key_name + version` 조합으로 존재 여부를 먼저 확인 후 삽입한다.

### 토픽 구성 (SQLD 시험 단원 기반 9개)

SQLD 시험 출제 범위를 기준으로 토픽을 정의하였다: 데이터 모델링의 이해 / SELECT 기본 / DDL·DML·TCL / SQL 함수 / 조인 / 서브쿼리 / 그룹함수·집계 / 윈도우 함수 / 계층 쿼리·PIVOT

### 프롬프트 템플릿 3종 v1

- `explain_error` : 오답 원인 해설 (temperature 0.5, maxTokens 512)
- `diff_explain` : SQL 쿼리 차이 비교 설명 (temperature 0.5, maxTokens 768)
- `similar` : 유사 문제 생성 (temperature 0.8, maxTokens 1024)

## 주의사항

- 이슈 명세의 파일 버전(V0_0_15)과 실제 적용 버전(V0_0_21)이 다름 — version.yml 기준 버전 확인 후 작성해야 하므로 작업 시점의 현재 버전(0.0.21)으로 통합
- Flyway 버전당 파일 1개 원칙에 따라 DDL과 DML을 단일 파일로 통합하였으며, 이 규칙을 `server/CLAUDE.md`에 명시하여 이후 작업에서 동일 실수를 방지
- `flyway.validate-on-migrate: false` 설정이 application.yml에 추가되어 있으므로, 운영 환경 배포 전 체크섬 검증 정책 재검토 필요
