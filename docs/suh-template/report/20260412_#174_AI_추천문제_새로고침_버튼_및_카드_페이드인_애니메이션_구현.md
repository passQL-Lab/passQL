# ⚙️[기능추가][홈] AI 추천문제 새로고침 버튼 및 카드 페이드인 애니메이션 구현

## 개요

홈화면 AI 추천문제 섹션에 새로고침 버튼을 추가하여, 사용자가 페이지 리로드 없이 다른 추천 문제 세트를 즉시 요청할 수 있도록 구현했다. 버튼 클릭 시 아이콘 1회전 애니메이션과 새 카드 순차 페이드인으로 명확한 시각적 피드백을 제공한다.

## 변경 사항

### 프론트엔드 로직
- `client/src/pages/Home.tsx`: `useRecommendations` 훅에서 `refetch`, `isFetching` 추출. `spinning`(버튼 애니메이션 트리거), `refreshKey`(카드 재마운트 트리거) state 추가. `handleRefresh` 콜백으로 refetch 후 카드 애니메이션 순차 실행

### 스타일
- `client/src/styles/components.css`: `animate-refresh-once` (360도 1회전, 0.4s), `animate-card-in` + nth-child 딜레이 클래스 (카드 순차 페이드인, 0~240ms) 추가

## 주요 구현 내용

**버튼 애니메이션 제어**
- `spinning` state로 `animate-refresh-once` 클래스를 on/off하는 방식 채택. `onAnimationEnd`로 애니메이션 종료 시 자동 초기화. `spinning || isFetching` 이중 가드로 중복 클릭 방지.

**카드 재진입 애니메이션**
- `refetch` 완료 후 `refreshKey`를 증가시켜 카드 컨테이너 div를 재마운트. React key 기반 재마운트로 `animate-card-in` 클래스가 재실행되며, `nth-child(1~5)` 딜레이 클래스로 카드가 순서대로 나타남.

**API 연동**
- `GET /questions/recommendations` 재호출. `staleTime` 내에도 강제 refetch되어 항상 새로운 문제 세트 반환.

## 주의사항

- `onAnimationEnd`가 `<button>`에 위치하나 실제 애니메이션은 내부 `<RefreshCw>` SVG에 적용됨 — 브라우저 이벤트 버블링 동작에 따라 `spinning` 고착 가능성 있음. 추후 `<RefreshCw>`로 이동 권장
- `refetchRecommendations().then()`은 실패 시 콜백이 실행되지 않아 `spinning`이 영구 `true`로 남을 수 있음 — `.finally()`로 교체 권장
- `hover:bg-gray-100`은 프로젝트 CSS 토큰 미사용 — 추후 `var(--color-surface)` 기반 클래스로 통일 권장
