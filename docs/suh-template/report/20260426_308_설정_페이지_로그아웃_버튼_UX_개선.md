# 🚀[기능개선][설정] 설정 페이지 로그아웃 버튼 UX 개선

## 개요
설정 페이지의 로그아웃 항목을 `SettingsRow` 공용 컴포넌트에서 분리해 전용 button으로 직접 렌더링하고, 텍스트·아이콘에 시맨틱 에러 색상(`sem-error`)을 적용해 파괴적 액션임을 시각적으로 명확히 표현했다. 함께 개발자 모드 Easter Egg 코드도 제거했다.

## 변경 사항

### 프론트엔드
- `client/src/pages/Settings.tsx`: 로그아웃 row 렌더링 방식 변경, Easter Egg 코드 제거

## 주요 구현 내용

**로그아웃 버튼 분리**
- 기존: `<SettingsRow>`에 로그아웃 액션을 끼워 넣는 방식 → 일반 설정 항목과 시각적 구분 불가
- 변경: `<button>` 직접 렌더링 + `text-sem-error` / `text-sem-error-text` 색상 적용
- 파괴적 액션(로그아웃)임을 색상으로 명시해 사용자가 실수로 누르는 UX 리스크 감소

**Easter Egg 코드 제거**
- 버전 row 5회 클릭 시 개발자 모드 잠금 해제 로직 전체 제거
- 관련 state(`devUnlocked`), ref(`clickCountRef`, `clickResetTimerRef`), 핸들러(`handleVersionClick`) 삭제
- cleanup effect에서 `clickResetTimerRef` clearTimeout 제거

## 주의사항
- 개발자 모드 접근 경로가 제거됐으므로, 추후 개발자 전용 기능이 필요하면 별도 접근 방식(URL 파라미터, 관리자 계정 등) 설계 필요
