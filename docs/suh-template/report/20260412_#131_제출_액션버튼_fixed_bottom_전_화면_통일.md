# 🚀 [기능개선][UI] 제출·액션 버튼 fixed bottom 전 화면 통일 #131

## 개요

`QuestionDetail`의 제출 버튼이 스크롤 영역 내부에 위치해, 제출 후 `PracticeFeedbackBar`(fixed bottom)와 동시에 화면에 존재하는 UI 불일치 문제를 해결했다. `QuestionDetail` 제출 버튼을 `fixed bottom-0 inset-x-0 z-20`으로 이동하고, `AnswerFeedback`의 기존 fixed 버튼 스타일도 동일하게 통일했다.

## 변경 사항

### QuestionDetail — 제출 버튼 fixed 전환
- `client/src/pages/QuestionDetail.tsx`: 제출 버튼 래퍼를 스크롤 영역 밖 `fixed bottom-0 inset-x-0 z-20`으로 이동. `pb-6 → pb-24`로 스크롤 하단 여백 확보(practiceMode/단독 모드 공통 적용)

### AnswerFeedback — 버튼 컨테이너 스타일 통일
- `client/src/pages/AnswerFeedback.tsx`: 컨테이너 배경색(정답/오답 색상) 제거 → `bg-surface-page border-t border-border`로 통일. 패딩을 내부 div로 이동. `h-[52px] → h-12`로 버튼 높이 통일

### PracticeSet — 제출 버튼 문구 수정
- `client/src/pages/PracticeSet.tsx`: 제출 버튼 문구 `"다음 문제"` → `"확인하기"` 변경 (마지막 문제는 `"결과 보기"` 유지)

## 주요 구현 내용

**z-index 계층 정립**

| 레이어 | z-index | 대상 |
|---|---|---|
| 피드백바 | z-30 | PracticeFeedbackBar (제출 후 덮음) |
| 액션 버튼 | z-20 | QuestionDetail 제출 버튼, AnswerFeedback 하단 버튼 |

제출 버튼(z-20)과 PracticeFeedbackBar(z-30) 간의 z-index 차이를 활용해, 제출 후 피드백바가 슬라이드업하며 제출 버튼을 자연스럽게 덮는 구조를 그대로 활용했다. `DailyChallenge`, `PracticeSet`은 `QuestionDetail` 수정만으로 자동 해결됐다.

**버튼 컨테이너 표준 구조**

```tsx
<div className="fixed bottom-0 inset-x-0 z-20 bg-surface-page border-t border-border">
  <div className="mx-auto max-w-120 px-4 py-4">
    {버튼}
  </div>
</div>
```

## 주의사항

- `AnswerFeedback`은 콘텐츠 영역이 더 넓어(`max-w-180`) `QuestionDetail`(`max-w-120`)과 max-width가 다르게 유지됨
- `PracticeFeedbackBar`는 변경하지 않음 — z-30 기준 레이어로 기존 구조 그대로 활용
