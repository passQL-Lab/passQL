# 개발자 모드 Easter Egg 잠금 해제 #208

### 📌 작업 개요

`Settings.tsx`의 개발자 섹션이 모든 사용자에게 노출되던 문제를 해결. 설정 화면의 버전 row를 2초 내 5번 연속 클릭하면 개발자 모드가 잠금 해제되고, `/dev` 전용 페이지에서 localStorage 초기화 기능을 제공하도록 개선. 잠금 해제 상태는 sessionStorage 기반으로 탭 세션 동안 유지.

---

### 🎯 구현 목표

- 개발자 전용 기능을 Easter Egg 방식으로 숨김 처리
- 잠금 해제 후 별도 `/dev` 페이지로 분리, route guard로 직접 접근 차단
- 2단계 확인 UX로 localStorage 초기화 실수 방지

---

### ✅ 구현 내용

#### SettingsRow — onClick prop 추가

- **파일**: `src/components/SettingsRow.tsx`
- **변경 내용**: `onClick?: () => void` prop 추가. 제공 시 row 전체에 `cursor-pointer hover:bg-surface active:bg-surface transition-colors` 적용
- **이유**: Easter egg 클릭 타겟을 row 전체로 확장하면서 기존 컴포넌트 호환성 유지

#### Settings.tsx — Easter Egg 클릭 카운터 및 잠금 해제

- **파일**: `src/pages/Settings.tsx`
- **변경 내용**:
  - `clickCountRef` + `clickResetTimerRef`로 2초 debounce 카운터 구현
  - 3회: `"개발자 모드까지 2번 남았습니다"` toast
  - 4회: `"개발자 모드까지 1번 남았습니다"` toast
  - 5회: `sessionStorage.setItem("devUnlocked", "1")` + `devUnlocked` state 전환 + "개발자 모드" row 노출
  - `devUnlocked` 초기값을 `sessionStorage` 기반으로 복원하여 페이지 재진입 시에도 row 유지
  - 기존 개발자 도구 섹션(localStorage 초기화 버튼) 완전 제거
- **이유**: React state 단독 사용 시 다른 페이지 갔다 돌아오면 row가 사라지는 버그 방지

#### DevPage.tsx — 신규 개발자 모드 전용 페이지

- **파일**: `src/pages/DevPage.tsx` (신규)
- **변경 내용**:
  - 뒤로 가기 헤더 + `SettingsSection / SettingsRow` 패턴으로 localStorage 초기화 기능 배치
  - 2단계 확인 UX: 1차 클릭 시 버튼이 "진짜요?"로 전환 + 인라인 경고 텍스트 노출, 2차 클릭 시 `localStorage.clear()` + `sessionStorage.clear()` 후 홈으로 리다이렉트
  - 3초 후 자동 상태 복원 (실수 방지)
  - 초기화 완료 시 `window.location.href = "/"` (풀 리로드 + 홈 이동)

#### App.tsx — `/dev` 라우트 및 route guard

- **파일**: `src/App.tsx`
- **변경 내용**:
  - `DevGuard` 컴포넌트 추가: `sessionStorage.getItem("devUnlocked")` 확인 후 없으면 `/settings`로 redirect
  - `/dev` 라우트를 AppLayout 밖에 별도 등록
- **이유**: URL 직접 입력으로 `/dev` 진입 차단

### 🔧 주요 설계 결정

**sessionStorage 선택 이유**  
탭 세션 동안 잠금 해제 유지 + `localStorage.clear()` 실행 시에도 영향받지 않음. 브라우저 탭을 닫으면 자동 해제.

**debounce 타이머 방식**  
매 클릭마다 2초 타이머를 재시작하는 debounce 패턴 사용. 마지막 클릭 기준으로 2초 내 추가 클릭 없으면 카운터 리셋. 실사용에서 잠깐 쉬어도 허용하는 Easter egg 특성에 적합.

**toast → 인라인 경고 텍스트 전환**  
localStorage 초기화 경고를 DevPage에서 toast 대신 카드 하단 인라인 텍스트(`text-xs text-sem-error-text`)로 표시. 긴 경고 메시지가 toast에서 레이아웃 깨짐을 유발했기 때문.

| 페이지 재진입 시 row 숨김 | `devUnlocked` 초기값 항상 `false`                             | `useState(() => sessionStorage.getItem("devUnlocked") === "1")` |
