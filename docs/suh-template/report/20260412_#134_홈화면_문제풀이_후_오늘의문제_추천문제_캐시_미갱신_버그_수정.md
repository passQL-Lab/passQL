# ❗[버그][홈화면] 문제 풀이 후 오늘의 문제·추천 문제 상태가 즉시 갱신되지 않는 버그

## 개요

오늘의 문제 또는 추천 문제를 풀고 홈화면에 복귀해도 완료 상태·추천 목록이 갱신되지 않던 버그를 수정했다. React Query `staleTime`(5분) 내 제출 완료 시 관련 쿼리 캐시가 무효화되지 않아 발생한 문제로, 각 제출 완료 시점에 `invalidateQueries` 호출을 추가하여 해결했다.

## 변경 사항

### 프론트엔드

- `client/src/pages/DailyChallenge.tsx`: 정답 제출 성공 후 `recommendations` 쿼리 캐시 무효화 추가, `useCallback` 의존성 배열에 `queryClient`·`uuid` 누락 수정
- `client/src/pages/QuestionDetail.tsx`: `useQueryClient` import 추가, 제출 성공 시 `recommendations` 쿼리 캐시 무효화 추가, `handleSubmit` `useCallback` 의존성에 `queryClient` 추가

## 주요 구현 내용

**원인**: `useTodayQuestion`, `useRecommendations` 훅의 `staleTime`이 5분으로 설정되어 있어, 제출 완료 후 홈 복귀 시에도 캐시된 데이터를 그대로 반환했다. `DailyChallenge`에는 `todayQuestion` 캐시 무효화만 존재했고 `recommendations` 무효화가 없었으며, `QuestionDetail` 단독 풀이 모드에는 어떤 캐시 무효화도 없었다.

**해결**: 제출 성공 콜백(`onSuccess` / `try` 블록)에서 `queryClient.invalidateQueries`를 호출해 서버에 재조회를 트리거한다. `invalidateQueries`는 비동기지만 `await` 없이 호출해 네비게이션을 즉시 진행하는 기존 패턴을 유지했다.

```
제출 완료
  ├─ DailyChallenge: invalidate todayQuestion + recommendations → setFeedback
  └─ QuestionDetail(단독): invalidate recommendations → navigate
```

## 주의사항

- `useProgress`(`solvedCount`, `streakDays` 등)는 이미 `staleTime: 0`으로 설정되어 있어 별도 무효화 불필요
- 오답 제출 시(`DailyChallenge` 로컬 피드백 경로)는 서버에 제출하지 않으므로 캐시 무효화 대상 아님
