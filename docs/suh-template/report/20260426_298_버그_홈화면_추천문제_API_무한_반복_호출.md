# ❗[버그][홈] 홈 화면 추천 문제 API 무한 반복 호출 버그

## 개요
홈 화면 진입 시 추천 문제 API가 무한 반복 호출되던 버그를 3단계에 걸쳐 근본적으로 해결했다. `queryKey`에 동적 배열이 포함되어 React Query 캐시 키가 매 렌더마다 달라지는 것이 근본 원인이었으며, `setQueryData` 패턴 전환 + 미인증 상태 호출 방지 + 에러 상태 복구로 완전히 수정됐다.

## 변경 사항

### 프론트엔드
- `client/src/hooks/useHome.ts`: queryKey 고정, `enabled` 조건 추가, `useRefreshRecommendations` 패턴 개선
- `client/src/pages/Home.tsx`: 에러 시 `isRefreshing` 복구, `useCallback` 의존성 안정화, `isFetching` 기반 버튼 disabled 동기화

## 주요 구현 내용

**1차 수정 — queryKey 직렬화 (`997cf7b`)**
- `excludeUuids` 배열을 `queryKey`에 직접 포함하면 참조가 매번 달라져 캐시 미스 → 무한 재요청 발생
- `JSON.stringify(excludeUuids)`로 직렬화하고 `useEffect` 의존성 최소화 시도

**2차 수정 — setQueryData 패턴 전환 (`e92aa66`)**
- `queryKey`를 고정(`["recommendations", accessToken]`)하고 `excludeUuids`는 queryKey에서 완전 제거
- 새로고침 시에만 `fetchRecommendations(excludeUuids)` 호출 후 `setQueryData`로 캐시 직접 갱신
- `useRefreshRecommendations` 훅 분리로 관심사 명확히 구분

**3차 수정 — 미인증 상태 방지 + 에러 복구 (`49ad7c6`, `b48daf2`)**
- `useRecommendations`에 `enabled: !!accessToken` 조건 추가 — 로그인 전 호출 원천 차단
- 새로고침 에러 시 `finally`로 `isRefreshing` 상태 복구 — 버튼 영구 비활성화 방지
- `useCallback` 의존성 배열 안정화

## 주의사항
- `seenUuids`는 세션 내 최대 30개(FIFO)로 제한 — 무제한 누적 시 POST body 비대화 방지
- `queryKey` 변경으로 기존 캐시가 무효화되므로, 배포 직후 첫 진입 시 추천 문제가 새로 로딩됨
