# ⚙️[기능개선][FE] 홈 및 네비게이션 탭 전환 시 섹션별 스태거 페이드인 애니메이션 구현

**라벨**: `작업전`
**담당자**: 

---

📝현재 문제점
---

- 홈화면 진입 시 `animate-greeting` / `animate-greeting-delayed` CSS 클래스만 적용되어 있어 그리팅 섹션 하나만 페이드인됨
- 오늘의 문제 카드, 학습현황, 합격 준비도, AI 추천문제 등 나머지 섹션은 애니메이션 없이 즉시 노출됨
- AI문제(CategoryCards), 통계(Stats), 설정(Settings) 탭은 애니메이션 전혀 없음
- 탭 전환 시 화면 전환감이 없어 모던한 느낌이 부족함

🛠️해결 방안 / 제안 기능
---

- `useStagger` 커스텀 훅 신규 구현 (`client/src/hooks/useStagger.ts`)
  - `useStagger(count)` → `(index: number) => string` 형태로 Tailwind 클래스 반환
  - 마운트 시 `opacity-0 translate-y-2` → `opacity-100 translate-y-0` 전환
  - 각 섹션 인덱스마다 `50ms × index` delay, duration 300ms, ease-out
  - CSS variable(`--stagger-delay`)로 delay 주입 — inline style 없이 처리
- `AppLayout`의 `<Outlet />`에 `key={location.pathname}` 추가하여 탭 전환마다 리마운트 → 스태거 자동 재실행
- 기존 `animate-greeting` / `animate-greeting-delayed` 클래스 제거 후 `useStagger`로 대체
- 페이지별 섹션 분해 및 적용:
  - **Home**: ①그리팅 줄1 ②그리팅 줄2+서브메시지 ③오늘의문제+일정 카드행 ④학습현황 ⑤합격준비도/통계 ⑥AI추천문제
  - **CategoryCards**: ①제목+서브텍스트 ②카드 그리드
  - **Stats**: ①제목+서브텍스트 ②상단 요약 카드 ③AI코멘트 ④레이더차트 ⑤바차트
  - **Settings**: ①제목 ②설정 카드 ③하단 로고+카피라이트
- `@keyframes staggerIn` CSS 정의 (`components.css`)
