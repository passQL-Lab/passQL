📝현재 문제점
---

- 현재 `GET /api/progress` API는 **풀 문제 수, 정답률, 연속 학습일, 합격 준비도**만 제공한다.
- 통계 페이지에 필요한 **토픽별 정답률(레이더 차트)**, **토픽별 문제 수(가로 막대 그래프)**, **AI 영역 분석 코멘트**를 지원하는 API가 없다.
- 프론트엔드에서 "내 실력, 한눈에" 화면을 구현하려면 3가지 추가 데이터가 필요하다:
  1. **토픽별 정답률** — 레이더 차트 (JOIN, 정규화, 서브쿼리, GROUP BY, DDL, 제약조건, WHERE, ORDER BY, DML, 윈도우 함수, 인덱스, 트랜잭션 등)
  2. **토픽별 총 문제 수** — 가로 막대 그래프
  3. **AI 영역 분석 코멘트** — "전반적으로 안정적인 실력을 보여주고 있어요. 실력을 더 높이고 싶다면 윈도우 함수, ORDER BY, GROUP BY 카테고리를 집중적으로 학습해보세요." 같은 자연어 분석

🛠️해결 방안 / 제안 기능
---

### 1. 신규 API 엔드포인트

| 엔드포인트 | 메서드 | 설명 | 응답 DTO |
|------------|--------|------|----------|
| `/api/progress/topic-analysis?memberUuid={uuid}` | GET | 토픽별 정답률 + 토픽별 문제 수 | `TopicAnalysisResponse` |
| `/api/progress/ai-comment?memberUuid={uuid}` | GET | AI 영역 분석 코멘트 (비동기 로딩) | `AiCommentResponse` |

- 기존 `/api/progress` (상단 숫자 3개)는 변경 없음
- LLM 호출 latency를 고려하여 AI 코멘트는 별도 API로 분리 (프론트에서 비동기 로딩)

### 2. 응답 DTO 설계

**TopicAnalysisResponse:**

```
TopicAnalysisResponse
├── topicStats: List<TopicStat>
│   ├── topicUuid: UUID
│   ├── displayName: String         // Topic.displayName (동적)
│   ├── totalQuestionCount: int     // 해당 토픽 총 문제 수 (막대그래프용)
│   ├── correctRate: double         // 최근 7일, 문제별 최근 시도 기준 정답률 (레이더차트용)
│   └── solvedCount: int            // 최근 7일 내 푼 문제 수
```

**AiCommentResponse:**

```
AiCommentResponse
├── comment: String                 // AI 생성 코멘트
└── generatedAt: LocalDateTime      // 생성 시각
```

### 3. 토픽별 정답률 계산 기준

- **문제별 최근 시도 기준** (기존 ReadinessCalculator와 동일 방식)
- **최근 7일 이내** Submission만 대상
- 계산: `Submission JOIN Question ON questionUuid → Question.topicUuid로 GROUP BY`
- 각 토픽별로 `ROW_NUMBER() OVER (PARTITION BY question_uuid ORDER BY submitted_at DESC)` 후 `rn=1`인 것만 집계
- 토픽 목록은 **Topic 테이블 기준 동적** (isActive=true인 토픽 전체)

### 4. 토픽별 문제 수 계산

- `Question` 테이블에서 `isActive=true`인 문제를 `topicUuid`로 GROUP BY COUNT
- 전체 기간 기준 (7일 제한 없음)

### 5. AI 영역 분석 코멘트 생성

사용자의 **전체 학습 현황을 종합 분석**하는 코멘트이다. 토픽별로 개별 코멘트를 생성하는 것이 아니라, 모든 토픽의 정답률/풀이수/연속학습일/합격준비도 등 전체 통계를 종합하여 **하나의 분석 문장**을 생성한다.

- **모델**: 기존 `GeminiClient` 재사용 (gemini-2.5-flash-lite)
- **입력**: 전체 학습 통계 (토픽별 정답률, 풀 문제 수, 연속 학습일, 합격 준비도 등)를 종합한 JSON
- **출력**: 한국어 종합 분석 코멘트 (2~3문장)
- **코멘트 내용**: 전반적인 실력 평가 + 약한 영역 파악 + 집중 학습 추천
- **예시**: "전반적으로 안정적인 실력을 보여주고 있어요. 실력을 더 높이고 싶다면 윈도우 함수, ORDER BY, GROUP BY 카테고리를 집중적으로 학습해보세요."

### 6. AI 코멘트 캐싱 전략

- **변화 기반 캐싱**: 새로운 Submission이 생길 때만 코멘트 재생성
- 캐시 키: `memberUuid`
- 캐시 무효화 조건: 해당 회원의 새 Submission 생성 시
- 구현 방식: DB 테이블 또는 Redis에 `(memberUuid, comment, generatedAt, lastSubmissionCount)` 저장
- `lastSubmissionCount`와 현재 Submission count 비교로 변화 감지

### 7. 서비스 구조

- **TopicAnalysisService** (신규): 토픽별 정답률/문제수 집계 전담
  - 위치: `PQL-Domain-Submission` 모듈
  - 의존: `SubmissionRepository`, `TopicRepository`, `QuestionRepository`
- **AiCommentService** (신규): AI 코멘트 생성 및 캐싱 전담
  - 위치: `PQL-Domain-AI` 모듈
  - 의존: `GeminiClient`, `TopicAnalysisService`, 캐시 저장소
- **ProgressController** 확장: 2개 엔드포인트 추가

### 8. 필요한 신규 Repository 쿼리

**SubmissionRepository:**

```sql
-- 토픽별 정답률 (최근 7일, 문제별 최근 시도)
WITH latest AS (
    SELECT s.question_uuid, s.is_correct, q.topic_uuid,
           ROW_NUMBER() OVER (PARTITION BY s.question_uuid ORDER BY s.submitted_at DESC) rn
    FROM submission s
    JOIN question q ON s.question_uuid = q.question_uuid
    WHERE s.member_uuid = :memberUuid
      AND s.submitted_at >= :since
)
SELECT topic_uuid,
       COUNT(*) as solved_count,
       AVG(CASE WHEN is_correct THEN 1.0 ELSE 0.0 END) as correct_rate
FROM latest
WHERE rn = 1
GROUP BY topic_uuid
```

**QuestionRepository:**

```sql
-- 토픽별 활성 문제 수
SELECT q.topic_uuid, COUNT(*) as total_count
FROM question q
WHERE q.is_active = true
GROUP BY q.topic_uuid
```
