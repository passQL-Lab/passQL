# ➕[기능추가][AI][관리자] Qdrant 임베딩 인덱스 관리 화면 구현

**라벨**: `작업전`
**담당자**: 

---

📝현재 문제점
---

- 문제 등록 시 Qdrant 색인은 VirtualThread로 자동 수행되나, 실패 여부를 관리자가 확인할 방법이 없음
- 전체 문제 재색인(bulk re-index) 기능이 Python AI 서버(`POST /api/ai/index-questions-bulk`)에 구현됐으나 관리자 화면에서 트리거할 수 없음
- 특정 문제의 색인 상태(Qdrant에 벡터가 있는지)를 조회하거나 단건 재색인하는 UI가 없음
- 미색인 문제(DB에는 있으나 Qdrant에 벡터 없음)를 탐지하는 수단이 없음
- 현재 관리자 모니터링 화면(`/admin/monitor`)에 Qdrant/임베딩 관련 지표가 전혀 없음

🛠️해결 방안 / 제안 기능
---

- 관리자 화면에 **Qdrant 인덱스 관리** 페이지 신설 (`/admin/embeddings`)
- 전체 재색인 버튼 — 활성 문제 전체를 `POST /api/ai/index-questions-bulk`로 일괄 색인, 진행 결과(succeeded/failed) 표시
- 개별 문제 재색인 버튼 — 문제 상세 페이지 또는 목록에서 단건 `POST /api/ai/index-question` 트리거
- 미색인 문제 목록 — DB의 활성 문제 중 Qdrant에 벡터가 없는 문제 탐지 및 일괄 색인 버튼
- Qdrant 컬렉션 상태 카드 — `passql_questions` 컬렉션의 총 포인트 수, 벡터 차원, 상태 표시 (Python AI 서버를 통해 조회)

⚙️작업 내용
---

- Python AI 서버: Qdrant 컬렉션 정보 조회 엔드포인트 추가 (`GET /api/ai/index-status`)
- Python AI 서버: 특정 문제 UUID의 색인 여부 확인 엔드포인트 추가 (`GET /api/ai/index-status/{questionUuid}`)
- Java `AiGatewayClient`: 위 엔드포인트 클라이언트 메서드 추가
- Java `AdminEmbeddingController` 신설 (PQL-Web)
- Thymeleaf 임베딩 관리 페이지 (`templates/admin/embeddings.html`) 구현
- 관리자 레이아웃 사이드바에 메뉴 항목 추가
