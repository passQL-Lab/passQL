# ⚙️[기능개선][Settings] 설정 화면 개발자 모드 Easter Egg 잠금 해제

**라벨**: `작업전`
**담당자**: 

---

📝현재 문제점
---

- `client/src/pages/Settings.tsx`의 개발자 섹션(localStorage 초기화 버튼)이 모든 사용자에게 항상 노출되어 있음
- 일반 클라이언트 사용자도 개발자 전용 기능을 볼 수 있어 혼란 및 오작동 위험 존재

🛠️해결 방안 / 제안 기능
---

- 개발자 섹션을 기본적으로 숨기고, Easter Egg 방식(앱 정보 5번 연속 클릭)으로만 잠금 해제
- 클릭 횟수별 daisyUI toast 피드백 제공
  - 3번: "개발자 모드까지 2번 남았습니다"
  - 4번: "개발자 모드까지 1번 남았습니다"
  - 5번: "개발자 모드가 활성화되었습니다" → 개발자 섹션 노출
- 잠금 해제 상태는 세션 한정 (새로고침 시 다시 숨겨짐)
- localStorage 초기화 1차 클릭 시 경고 toast 표시: "초기화하면 풀이 기록, 닉네임이 모두 삭제되고 새 계정으로 시작됩니다. 되돌릴 수 없어요."

⚙️작업 내용
---

- `Settings.tsx` 내 `devUnlocked` state 추가 및 개발자 섹션 조건부 렌더링 적용
- 버전 row 클릭 핸들러 및 카운트 ref 추가
- daisyUI toast 렌더링 블록 추가
- localStorage 초기화 warning 문구 개선
