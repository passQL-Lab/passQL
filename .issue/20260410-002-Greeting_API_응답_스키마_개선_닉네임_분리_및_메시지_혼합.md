# 🚀[기능개선][Greeting] 응답 스키마 개선 - 닉네임 분리, 이모지 제거, D-day 메시지 혼합

**라벨**: `작업전`
**담당자**:

---

📝현재 문제점
---

- 현재 `GreetingResponse`는 `{ message: "홍길동님, 좋은 아침이에요! ... 📚" }` 형태로 닉네임/본문/이모지가 하나의 문자열로 뭉쳐서 내려가고 있음.
- 프론트엔드에서 닉네임 부분만 별도 스타일링(하이라이트 등)을 적용하고 싶어도 문자열 파싱이 필요해 구현이 어렵고 지저분함.
- 응답 문자열에 이모지(📚 💪 🔥 ✨ 🎯 등)가 하드코딩되어 있어 프론트에서 일관된 아이콘/톤을 적용하기 어려움.
- 메시지 선택 로직이 구간 단위로 단일 풀만 사용해, D-day 구간(D-7 이내, D-30 이내)에 들어가면 "D-5 스퍼트!" 같은 긴급 메시지만 반복적으로 노출됨. 매일 들어오는 사용자 입장에서 다양성이 부족하고 피로감이 있음.
- 회원 조회 실패 시 `MEMBER_NOT_FOUND` 예외를 던져, 예외 상황에서도 홈 인사말을 표시해야 하는 프론트의 fallback 요구를 만족시키지 못함.

🛠️해결 방안 / 제안 기능
---

**1. 응답 스키마 개선**

- `GreetingResponse`를 다음 3개 필드로 분리:
  - `nickname`: 회원 닉네임 (예외 케이스에서는 기본값 `"회원"`)
  - `message`: `{nickname}` 플레이스홀더가 포함된 본문 템플릿
  - `context`: 인사말 컨텍스트 enum (`GENERAL` / `EXAM_DAY` / `URGENT` / `COUNTDOWN`)
- 프론트는 `message.replace("{nickname}", nickname)`로 간단히 렌더하고, 닉네임 부분만 원하는 스타일로 감쌀 수 있게 됨.
- `context` 필드로 상황별 UI 톤 분기(시험 당일 강조, 일반 인사 등) 여지 확보.

**2. 이모지 전면 제거**

- 모든 메시지 풀(`generalMessages`, `examDayMessages`, `urgentMessages`, `countdownMessages`)에서 이모지를 제거해 텍스트만 내려줌.
- 아이콘 노출 여부/종류는 프론트가 `context` 값에 따라 자체 결정.

**3. 예외 케이스 fallback 처리**

- 회원 조회 실패 / 비로그인 / 닉네임 공백인 경우 예외 대신 `nickname = "회원"` + `context = GENERAL` 기반 일반 인사말로 폴백.
- 예외 케이스에서는 D-day 관련 메시지(EXAM_DAY/URGENT/COUNTDOWN)를 사용하지 않음. "선택한 시험"이라는 개인 컨텍스트가 성립하지 않기 때문.

**4. D-day 구간 메시지 혼합 (가중치 방식)**

기존: 구간에 해당하면 해당 풀 단일 사용.
변경: D-day 구간이어도 일반 메시지를 같이 섞어, 긴박할수록 D-day 메시지 비중을 높이는 방식으로 개선.

| 구간 | D-day 메시지 | 일반 메시지 | 비고 |
|------|:---:|:---:|------|
| D-day = 0 (당일) | 100% | 0% | 당일은 EXAM_DAY 고정 |
| D-1 ~ D-3 | 70% | 30% | URGENT |
| D-4 ~ D-7 | 50% | 50% | URGENT |
| D-8 ~ D-30 | 40% | 60% | COUNTDOWN |
| D-30 초과 / 시험 종료 | 0% | 100% | GENERAL |

- 구현은 풀을 물리적으로 합치지 않고, `ThreadLocalRandom.nextInt(100)` 임계값으로 먼저 어느 풀에서 뽑을지 결정한 뒤 해당 풀 내부에서 균등 랜덤 추출 (각 풀의 크기 차이에 영향받지 않고 의도한 비율 유지).
