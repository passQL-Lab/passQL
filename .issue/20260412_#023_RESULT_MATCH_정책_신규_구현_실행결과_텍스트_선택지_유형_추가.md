# ➕[기능추가][문제설계] RESULT_MATCH 정책 신규 구현 — 실행 결과 텍스트 선택지 유형 추가

**라벨**: `작업전`
**담당자**: 

---

📝현재 문제점
---

- `AI_ONLY` 정책은 "주어진 SQL과 동일한 결과를 내는 SQL은?" 유형으로 재정의됨 (#108)
- 그러나 SQLD 시험의 핵심 유형인 **"다음 SQL의 실행 결과로 올바른 것은?"** 유형이 아직 없음
- 이 유형은 선택지가 SQL 쿼리가 아닌 **실행 결과 테이블** 형태여야 함
- 사용자가 SQL을 머릿속으로 실행해 결과를 예측하는 것이 핵심 학습 목표

🛠️해결 방안 / 제안 기능
---

- `ChoiceSetPolicy` enum에 `RESULT_MATCH` 추가
- 선택지 body 포맷: JSON 배열 (`[{"NAME":"홍길동","DEPT_NAME":"개발팀"}, ...]`)
- `kind = TEXT`로 저장 — 기존 `ChoiceKind.TEXT` 활용
- 선택지 생성: answerSql 실행 결과를 기준으로 AI가 정답 결과 1개 + 오답 결과 3개 생성
- 검증: body JSON 파싱 후 answerSql 실행 결과와 문자열 비교 (SQL 실행 방식 아님)
- 프론트: `kind === "TEXT"`면 컴팩트 결과 테이블로 렌더링, 실행 버튼 없음

⚙️작업 내용
---

**백엔드**
- `ChoiceSetPolicy.RESULT_MATCH` enum 값 추가
- `generate_choice_set_result_match` 프롬프트 마이그레이션 추가
- `ChoiceSetGenerationService` — `RESULT_MATCH` 분기 추가 (answerSql 실행 후 AI 호출)
- `SandboxValidator` — `RESULT_MATCH` 검증 로직 추가 (JSON body vs answerSql 결과 비교)
- `ChoiceSetSaveService` — `RESULT_MATCH`일 때 `kind = TEXT`로 저장

**프론트엔드**
- `ChoiceCard` — `kind === "TEXT"`면 컴팩트 결과 테이블 렌더링, 실행 버튼 제거
- `ResultTable` 컴팩트 변형 컴포넌트 (선택지용)
- 가이드라인 MD 2개 — `RESULT_MATCH` 정책 설명 추가
