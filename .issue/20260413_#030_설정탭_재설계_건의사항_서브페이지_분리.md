# ⚙️[기능개선][Settings] 설정 탭 재설계 — 건의사항 서브페이지 분리

**라벨**: `작업전`
**담당자**: 

---

📝현재 문제점
---

- 설정 탭(`Settings.tsx`)에 건의사항 입력 폼(`FeedbackForm`)과 목록(`FeedbackList`)이 풀(full) UI로 펼쳐져 노출되어 있음
- 설정 항목이 늘어날수록 설정 탭이 무거워지는 구조적 문제
- iOS 설정 앱, 토스 등 표준 모바일 패턴과 다르게 설정 탭이 기능 페이지 역할을 겸하고 있음

🛠️해결 방안 / 제안 기능
---

- 설정 탭은 순수 **row 목록**만 보여주는 구조로 변경
- 건의사항 섹션은 `SettingsRow` + `ChevronRight` 아이콘으로 교체, 클릭 시 `/settings/feedback` 서브페이지로 이동
- `/settings/feedback`은 AppLayout 밖 독립 라우트 — 탭바 없는 몰입형 화면 (기존 `QuestionDetail` 패턴 동일)
- `FeedbackForm`, `FeedbackList` 컴포넌트는 수정 없이 서브페이지로 이동

⚙️작업 내용
---

- `App.tsx`: `/settings/feedback` 독립 라우트 추가
- `Settings.tsx`: 건의사항 섹션 → `SettingsRow` + ChevronRight로 교체, FeedbackForm/List/관련 import 제거
- `pages/SettingsFeedback.tsx` 신규 생성: 뒤로가기 헤더 + 오프라인 배너 + FeedbackForm + FeedbackList
