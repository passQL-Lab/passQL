# ⚙️[기능개선][FE] SchemaViewer 테이블 레이아웃 및 시각성 개선

**라벨**: `작업전`
**담당자**: 

---

📝현재 문제점
---

- 스키마 테이블 카드에 배경색이 없어 페이지 배경(`#FAFAFA`)에 묻혀 카드 경계가 불분명함
- 테이블 목록이 `flex` 가로 나열 구조여서 테이블이 4개일 경우 좁은 영역에 찌그러져 칼럼명이 잘리거나 겹침
- 테이블 헤더에 테이블명만 단독 표시되어 "이것이 테이블이다"라는 시각적 식별이 어려움
- 칼럼 수가 많은 테이블에서 가로 스크롤 구조가 없어 내용이 잘림

🛠️해결 방안 / 제안 기능
---

- 각 테이블 카드에 `background: var(--color-surface-card)` (`#FFFFFF`) 배경색 명시
- 테이블 목록 컨테이너를 세로 스택(`space-y-3`)으로 변경하여 테이블별 독립 카드 표현
- 테이블 헤더에 `Table2` 아이콘(lucide-react) + 테이블명 조합으로 시각적 식별 강화
- 각 카드 내부에 `overflow-x-auto` 래퍼를 추가하여 칼럼이 많아도 가로 스크롤로 확인 가능하게 처리
- 샘플 데이터 카드에도 동일하게 배경색 + 내부 가로 스크롤 적용

⚙️작업 내용
---

- `client/src/components/SchemaViewer.tsx` 수정
  - `Table2` 아이콘 import 추가
  - 컨테이너: `flex gap-2 overflow-x-auto` → `space-y-3`
  - 카드 래퍼: `scrollSnapAlign` 제거, `space-y-2` 유지
  - 스키마 구조 카드: 배경색 추가, 테이블 헤더 아이콘+텍스트 변경, 내부 `overflow-x-auto` 추가
  - 샘플 데이터 카드: 배경색 추가, 내부 `overflow-x-auto` 추가
