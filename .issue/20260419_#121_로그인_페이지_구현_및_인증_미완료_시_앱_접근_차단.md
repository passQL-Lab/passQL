# ➕[기능추가][Auth] 로그인 페이지 구현 및 인증 미완료 시 앱 접근 차단

**라벨**: `작업전`
**담당자**: 

---

📝현재 문제점
---

- 현재 앱은 별도 로그인 없이 익명 UUID(`passql_member_uuid`)로 자동 등록(`POST /members/register`)되는 구조
- 랜딩 페이지(홈)에서 표시되는 개인화 데이터(학습 현황, 오늘의 문제 풀이 여부, 추천 문제, 인사 메시지 등)는 모두 인증된 회원 UUID에 종속됨
- 소셜 로그인(OAuth) 및 JWT 인증 시스템이 백엔드에 구현되어 있으나, 프론트엔드에 로그인 진입점이 없어 인증 플로우가 연결되지 않은 상태
- 비로그인 상태에서도 모든 라우트에 접근 가능하여 인증 기반 API 호출 시 오류 발생 가능

🛠️해결 방안 / 제안 기능
---

- 로그인 페이지(`/login`) 신규 구현 — 소셜 로그인 버튼(Google, Kakao 등 백엔드 지원 provider) UI
- 인증 상태 전역 관리 — JWT 토큰을 `memberStore`에 통합하거나 별도 `authStore` 도입
- `PrivateRoute` 또는 라우터 레벨 가드 추가 — 로그인 완료 전 모든 라우트를 `/login`으로 redirect
- OAuth 콜백 처리 라우트(`/oauth/callback`) 구현 — 인증 코드 교환 후 JWT 저장 및 홈으로 이동
- 로그인 성공 후 기존 익명 데이터 마이그레이션 여부 백엔드와 협의

⚙️작업 내용
---

- `src/pages/Login.tsx` — 로그인 페이지 UI (소셜 로그인 버튼, passQL 브랜딩)
- `src/pages/OAuthCallback.tsx` — OAuth 인증 코드 수신 및 JWT 교환 처리
- `src/stores/authStore.ts` — JWT 토큰 및 인증 상태 관리 (또는 memberStore 확장)
- `src/components/PrivateRoute.tsx` — 인증 여부에 따른 라우트 접근 제어
- `src/App.tsx` — 라우터에 `/login`, `/oauth/callback` 추가 및 인증 가드 적용
- `src/api/auth.ts` — 로그인/토큰 갱신/로그아웃 API 함수 추가
