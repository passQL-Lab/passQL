# ⚙️[기능개선][Server] Flyway 마이그레이션 SQL 멱등성 안전장치 규칙 정립

**라벨**: `작업전`
**담당자**: 

---

📝현재 문제점
---

- `V0_0_119__fix_feedback_uuid_column_types.sql` 초안에서 안전장치 없이 `ALTER TABLE feedback ALTER COLUMN ... TYPE UUID` 를 작성함
- 이미 `uuid` 타입인 환경이나 컬럼이 없는 환경에서 실행 시 오류 발생 가능
- 신규 마이그레이션 작성 시 DDL 변경이 멱등성 없이 작성되는 패턴이 반복됨
- `CLAUDE.md` DB 스키마 관리 규칙에 타입 변환용 `ALTER` SQL 작성 가이드가 누락되어 있음

🛠️해결 방안 / 제안 기능
---

- `ALTER COLUMN TYPE` 등 타입 변환 SQL은 `DO $$ BEGIN ... END $$;` PL/pgSQL 블록으로 감싸고, `information_schema.columns`로 현재 타입을 확인 후 조건 실행
- `CLAUDE.md` DB 스키마 관리 규칙에 아래 항목 추가:
  - DDL 변경(`ALTER`, 컬럼 추가/삭제/타입변경)은 반드시 멱등성을 보장해야 한다
  - 타입 변환은 조건부 실행(`IF EXISTS ... THEN ... END IF`) 패턴 사용
  - 작업 체크리스트에 "마이그레이션 SQL이 멱등성을 갖는가?" 항목 추가
- 관련 파일: `server/CLAUDE.md`, `server/PQL-Web/src/main/resources/db/migration/`
