# ⚙️ [기능추가][Client] React Client 초기 프로젝트 구성

📝 현재 문제점
---

- `client/` 디렉토리가 `.gitkeep`만 존재하는 빈 상태로, 프론트엔드 개발을 시작할 수 없는 상황입니다.
- PRD에 정의된 4탭 SPA 구조(홈/문제/통계/설정)와 문제 상세 화면을 구현하기 위한 기반 프로젝트 셋업이 필요합니다.
- 6일 데드라인 내 MVP 완성을 위해 최소 의존성으로 빠르게 부트스트랩이 필요합니다.

🛠️ 해결 방안 / 제안 기능
---

- Vite + React 18 + TypeScript 기반으로 `client/` 프로젝트를 초기화합니다.
- PRD 2.3절의 프론트엔드 기술 스택 중 MVP에 필수적인 최소 의존성만 우선 도입합니다.
- 4탭 라우팅 구조와 빈 페이지 컴포넌트, 사용자 식별(localStorage UUID) 기반을 마련하여 이후 화면 구현이 바로 이어질 수 있도록 합니다.
- CodeMirror, Recharts, react-markdown 등 특정 화면에서만 필요한 라이브러리는 해당 화면 구현 시점에 추가합니다.
