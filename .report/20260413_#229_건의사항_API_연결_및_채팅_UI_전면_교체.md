# 건의사항 API 연결 및 채팅 UI 전면 교체

**이슈**: [#229](https://github.com/passQL-Lab/passQL/issues/229)

---

### 📌 작업 개요

백엔드 건의사항 API 구현 완료에 맞춰 임시 404 fallback 로직을 제거하고 실제 API와 연결. 아울러 건의사항 화면을 메신저 스타일 채팅 버블 UI로 전면 재설계.

---

### 🔍 문제 분석

기존 `useMyFeedback` 훅은 `GET /feedback/me` 응답이 404일 때 `{ items: [] }`로 graceful fallback 처리하는 임시 코드를 포함하고 있었다. 백엔드 API 미구현 기간 대응용이었으며, 실제 API 정상화 이후에도 에러를 무시하는 문제가 있었다.

---

### ✅ 구현 내용

#### 1. useFeedback.ts — 404 fallback 제거
- **파일**: `src/hooks/useFeedback.ts`
- **변경 내용**: `queryFn` 내 404 catch 분기 완전 제거. 에러는 React Query `isError`로 버블링되어 `ErrorFallback` 컴포넌트가 렌더링됨
- **추가**: `staleTime: 0` 설정. 페이지 진입 시마다 최신 상태를 반영해 `PENDING → REVIEWED → APPLIED` 상태 변화를 즉시 감지

#### 2. SubpageLayout — fullHeight 모드 추가
- **파일**: `src/components/SubpageLayout.tsx`
- **변경 내용**: `fullHeight` prop 추가. `true` 시 `h-screen flex-col overflow-hidden` 레이아웃으로 전환되어 헤더 고정 + 콘텐츠 스크롤 + 하단 입력 고정 구조 지원
- **이유**: 채팅형 레이아웃에서 하단 입력창이 고정된 상태로 버블 목록만 스크롤되는 구조를 구현하기 위함

#### 3. SettingsFeedback — 채팅 레이아웃 적용
- **파일**: `src/pages/SettingsFeedback.tsx`
- **변경 내용**: `SubpageLayout fullHeight` 모드로 전환. 오프라인 배너(`WifiOff`) + 버블 목록(`FeedbackList`) + 하단 고정 입력창(`FeedbackForm`) 구조로 재구성

#### 4. FeedbackItem — 채팅 버블 컴포넌트
- **파일**: `src/components/FeedbackItem.tsx`
- **변경 내용**: 상태별 버블 스타일 구분
  - `PENDING`: 인디고 solid 배경 + 흰 텍스트 (전송 완료 상태 강조)
  - `REVIEWED`: 연한 인디고 배경 + 인디고 보더 (관리자 확인 완료)
  - `APPLIED`: 연한 초록 배경 + 초록 보더 (반영됨)
- 버블 하단에 상대 시간(방금/N분 전/오늘/어제/N일 전) + 상태 pill 표시
- `toDateKey()` 헬퍼 export: ISO 8601 → 로컬 `YYYY-MM-DD` 변환 (날짜 구분선 비교용)

#### 5. FeedbackList — 날짜 구분선 및 날짜 자정 버그 수정
- **파일**: `src/components/FeedbackList.tsx`
- **변경 내용**:
  - API 반환(최신순)을 reverse하여 오래된 것이 위, 최신이 아래 표시
  - 날짜가 바뀌는 지점에 `DateDivider` 자동 삽입
  - 로딩 중 버블 스켈레톤(animate-pulse) / 빈 상태 EmptyState / 에러 ErrorFallback 처리
  - stagger 애니메이션 주입
- **버그 수정**: `DateDivider` 날짜 레이블 계산 시 ISO 문자열을 직접 슬라이스하면 UTC 기준이라 로컬 자정 경계에서 날짜가 어긋나는 문제 수정. `setHours(0,0,0,0)`으로 로컬 기준 자정 비교로 변경

#### 6. FeedbackForm — 하단 고정 입력창
- **파일**: `src/components/FeedbackForm.tsx`
- **변경 내용**:
  - 글자수 카운터(0/500) — 1자 이상 입력 시 인디고 강조
  - 전송 실패 시 인라인 에러 배너(`AlertTriangle`) + 재시도 버튼
  - 전송 성공 시 textarea 초기화 + 목록 캐시 자동 무효화(refetch)
  - 오프라인 `disabled` 상태 시 textarea + 버튼 비활성

### 🔧 특이사항

**날짜 자정 경계 버그**

기존 날짜 구분선은 ISO 문자열의 앞 10자(`"YYYY-MM-DD"`)를 그대로 사용하여 날짜 키를 생성했다. 이 방식은 UTC 기준이므로 KST(UTC+9) 환경에서 자정 전후 9시간 동안 날짜가 실제 로컬 날짜와 다르게 표시되는 버그가 있었다. `toDateKey()` 함수에서 `Date` 객체의 로컬 메서드(`getFullYear`, `getMonth`, `getDate`)를 사용하도록 수정하여 해결.

**staleTime 0 선택 이유**

건의사항 상태(`PENDING → REVIEWED → APPLIED`)는 관리자 액션으로 변경되므로 프론트가 변화 시점을 예측할 수 없다. 매 페이지 진입마다 서버 상태를 재확인하도록 `staleTime: 0`으로 설정.

### 🧪 검증 사항

- 건의사항 제출 → 목록 자동 refetch → 새 버블 노출 확인
- 상태 값(PENDING / REVIEWED / APPLIED)별 버블 색상 및 pill 렌더링 확인
- 오프라인 상태 시 오프라인 배너 + 입력창 비활성 확인
- 자정 경계(UTC 00:00 = KST 09:00) 전후 날짜 구분선 정확성 확인
