# 질문 삭제, 수정에서 JSON 가져오기 기능 누락 문제 개선 필요

## 개요

관리자 문제 관리 페이지에서 (1) 문제 Cascade 삭제 기능 신규 구현, (2) 문제 수정 폼에 JSON 불러오기/붙여넣기 기능 추가, (3) 기존 등록 폼의 JSON 로직을 공통 JS로 추출하여 코드 중복을 해소했다. 삭제 시 8개 자식 테이블을 FK 역순으로 정리하며, 진행중인 퀴즈 세션에서 사용 중인 문제는 삭제를 거부하는 안전장치를 포함한다.

## 변경 사항

### 백엔드 — Cascade 삭제 API

- `ErrorCode.java`: `QUESTION_IN_ACTIVE_SESSION(HttpStatus.CONFLICT)` 에러 코드 추가
- `AdminQuestionDeleteService.java` **(신규)**: `PQL-Web/service/`에 삭제 오케스트레이션 서비스 생성. `PQL-Domain-Question`과 `PQL-Domain-Submission` 간 순환 의존을 피하기 위해 양쪽 모듈 모두 접근 가능한 `PQL-Web`에 배치
- `AdminQuestionController.java`: `@DeleteMapping("/{uuid}")` 추가. `RedirectAttributes`로 flash 메시지 전달
- `application.yml`: `spring.mvc.hiddenmethod.filter.enabled: true` 추가 — HTML form의 `_method=delete` 히든 필드가 `@DeleteMapping`으로 라우팅되도록 필터 활성화

### 백엔드 — Repository 벌크 삭제 메서드

- `QuizSessionRepository.java`: `findByStatus(QuizSessionStatus)` 추가 — 진행중 세션 조회
- `QuestionChoiceSetRepository.java`: `deleteByQuestionUuid(UUID)` 추가
- `QuestionChoiceSetItemRepository.java`: `deleteByChoiceSetUuidIn(List<UUID>)` 추가
- `DailyChallengeRepository.java`: `deleteByQuestionUuid(UUID)` 추가
- `SubmissionRepository.java`: `deleteByQuestionUuid(UUID)` 추가
- `ExecutionLogRepository.java`: `deleteByQuestionUuid(UUID)` 추가

### 프론트엔드 — 문제 목록 삭제 UI

- `questions.html`: 각 행 액션 컬럼에 삭제 버튼(trash-2 아이콘) 추가. 클릭 시 DaisyUI dialog 모달로 문제 제목·경고 문구 표시 후 확인 삭제. Flash 메시지(성공/에러) 표시 영역 추가

### 프론트엔드 — JSON 불러오기 공통화

- `question-json-loader.js` **(신규)**: `applyJson`, `loadJsonFile`, `openJsonPasteModal`, `applyJsonFromPaste` 4개 함수를 공통 JS로 추출. `initQuestionJsonLoader({ confirmOverwrite })` 설정 객체로 등록/수정 폼 양쪽 지원
- `question-register.html`: 인라인 JSON 함수 4개 제거 → 공통 JS `<script src>` 호출로 교체. `th:attr` → `th:data-code` 문법 통일
- `question-edit.html`: 헤더에 JSON 파일 불러오기/붙여넣기 버튼 추가. JSON 붙여넣기 모달 추가. 토픽 select option에 `th:data-code` 속성 추가. 스키마 필드들을 `id="schemaSection"` div로 감싸 실행 모드 토글 시 표시/숨김 동작. `confirmOverwrite: true`로 덮어쓰기 확인

## 주요 구현 내용

### Cascade 삭제 순서 (FK 의존성 역순)

```
1. 안전장치: IN_PROGRESS QuizSession에서 questionOrderJson 포함 검사 → 있으면 409 Conflict
2. ExecutionLog 삭제
3. Submission 삭제
4. DailyChallenge 삭제
5. QuestionConceptTag 삭제
6. QuestionChoiceSetItem 삭제 (ChoiceSet UUID 목록 → deleteByChoiceSetUuidIn)
7. QuestionChoiceSet 삭제
8. QuestionChoice 삭제
9. Question 본체 삭제
```

전체 과정이 `@Transactional` 단일 트랜잭션으로 실행되며, 어느 단계에서 실패하든 전체 롤백된다.

### 크로스 모듈 의존성 해결

`QuestionService`(PQL-Domain-Question)에서 `SubmissionRepository`(PQL-Domain-Submission)를 직접 주입하면 순환 의존(`Question → Submission → Question`)이 발생한다. 이를 피하기 위해 `AdminQuestionDeleteService`를 `PQL-Web` 모듈에 배치하여 양쪽 도메인 모듈의 Repository를 모두 주입받도록 구성했다.

### HiddenHttpMethodFilter 활성화

Spring Boot 2.2+ 에서는 `HiddenHttpMethodFilter`가 기본 비활성화다. HTML form에서 `<input type="hidden" name="_method" value="delete"/>`로 DELETE 요청을 보내려면 `spring.mvc.hiddenmethod.filter.enabled: true` 설정이 필요하다. 기존 `prompts.html`의 `th:method="put"` 패턴도 이 설정에 의존한다.

## 주의사항

- **QuizSession 안전장치**: `String.contains(uuid)` 방식으로 JSON 배열에서 UUID 포함 여부를 검사한다. UUID v4의 36자 고정 포맷 특성상 위양성은 사실상 발생하지 않으나, 엄밀한 JSON 파싱이 아닌 문자열 검색이라는 점은 인지해야 한다.
- **Hard delete**: 문제 삭제 시 관련 Submission·ExecutionLog가 함께 삭제된다. 운영 초기이므로 현재는 적절하나, 향후 사용자 학습 이력 보존이 중요해지면 Soft delete 전환을 검토해야 한다.
- **범위 밖**: Soft delete, 벌크 삭제(다중 선택), 삭제 복원, 감사 로그는 이번 범위에 포함되지 않는다.
