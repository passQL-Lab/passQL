# 문제 등록 가이드에 테이블 이름 작성 규칙 추가 #193

## 개요

관리자가 SQLD 문제를 등록할 때 `SQLD_14`, `SQLD46_01_EMP` 처럼 시험 회차·문제 번호를 테이블 이름에 그대로 사용하는 관행이 있었다. 각 문제는 독립된 Sandbox에서 실행되므로 문제 간 테이블 이름 충돌이 없는데, 이 사실이 가이드 어디에도 명시되어 있지 않아 불필요하게 긴 이름이 학습자 화면에 그대로 노출됐다. 단건·대량 가이드 MD 파일과 관리자 페이지 가이드 모달에 테이블 이름 작성 규칙을 추가했다.

## 변경 사항

### 가이드 문서

- `server/PQL-Web/src/main/resources/static/docs/question-single-guide.md`: `schemaDdl 작성 규칙` 섹션 아래 `테이블 이름 작성 규칙 ★` 항목 추가 — Sandbox 격리 원리 설명, 나쁜 예/좋은 예 코드 블록, 상황별 권장 이름 표 포함
- `server/PQL-Web/src/main/resources/static/docs/question-bulk-guide.md`: 동일 규칙 추가 (예시를 JSON 문자열 형태로 조정)

### 관리자 페이지 가이드 모달

- `server/PQL-Web/src/main/resources/templates/admin/question-register.html`: 가이드 모달에 `⚠️ 테이블 이름 작성 규칙` UI 섹션 추가(alert + mockup-code 블록), AI 변환 프롬프트 텍스트의 `schemaDdl 작성:` 항목에 테이블 이름 규칙 줄 삽입
- `server/PQL-Web/src/main/resources/templates/admin/questions.html`: 대량 등록 페이지 가이드 모달에 동일 UI 섹션 추가 및 AI 프롬프트 텍스트 수정

## 주요 구현 내용

**Sandbox 격리 원리 명시**: 가이드에 "각 문제는 독립된 Sandbox(격리된 DB)에서 실행된다"는 사실을 명시했다. 이로써 관리자가 문제 간 충돌을 걱정하지 않고 `EMP`, `DEPT`, `T1` 같은 짧고 의미 있는 이름을 사용할 수 있음을 안내한다.

**상황별 권장 이름 표**: 임직원·부서·주문·제품 등 데이터 성격에 따른 권장 이름을 표 형태로 제공했다. 의미가 없는 더미 데이터(AND/OR 우선순위 테스트 등)에는 `T1`, `T2`, `TAB_A` 사용을 권장한다.

**AI 프롬프트 동기화**: 관리자가 SQLD 원문 문제를 AI에게 붙여 JSON으로 변환할 때도 올바른 테이블 이름이 생성되도록, AI 변환 프롬프트 텍스트의 `schemaDdl 작성:` 항목에도 동일 규칙을 삽입했다.

## 주의사항

- 기존에 이미 등록된 문제의 테이블 이름(`SQLD_14` 등)은 이번 변경으로 자동 수정되지 않는다. 기존 문제는 필요 시 수동으로 재등록하거나 DB에서 직접 수정해야 한다.
- AI 프롬프트에 규칙을 추가했더라도 AI가 반드시 이를 따른다는 보장은 없다. 등록 전 관리자가 생성된 JSON의 테이블 이름을 직접 검토하는 것이 바람직하다.
