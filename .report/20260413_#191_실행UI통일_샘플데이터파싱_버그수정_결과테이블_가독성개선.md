# 실행 UI 통일·샘플 데이터 파싱 버그 수정·결과 테이블 가독성 개선 #191

## 개요

문제 풀기 화면에서 실행 버튼 스타일이 세 곳에서 제각각이었고, `INSERT INTO T1 VALUES (1),(2),(3)` 형식의 샘플 데이터가 깨져 표시되는 파싱 버그가 있었다. 또한 `SqlPlayground`의 네트워크 오류가 사용자에게 전달되지 않았고, 선택된 ChoiceCard 위의 실행 결과 테이블이 흰 박스로 감싸져 시각적으로 튀는 문제가 있었다. 이를 전면 정리했다.

## 변경 사항

### 실행 버튼 통일

- `client/src/components/ChoiceCard.tsx`: `btn btn-xs btn-outline` (daisyUI) → `btn-run-all`로 교체. 선택된 카드(파란 배경) 위에서는 `btn-run-all--inverted` 클래스 적용
- `client/src/components/SqlPlayground.tsx`: `btn-compact` → `btn-run-all`로 교체. `Play` 아이콘 대신 인라인 SVG 삼각형으로 통일
- `client/src/styles/components.css`: `btn-run-all--inverted` variant 추가 (흰색 반투명 배경·보더)

### 샘플 데이터 파싱 버그 수정

- `client/src/components/SchemaViewer.tsx`: `parseSampleData` 정규식 개선. 기존 `(.+?)` 방식은 `VALUES (1),(2),(3)` 형식에서 첫 번째 `)` 이후를 버렸음. VALUES 이후 세미콜론 전체를 캡처 후 `(...)` 단위로 행을 재분리하는 방식으로 수정

### SqlPlayground 개선

- `client/src/components/SqlPlayground.tsx`: `handleExecute`에 `catch` 블록 추가 — 네트워크 오류·5xx 시 `error-card`로 에러 메시지 표시, 이전 실행 결과도 초기화
- 미사용 `questionUuid` prop 제거 (interface 및 호출부 동시 정리)
- 헤더를 `Terminal` 아이콘 + "직접 실행해보기" 레이블로 교체 — 다른 섹션 레이블(해설, SQL 실행 비교)과 스타일 통일
- `textarea` 인라인 `style` 5개 → `sql-playground-textarea` CSS 클래스로 분리
- `client/src/pages/QuestionDetail.tsx`: `SqlPlayground`에 `questionUuid` prop 전달 제거

### 결과 테이블 가독성 개선

- `client/src/components/ResultTable.tsx`: `inverted` prop 추가. 선택된 카드 위에서는 흰 wrapper 대신 반투명 배경+흰색 텍스트로 처리
- `client/src/styles/components.css`: `result-table-meta--inverted`, `result-table-border--inverted`, `data-table--inverted` 클래스 추가

## 주요 구현 내용

**파싱 버그 수정**: `INSERT INTO T1 VALUES (1),(2),(3);`에서 기존 정규식 `\((.+?)\)`은 non-greedy로 인해 `(1)` 이후를 버렸다. 수정 후에는 `VALUES` 이후 `;` 전체를 하나의 문자열로 캡처한 뒤, 별도의 `\(([^)]*)\)` 정규식으로 각 괄호 단위 행을 재분리한다. `T1: [[1],[2],[3]]` 형태로 정확히 파싱된다.

**inverted 테이블**: 선택된 ChoiceCard(인디고 배경) 위에서 흰 `div` wrapper를 씌우면 카드 안에서 박스가 튀어 보인다. `inverted=true`일 때 테이블 border를 `rgba(255,255,255,0.25)`, 배경을 `rgba(255,255,255,0.15)`로, 텍스트를 흰색으로 처리하여 파란 배경과 자연스럽게 어우러지게 했다.

## 주의사항

- 선택된 카드 위 `error-card`(빨간 배경)는 inverted 처리 없이 그대로 표시된다. 파란 배경 위 빨간 에러 카드가 시각적으로 충분히 구분되므로 별도 처리 불필요.
- `parseSampleData`는 `VALUES` 절 안에 중첩 괄호(예: 함수 호출)가 있는 경우 파싱이 틀릴 수 있다. 현재 샘플 데이터 형식(단순 값 목록)에서는 문제 없음.
