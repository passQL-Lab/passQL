### 📌 작업 개요

passQL 프론트엔드 7개 화면의 mock 상수 데이터를 실제 백엔드 API로 전면 교체. TanStack Query 기반 커스텀 훅 7종 생성, 선택지 자동 실행 캐시 구현, Member API 연동(register/fetchMe/regenerateNickname), AiExplanationSheet 실 API 연결, 성능 최적화(React.memo, useMemo, useRef), 에러 처리 통일, 테스트 인프라 구축(vitest + 40건, 커버리지 96%).

**보고서 파일**: `.report/20260408_#13_Mock_데이터_실제_API_연동.md`

### 🎯 구현 목표

- 7개 화면(Home, Questions, QuestionDetail, AnswerFeedback, Stats, Settings + AiSheet)의 mock → 실 API 전환
- TanStack Query 훅 패턴으로 데이터 페칭 통일
- Member 도메인 API 연동 (register, fetchMe, regenerateNickname)
- 선택지 execute 캐시 정책 구현 (동일 선택지 재호출 방지)
- 성능 최적화 (React.memo, useMemo, useCallback useRef 패턴)
- 에러 처리 통일 + 테스트 커버리지 확보

### ✅ 구현 내용

#### Phase 1: TanStack Query 훅 생성 (src/hooks/)

- **`useProgress.ts`**: fetchProgress(staleTime: 0) + fetchHeatmap(staleTime: 5분) 조회 훅
- **`useQuestions.ts`**: fetchQuestions 조회 훅 (params → queryKey 자동 반영, staleTime: 30초)
- **`useTopics.ts`**: fetchTopics 조회 훅 (staleTime: Infinity — 정적 데이터)
- **`useQuestionDetail.ts`**: fetchQuestion 조회 + useExecuteChoice mutation + useSubmitAnswer mutation
- **`useMember.ts`**: fetchMe 조회 (닉네임 memberStore 동기화) + useRegenerateNickname mutation

#### Phase 2: 페이지별 API 연동

- **Home**: useProgress → 통계 카드, useMember → 닉네임 표시, skeleton loading, ErrorFallback
- **Questions**: useQuestions + useTopics → 필터 드롭다운(토픽/난이도) + 페이지네이션 + `Link` 라우팅
- **QuestionDetail**: useQuestionDetail → 문제 데이터, useExecuteChoice → 선택지 클릭 시 SQL 자동 실행, `Record<choiceKey, ExecuteResult>` 로컬 캐시, useSubmitAnswer → navigate state로 결과 전달
- **AnswerFeedback**: location.state에서 submitResult 수신, state 없으면 /questions redirect
- **Stats**: useProgress + useHeatmap → 요약 카드 + 히트맵 그리드, ErrorFallback
- **Settings**: useMemberStore UUID + 닉네임 표시, clipboard 복사, regenerateNickname 활성화

#### Phase 3: Member API 연동

- **`src/api/members.ts`** (신규): register(), fetchMe(query param), regenerateNickname(query param) 함수
- **`src/stores/memberStore.ts`** (리팩토링): ensureRegistered() 비동기 함수 추가. 첫 진입 시 POST /members/register → UUID + 닉네임 발급, 실패 시 fallback UUID. nickname 상태 + localStorage 동기화
- **`src/App.tsx`**: useEffect로 앱 시작 시 ensureRegistered() 호출

#### Phase 4: AiExplanationSheet 실 API 연결

- **AiExplanationSheet** (리팩토링): mock 타이머 제거, `isLoading`/`text` props 기반으로 전환. 외부에서 mutation 결과를 주입하는 구조
- **QuestionDetail**: ChoiceCard의 "AI에게 물어보기" → explainError mutation → AiSheet 열기
- **AnswerFeedback**: "AI에게 자세히 물어보기" → diffExplain mutation → AiSheet 열기
- **ChoiceCard**: `onAskAi` optional props 추가

#### Phase 5: API 문서 동기화

- **`src/types/api.ts`**: MemberRegisterResponse, MemberMeResponse, NicknameRegenerateResponse, ExplainErrorPayload, DiffExplainPayload 타입 추가
- **`src/api/questions.ts`**: executeChoice body `{ choiceKey }` → `{ sql }` 수정 (백엔드 실제 스펙에 맞춤)
- **`src/api/ai.ts`**: `Record<string, unknown>` → ExplainErrorPayload / DiffExplainPayload 타입 안전 적용
- **`.claude/rules/api-guide.md`**: Members 구현 상태 O, execute body sql, diff-explain 백엔드 프록시 스펙 반영

#### Phase 6: 성능 최적화

- **StarRating**: 공용 컴포넌트 추출 + React.memo (Questions, QuestionDetail 중복 제거)
- **ChoiceCard**: React.memo 분리 → 선택지 변경 시 4개→2개만 리렌더
- **QuestionDetail 콜백**: useRef로 executeCache 참조 → handleExecute/handleSelect 의존성 제거 → 콜백 안정화
- **AiExplanationSheet**: renderMarkdown useMemo (텍스트 변경 시에만 재파싱)
- **Stats**: 히트맵 그리드 useMemo (progress 변경 시 리렌더 방지)

#### Phase 7: 에러 처리 통일

- **shouldRetry 함수** (신규): ApiError instanceof 체크로 5xx 에러 retry 방지. main.tsx의 duck typing → 타입 안전 함수로 리팩토링
- **ErrorFallback 컴포넌트** (신규): "데이터를 불러올 수 없습니다" + 재시도 버튼. Home, Questions, Stats에 적용
- **refetchOnWindowFocus: false**: 탭 전환 시 불필요한 API 재호출 방지

#### Phase 8: 테스트 인프라 + 단위 테스트

- **`src/api/client.test.ts`** (11건): apiFetch 성공/실패/타임아웃, ApiError 생성, 헤더 병합, POST body 전달

- **`src/components/ErrorFallback.test.tsx`** (5건): 기본 메시지, 커스텀 메시지, 재시도 버튼 렌더링/클릭
- **`src/components/StarRating.test.tsx`** (5건): 별 3개 렌더링, 레벨별 채움 검증
