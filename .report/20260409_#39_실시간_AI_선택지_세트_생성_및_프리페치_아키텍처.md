# 실시간 AI 선택지 세트 생성 및 프리페치 아키텍처

## 개요

문제(question)에 고정된 선택지를 분리하여, 퀴즈 진입 시마다 AI가 새로운 선택지 세트를 실시간 생성하고 샌드박스 검증을 거쳐 제공하는 아키텍처를 구현했다. 현재 문제를 푸는 동안 다음 문제의 선택지를 백그라운드 프리페치하여 체감 지연을 최소화하며, MVP는 AI_ONLY 단일 모드로 동작하되 CURATED/HYBRID 확장 경로를 스키마·코드 분기점으로 예약했다. 전체 60개 파일, +6,904줄 변경.

## 주요 구현 내용

### 실시간 생성 + 프리페치 파이프라인

퀴즈 진입 시 `QuizSessionService.createSession()`이 AI_ONLY 문제 중 10개를 셔플하여 세션을 생성한다. `getQuestionAt(index)`가 호출되면 `ChoiceSetResolver`가 프리페치 캐시(consumed_at IS NULL, source=AI_PREFETCH, status=OK)를 먼저 조회하고, 없으면 `ChoiceSetGenerationService`로 런타임 생성한다. 동시에 `PrefetchService.prefetchNext()`가 @Async로 다음 문제의 선택지를 백그라운드 생성한다.

### 3회 재시도 + 샌드박스 검증

`ChoiceSetGenerationService`는 AI로 선택지 4개를 생성한 후, `SandboxValidator`로 검증한다. SandboxPool이 임시 DB를 생성하고, DDL·INSERT를 적용한 뒤 answerSql 실행 결과를 ground truth로 삼아 각 선택지 SQL의 결과와 비교한다. correctCount != 1이면 최대 3회 재생성하며, 3회 모두 실패하면 status=FAILED로 저장하고 사용자에게 재시도 안내한다.

### AI 호출 이중화

메인 경로는 Spring → Python AI Server(FastAPI + google-genai SDK → Gemini 2.5 flash-lite)이며, Resilience4j CircuitBreaker(sliding-window:10, failure-rate:50%, wait:30s)와 Retry(max:2, wait:500ms)로 보호된다. 서킷 오픈 또는 Python 서버 장애 시 `GeminiFallbackClient`가 Spring 내에서 Gemini API를 직접 호출한다.

### 관리자 AI 보조 문제 등록

3단계 위저드: (1) 스키마·INSERT·의도 입력 → (2) AI가 문제 stem + answerSql + 시드 선택지 4개 생성 → (3) 관리자가 편집 후 저장. 저장 시 Question + QuestionChoiceSet(source=ADMIN_SEED) + Items가 트랜잭션으로 일괄 생성된다.

## 주의사항

- **SandboxPool 커넥션 누수**: acquire() 시 매번 새 DataSource를 생성하지만 close하지 않아 커넥션 풀 누수 가능. HikariDataSource.close() 호출 필요.
- **@Transactional 프록시**: ChoiceSetGenerationService 내부의 protected 메서드에 @Transactional 적용은 Spring AOP 프록시에서 동작하지 않음. public으로 변경하거나 별도 빈으로 분리 필요.
- **Jackson snake_case 매핑**: AiGatewayClient의 SNAKE_CASE ObjectMapper와 Java record 호환성 검증 필요 (record는 기본 생성자가 없어 역직렬화 이슈 가능).
- **JPQL FQCN enum 참조**: QuestionRepository의 `findActiveAiOnlyUuids()` 쿼리에서 enum FQCN 참조가 Hibernate 버전에 따라 동작하지 않을 수 있음.
- **Python gemini_client.py 동기 호출**: google-genai SDK의 동기 메서드를 async FastAPI 핸들러에서 호출 — 이벤트 루프 블로킹 가능. asyncio.to_thread() 래핑 또는 비동기 SDK 사용 권장.
- **application-prod.yml은 .gitignore**: 설정 파일이 버전 관리에 포함되지 않으므로, 배포 환경에서 수동 설정 필요.
- **빌드/테스트 미검증**: 내부망 환경으로 인해 빌드 및 테스트가 수행되지 않았음. 배포 전 반드시 컴파일·통합 테스트 필요.
