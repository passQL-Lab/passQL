# ❗ [버그][QuestionDetail] "풀이를 그만할까요?" 확인 모달 레이아웃 깨짐 (데스크톱) #142

### 📌 작업 개요

데스크톱 환경에서 "풀이를 그만할까요?" 확인 모달 레이아웃이 깨지는 버그 수정.
모달 카드에 적용된 `md:` 반응형 클래스가 flex 컨테이너 안에서 잘못 동작해
너비가 극도로 좁아지고 텍스트가 세로 나열되는 현상 발생.
동시에 뒤로가기 버튼 클릭 시 모달이 열리지 않던 동작 버그와 이중 확인 버그도 함께 수정.

---

### 🔍 문제 분석

**레이아웃 깨짐 원인**

`ConfirmModal`의 카드 div에 `md:max-w-sm`, `md:rounded-xl`, `md:pb-6` 등 `md:` 브레이크포인트 클래스가 적용되어 있었음.
오버레이가 `flex items-end justify-center`인 상황에서 카드에 `max-width` 제약이 생기자
flex 아이템이 콘텐츠 너비로 수축하며 모달 전체가 극도로 좁아짐.

**모달 미트리거 원인**

`PracticeSet`의 뒤로가기(← ArrowLeft) 버튼이 `onClick={() => navigate("/")}` 직접 호출로 구현되어 있었음.
문제를 하나 풀고 피드백 화면인 상태에서는 `navigate("/")`가 `useBlocker`에 차단되지 않고 바로 이동하는 경우가 있어 확인 모달을 거치지 않음.

**이중 확인 버그 원인**

`exitModalOpen` 경로(Home 버튼)에서 "나가기" 확인 시 `navigate("/")`를 직접 호출하면
`useBlocker(!shouldNavigateToResult)` 조건이 여전히 `true`이므로 이 navigation이 다시 차단됨.
모달이 닫혔다가 즉시 재열려 사용자가 두 번 확인해야 하는 흐름 발생.

---

### ✅ 구현 내용

#### 1. ConfirmModal — 반응형 제거 + 스크롤 잠금 개선
- **파일**: `src/components/ConfirmModal.tsx`
- **변경 내용**
  - 카드 div에서 `md:items-center`, `md:max-w-sm`, `md:rounded-xl`, `md:pb-6`, `md:hidden` 등 `md:` 반응형 클래스 전체 제거
  - 모든 화면 크기에서 `w-full rounded-t-2xl` 바텀시트 레이아웃 단일화
  - `document.body.style.overflow = ""` → `prevOverflow` 저장 후 복원으로 변경 (중첩 모달 스크롤 잠금 보호)

#### 2. PracticeSet — 이탈 확인 흐름 재구현
- **파일**: `src/pages/PracticeSet.tsx`
- **변경 내용**
  - Home 아이콘 → `ArrowLeft` 아이콘으로 변경 (다른 화면과 통일)
  - 버튼 `onClick` → `setExitModalOpen(true)`로 교체 (기존 `navigate("/")` 직접 호출 제거)
  - `exitModalOpen` state 추가 — blocker와 독립적으로 모달 열림 제어
  - `exitConfirmed` state 추가 — Home 버튼 경로 이중 차단 방지

#### 3. 이중 차단 방지 로직
`exitConfirmed` 플래그 도입으로 blocker 비활성화 후 navigate 실행:
- `blocker` 조건: `!shouldNavigateToResult && !exitConfirmed`
- "나가기" 확인(Home 버튼 경로): `setExitConfirmed(true)` → blocker 비활성화
- `useEffect([exitConfirmed])`: `exitConfirmed === true`가 되면 `navigate("/")`
- 브라우저 뒤로가기 경로: 기존 `blocker.proceed?.()` 유지

#### 4. Tailwind 정규 클래스 치환
- **파일**: `src/components/ChoiceCard.tsx`, `src/components/ChoiceReview.tsx`, `src/pages/QuestionDetail.tsx`
- `break-words` → `wrap-break-word` (Tailwind v4 정규 클래스)
- `bg-[#F3F4F6]` → `bg-surface-code` (디자인 토큰 사용)

#### 5. 접근성 개선
- **파일**: `src/pages/PracticeSet.tsx`
- ArrowLeft 버튼에 `aria-label="풀이 나가기"` 추가 (아이콘 전용 버튼 스크린리더 지원)

### 🔧 주요 변경사항 상세

#### ConfirmModal.tsx — 단일 레이아웃

```
Before: fixed inset-0 z-50 flex items-end justify-center
        + 카드에 md:items-center md:max-w-sm md:rounded-xl ...

After:  fixed inset-0 z-50 flex items-end justify-center (오버레이 동일)
        + 카드: w-full rounded-t-2xl (모든 화면에서 바텀시트)
```

데스크톱에서도 화면 너비 100% 바텀시트로 표시. 디자인 가이드(모든 화면 크기 일관성)에 따른 결정.

#### PracticeSet.tsx — exitConfirmed 패턴

```
blocker = useBlocker(!shouldNavigateToResult && !exitConfirmed)

Home 버튼 → setExitModalOpen(true)
onConfirm:
  - blocker.state === "blocked" → blocker.proceed?.()  // 브라우저 뒤로가기
  - else → setExitConfirmed(true)                      // Home 버튼
useEffect: exitConfirmed → navigate("/")               // blocker 비활성화 후 이동
```

**특이사항**: `setExitConfirmed(true)`와 `navigate()`를 같은 tick에 실행하면 blocker가 아직 활성 상태라 차단됨. React 리렌더링 사이클을 활용해 effect에서 navigate를 지연 실행하는 방식으로 해결.

### 📌 참고사항

- 데스크톱에서 중앙 팝업 모달 형태를 원한다면 오버레이를 `flex items-center`로 바꾸고 카드에 `max-w-sm mx-auto rounded-2xl` 적용 필요. 단, 이 경우 모바일/데스크톱 분기가 생기므로 별도 이슈로 논의 권장.
- `exitConfirmed` 플래그는 컴포넌트 언마운트 시 자동 초기화되므로 별도 reset 로직 불필요.
