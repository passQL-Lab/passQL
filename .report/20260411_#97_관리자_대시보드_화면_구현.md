# ➕[기능추가][관리자] 관리자 대시보드 화면 구현

## 개요

`/admin` 경로에 대한 컨트롤러·서비스가 없어 404가 발생하던 문제를 해결했다. `AdminDashboardController`와 `AdminDashboardService`를 신규 구현하고, `dashboard.html`을 보강해 문제 수·회원 현황·실행 로그 등 핵심 운영 지표를 한눈에 파악할 수 있는 대시보드를 완성했다. 또한 루트(`/`) 접속 시 관리자 대시보드로 자동 리다이렉트되도록 하고, Swagger 문서 링크를 사이드바와 빠른 링크에 추가했다.

---

## 변경 사항

### 신규 파일

- `server/PQL-Application/.../dto/DashboardStats.java`
  - 대시보드 집계 결과 DTO (`record`)
  - 문제 통계(`totalQuestions`, `questionsByTopic`), 회원 통계(`totalMembers`, `activeMembers`, `suspendedMembers`), 제출 통계(`todaySubmissions`), 실행 통계(`aiCallCount`, `errorRate`) 포함
  - 중첩 record `TopicQuestionCount`로 토픽별 문제 수 표현

- `server/PQL-Application/.../service/AdminDashboardService.java`
  - Application 레이어 집계 서비스
  - `QuestionRepository`, `MemberRepository`, `SubmissionRepository`, `ExecutionLogRepository`, `TopicRepository` 조합
  - `collect()` 메서드 1개로 단일 진입점 제공

- `server/PQL-Web/.../controller/admin/AdminDashboardController.java`
  - `GET /admin`, `GET /admin/` 라우팅
  - `AdminDashboardService.collect()`로 통계 수집 후 Model 바인딩
  - `SubmissionService.getRecentLogs()`로 최근 실행 로그 20건 전달

- `server/PQL-Web/.../controller/RootController.java`
  - `GET /` → `redirect:/admin` 처리
  - `localhost:8080` 접속 시 관리자 대시보드로 바로 이동

### 수정 파일

- `server/PQL-Domain-Submission/.../repository/SubmissionRepository.java`
  - `countBySubmittedAtAfter(LocalDateTime since)` 추가
  - 오늘 00:00 이후 전체 제출 건수를 `findAll()` 없이 단일 count 쿼리로 집계

- `server/PQL-Web/.../templates/admin/dashboard.html`
  - 회원 통계 카드 3개 추가 (전체·활성·정지)
  - 토픽별 문제 분포 — 비율 막대 바 차트
  - 빠른 링크 그리드 — Swagger API 문서 버튼 추가 (새 탭, 전체 너비)
  - 최근 실행 로그 테이블 유지

- `server/PQL-Web/.../templates/admin/layout.html`
  - 사이드바 하단에 구분선 + Swagger API 문서 링크 추가 (새 탭)

## 주요 구현 내용

### 다중 도메인 집계 — Application 레이어 분리

`AdminDashboardService`는 Member·Question·Submission·Meta 네 개 도메인 Repository를 조합한다. 단일 도메인 서비스 안에 이 로직을 넣으면 도메인 경계가 무너지므로, 기존 `HomeService` 패턴을 따라 `PQL-Application` 모듈에 배치했다.

### 토픽별 문제 분포 시각화

`QuestionRepository.countActiveByTopic()`의 native group-by 결과(`Object[]`)를 `TopicRepository` 전체 조회와 메모리 내 조인해 `topicUuid → displayName` 매핑을 구성한다. 조인 실패 시 UUID 앞 8자리를 폴백으로 표시한다. 결과는 문제 수 내림차순 정렬 후 전체 대비 % 비율로 막대 바를 렌더링한다.

### 오늘 제출 건수 집계 최적화

기존 `findAll()` 후 스트림 필터 방식 대신, `countBySubmittedAtAfter(LocalDateTime since)` Spring Data JPA 파생 메서드를 추가해 단일 `COUNT` 쿼리로 처리했다.

### Swagger 링크 통합

Swagger UI 경로(`/docs/swagger`)를 사이드바 하단과 대시보드 빠른 링크 양쪽에 배치했다. 양쪽 모두 `target="_blank"`로 새 탭에서 열린다.

## 주의사항

- `AdminDashboardService.collect()`는 `findByIsActiveTrue().size()`로 활성 문제 수를 집계한다. 문제 수가 매우 많아지면 `countByIsActiveTrue()` 파생 메서드를 `QuestionRepository`에 추가해 count 쿼리로 교체하는 것이 바람직하다.
- 회원 통계는 `memberRepository.findAll()` 후 스트림 필터링이므로, 회원 수가 수만 건을 넘으면 상태별 count 쿼리 분리를 고려해야 한다.
- `aiCallCount`는 현재 항상 `0`으로 반환된다. 향후 AI 호출 로그 도메인이 생기면 `MonitorStats`의 해당 필드를 실제 집계값으로 교체하면 된다.
