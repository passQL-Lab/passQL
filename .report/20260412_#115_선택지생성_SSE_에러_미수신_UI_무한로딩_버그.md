# ❗[버그][문제] 선택지 생성 SSE 에러 미수신 시 UI 무한 로딩 버그 #115

## 개요

선택지 생성(`/generate-choices`) SSE 스트림에서 백엔드 에러가 발생하거나 스트림이 조용히 종료될 경우, 프론트엔드가 `onError` 이벤트를 수신하지 못해 "선택지 생성 중..." 스피너가 무한 지속되는 버그를 수정했다. 클라이언트 타임아웃(60초), `STREAM_CLOSED` 탐지, 에러 표시 + 재시도 UI를 구현하여 탈출 경로를 확보했다.

## 변경 사항

### 프론트엔드 — SSE 클라이언트 (`questions.ts`)

- `client/src/api/questions.ts`: `generateChoices()` 함수 전면 개선
  - `receivedTerminalEvent` 플래그 추가 — `complete`/`error` 이벤트 수신 여부 추적
  - `done === true` 시 플래그 미수신 + abort 미실행 조건에서 `STREAM_CLOSED` 에러 콜백 발생 — 조용한 스트림 종료를 에러로 명시화
  - HTTP 에러 응답(4xx/5xx) 즉시 `onError` 처리 (`HTTP_${status}` 코드, 5xx만 retryable)
  - `response.body` 없음 처리 (`NO_BODY` 에러)
  - JSON 파싱 실패 시 `PARSE_ERROR`로 즉시 에러 전환 + while 루프 중단 (`abortProcessing` 플래그)
  - `AbortError`는 정상 cleanup이므로 `onError` 미호출

### 프론트엔드 — 문제 상세 페이지 (`QuestionDetail.tsx`)

- `client/src/pages/QuestionDetail.tsx`: SSE 에러 처리 및 타임아웃 UI 추가
  - `sseError` state 타입: `{ code: string; retryable: boolean } | null`
  - SSE_TIMEOUT_MS = 60초 타임아웃 설정 — `setTimeout`으로 타임아웃 발동 시 스트림 abort + `TIMEOUT` 에러 상태 전환
  - `streamCleanup` 참조 저장 — 타임아웃 콜백에서 스트림 abort 가능
  - `onComplete`/`onError` 수신 시 `clearTimeout`으로 타임아웃 취소
  - 에러 코드별 사용자 메시지: `TIMEOUT` → "선택지 생성 시간이 초과되었어요", `STREAM_CLOSED` → "서버 연결이 예기치 않게 끊겼어요", 기타 → "선택지 생성에 실패했어요"
  - `retryable=true` 에러 시 `<RefreshCw>` 아이콘 + "다시 시도" 버튼 표시 — `sseRetryCount` 증가로 `useEffect` 재실행
  - `sseRetryCount`를 `useEffect` 의존성에 포함 — 재시도 시 SSE 재연결 트리거

### 프론트엔드 — 타입 정의 (`api.ts`)

- `client/src/types/api.ts`: SSE 이벤트 타입 추가/확장
  - `SseErrorEvent`: `code`, `message`, `retryable` 필드 포함
  - `ChoiceSetGenerateResponse`, `SseStatusEvent` 타입 정의
  - `choiceSetPolicy` 필드 `QuestionDetail`에 추가 (RESULT_MATCH 선택지 실행 버튼 숨김 분기 기반)

## 주요 구현 내용

**조용한 종료 탐지**: EventSource 기반 SSE와 달리 fetch + ReadableStream 방식에서는 `reader.read()`의 `done === true`가 스트림 종료를 의미한다. 이 시점에 `receivedTerminalEvent`가 false이고 `AbortController`가 abort되지 않은 경우를 "서버가 complete/error 없이 연결을 닫은" 상황으로 판단하여 `STREAM_CLOSED` 에러를 발생시킨다.

**클라이언트 타임아웃**: AI 호출 + 샌드박스 3회 재시도 합산 예상 시간 기준 60초를 설정한다. 타임아웃 발동 시 진행 중인 스트림을 abort하여 이후 콜백이 호출되지 않도록 막고, 에러 상태로 전환한다. `streamCleanup` 변수가 타임아웃 콜백과 컴포넌트 unmount cleanup 양쪽에서 공유된다.

**재시도 UX**: `sseError.retryable === true`인 경우에만 "다시 시도" 버튼을 표시한다. 클릭 시 `sseChoices`, `sseChoiceSetId`, `sseError`를 null로 초기화하고 `sseRetryCount`를 증가시켜 `useEffect`가 새 SSE 연결을 시작하도록 트리거한다. `PARSE_ERROR`처럼 반복해도 무의미한 에러는 `retryable=false`로 재시도 버튼을 숨긴다.

## 주의사항

- 백엔드가 `event:error` 이벤트를 정상적으로 전달하는 경우 타임아웃은 `clearTimeout`으로 취소됨 — 오작동 없음
- 타임아웃 60초는 AI + 샌드박스 최악 시나리오 기준이므로 서버 응답이 개선되면 단축 가능
- `STREAM_CLOSED` 탐지는 abort된 스트림과 구별하여 오탐을 방지 (`abortController.signal.aborted` 체크)
