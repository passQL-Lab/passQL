# 환영 메시지 응답 스키마 개선 — FE 연동

### 📌 작업 개요

백엔드 #53에서 `GreetingResponse` 스키마를 `{ message: string }` 단일 필드에서 `{ nickname, message, messageType }` 3필드로 분리함에 따라, 프론트엔드에서 신규 스키마를 연동하고 `messageType` 분기 UI를 완성.

---

### 🎯 구현 목표

- `{nickname}` 플레이스홀더 치환 처리
- `messageType` 4종(GENERAL / EXAM_DAY / URGENT / COUNTDOWN) 분기 서브텍스트 표시
- api-guide 문서 최신화

---

### ✅ 구현 내용

#### messageType 분기 UI
- **파일**: `src/pages/Home.tsx`
- **변경 내용**: `greeting.messageType` 값에 따라 인사 메시지 하단에 서브텍스트 조건부 표시
  - `EXAM_DAY`: "오늘 시험이에요!" (시맨틱 에러 색상 — 긴박감 강조)
  - `URGENT`: "시험이 얼마 남지 않았어요" (경고 색상)
  - `COUNTDOWN`: "시험 D-{N}" — `readiness.daysUntilExam` 값 표시, null이면 "시험일이 다가오고 있어요" 폴백
  - `GENERAL`: 서브텍스트 없음

#### 닉네임 플레이스홀더 치환
- **파일**: `src/pages/Home.tsx`
- **변경 내용**: `greeting.message.replace("{nickname}", greeting.nickname)` 단순 치환으로 렌더링
- **이유**: 서버가 `{nickname}` 리터럴을 포함한 템플릿으로 내려주므로 프론트에서 치환 후 표시. 닉네임 스타일링 분리는 현재 단계에서 미적용 (단순 replace로 충분).

#### api-guide 문서 업데이트
- **파일**: `.claude/rules/api-guide.md`
- **변경 내용**: `fetchGreeting` 항목 설명을 신규 스키마(#53) 기준으로 교체
  - `GreetingResponse { nickname, message, messageType }` 명시
  - `{nickname}` 플레이스홀더 치환 규칙
  - `messageType` 4종 값 목록
  - D-day 구간 가중치 혼합 동작 설명
  - 회원 조회 실패 시 `nickname="회원"` + GENERAL 폴백 명시

---

### 🔧 주요 변경사항 상세

#### COUNTDOWN 서브텍스트
`readiness.daysUntilExam`을 `progress` 응답에서 직접 참조. `daysUntilExam`이 null인 경우(시험 일정 미선택 등) "시험일이 다가오고 있어요"로 graceful 폴백 처리.

#### 타입 정의 (기존 #52에서 완료)
`GreetingMessageType = "GENERAL" | "EXAM_DAY" | "URGENT" | "COUNTDOWN"` 및 `GreetingResponse { nickname, message, messageType }` 타입은 이전 작업에서 이미 `src/types/api.ts`에 정의됨.

### 📌 참고사항

- 백엔드 #53 스키마 변경은 breaking change. `message` 필드를 그대로 렌더할 경우 `{nickname}` 리터럴이 화면에 노출되므로 반드시 치환 후 표시.
- 서버가 더 이상 이모지를 내려주지 않으므로 아이콘이 필요하면 프론트에서 `messageType` 값에 따라 자체 매핑 필요 (현재 미구현 — 텍스트만 표시).
