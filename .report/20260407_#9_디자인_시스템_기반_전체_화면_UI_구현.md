### 📌 작업 개요

passQL 프론트엔드 전체 화면 UI를 디자인 시스템(Prompt 0~7) 기반으로 구현. 빈 스캐폴딩 상태였던 7개 화면을 mock 데이터로 채우고, 디자인 시스템을 Prompt0 스펙으로 전면 재구축. 반응형 네비게이션(Desktop 사이드바 + Mobile 하단탭) 적용. API 연동 없이 순수 UI/레이아웃에 집중.

**보고서 파일**: `.report/20260407_#9_디자인_시스템_기반_전체_화면_UI_구현.md`

### 🎯 구현 목표

- 디자인 시스템(색상, 토큰, 타이포그래피, 컴포넌트)을 Prompt0 스펙으로 재구축
- 반응형 레이아웃(Desktop 사이드바 220px + Mobile 하단탭 56px) 구현
- 7개 화면(홈, 문제 목록, 문제 상세, 결과, AI 해설, 통계, 설정) UI를 mock 데이터로 구현
- Lucide React 아이콘 라이브러리로 통일 (이모지 제거)

### 🧪 테스트 및 검증

- `npm run build` 전체 빌드 통과 (TypeScript 컴파일 + Vite 빌드)
- 각 화면 라우트별 브라우저 확인:
  - `/` — 홈 (인사말, 오늘의 문제, 스트릭, 통계)
  - `/questions` — 문제 목록 (필터, 카드 5개)
  - `/questions/1` — 문제 상세 (4종 선택지 상태)
  - `/questions/1/result` — 정답 결과
  - `/questions/1/result?correct=false` — 오답 결과
  - `/stats` — 통계 (히트맵, 최근 틀린 문제)
  - `/settings` — 설정 (UUID, 닉네임, 버전)
