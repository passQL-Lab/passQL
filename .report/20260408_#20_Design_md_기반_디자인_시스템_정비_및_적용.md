### 작업 개요

Design.md에 Linear 다크 모드 디자인 시스템이 그대로 적용되어 있어 passQL 실제 구현(라이트 모드)과 불일치하는 문제 해결. Design.md를 passQL 라이트 모드 기반으로 전면 커스텀하고, CSS 토큰/컴포넌트 스타일을 문서와 동기화.

**이슈**: [#20](https://github.com/passQL-Lab/passQL/issues/20)

---

### 구현 목표

- Design.md를 passQL 라이트 모드 디자인 시스템으로 교체
- CSS 토큰(tokens.css) - Design.md - 컴포넌트 CSS(components.css) 3곳 동기화
- Typography letter-spacing, spacing scale, border radius 등 세부 스타일 정비
- 토스 UX 심리학 원칙 문서화 및 UI 라이팅 개선

---

### 구현 내용

#### 1. Design.md 전면 커스텀

- **파일**: `.claude/rules/Design.md`
- **변경 내용**: Linear 다크 모드 기반 디자인 시스템을 passQL 라이트 모드 기반으로 교체
- **주요 스펙**: Pretendard Variable(UI) + JetBrains Mono(코드), `#FAFAFA` 페이지 배경, `#4F46E5` 브랜드 인디고, `#E5E7EB` 보더, 그림자 없음(배경색 차이로 elevation 표현)

#### 2. CSS 토큰 정비

- **파일**: `src/styles/tokens.css`
- **변경 내용**: Design.md 스펙에 맞춰 디자인 토큰 전면 정비
  - 색상 체계: surface/brand/text/border/semantic/heatmap 토큰 정의
  - 스페이싱 스케일: 4px base unit 기반 7단계(4/8/12/16/20/24/32px)
  - 트랜지션: `cubic-bezier(0.4, 0, 0.2, 1)` ease-smooth 커브 추가

#### 3. Typography letter-spacing 추가

- **파일**: `src/styles/typography.css`
- **변경 내용**: heading에 negative tracking 적용
  - `.text-h1`: `letter-spacing: -0.5px`
  - `.text-h2`: `letter-spacing: -0.3px`
  - body/secondary: `line-height: 1.7`로 통일

#### 4. 컴포넌트 스타일 Design.md 동기화

- **파일**: `src/styles/components.css`
- **변경 내용**: 12개 컴포넌트 클래스를 Design.md 스펙에 맞춰 정비
  - 카드: `border-radius: 16px`, `padding: 24px`, hover 시 `transform + background-color` 트랜지션
  - 버튼(primary/secondary/compact): hover 시 `translateY(-1px)` 마이크로 모션 추가
  - 코드 블록: `border-radius: 12px`, `border-left: 4px solid brand`
  - 에러/성공 카드: `border-radius: 12px`로 통일
  - 토스트: `border-radius: 12px`
  - 필터 드롭다운: hover 시 `translateY(-1px)` 모션

#### 5. UI/UX 구조 개선

- **파일**: `src/pages/Home.tsx`, `src/pages/QuestionDetail.tsx`, `src/pages/Questions.tsx`, `src/pages/Settings.tsx`, `src/pages/Stats.tsx`, `src/pages/AnswerFeedback.tsx`
- **변경 내용**:
  - CTA 버튼 순서 정비 (primary -> secondary 순서 통일)
  - 페이지 제목 계층 정리
  - box-shadow 제거 (배경색 차이로 elevation 표현)
  - 유니코드 심볼 제거, lucide-react 아이콘으로 교체

#### 6. UX 라이팅 개선

- **파일**: 각 페이지 컴포넌트
- **변경 내용**: 토스 라이팅 원칙(Clear, Concise, Casual) 적용
  - 불필요한 문구 제거 (Weed Cutting)
  - 복사 완료 피드백 토스트 추가
  - 사용자 행동 유도 문구를 권유형으로 변경

#### 7. 토스 UX 디자인 원칙 문서 추가

- **파일**: `.claude/rules/TOSS_UI_UX_DESIGN.md`
- **변경 내용**: 토스의 10가지 UX 심리학 법칙 + 8가지 라이팅 원칙 + 5가지 코어밸류 정리. 향후 UI 개발 시 디자인 판단 기준으로 활용.

#### 8. 토스 감성 모션 적용

- **파일**: `src/styles/components.css`
- **변경 내용**:
  - 버튼/카드/필터에 `translateY(-1px)` hover 마이크로 모션
  - 트랜지션 duration: 200~250ms
  - `cubic-bezier(0.4, 0, 0.2, 1)` ease 커브 전체 적용
  - line-height: body 텍스트 1.7로 통일 (가독성 개선)

#### 9. Mock 데이터 환경변수 지원

- **파일**: `src/api/mock-data.ts`, `vite.config.ts`
- **변경 내용**: `VITE_USE_MOCK` 환경변수로 mock 데이터 on/off 제어 가능하도록 구현. 실제 API 연동과 mock 전환이 용이해짐.

#### 10. Mock 데이터 엔드포인트 매핑 테스트

- **파일**: `src/api/mock-data.test.ts`
- **변경 내용**: 21건의 엔드포인트 매핑 테스트 추가. mock 데이터가 실제 API 스펙과 일치하는지 검증.

### 주요 변경사항 상세

#### 디자인 토큰 동기화 구조

```
Design.md (문서, single source of truth)
    |
    v
tokens.css (CSS 변수 정의)
    |
    v
components.css (컴포넌트 스타일 적용)
typography.css (타이포그래피 적용)
```

Design.md 스펙 -> CSS 토큰 -> 컴포넌트 CSS 3곳이 동일한 값을 참조하도록 정합성 확보.

#### border-radius 통일 기준

| 요소 | radius | 용도 |
|------|--------|------|
| 버튼, 입력 필드 | 8px | 인터랙티브 요소 |
| 카드, 코드 블록, 에러/성공 카드, 토스트 | 12~16px | 컨테이너 |
| 모달, 바텀시트 | 16px | 오버레이 |
| 필터, 뱃지 | 999px | pill 형태 |

#### hover 모션 패턴

모든 인터랙티브 요소에 동일한 마이크로 모션 적용:
- `transform: translateY(-1px)` (살짝 떠오르는 느낌)
- `transition: 200ms cubic-bezier(0.4, 0, 0.2, 1)`
- primary 버튼은 `opacity: 0.9` 추가
