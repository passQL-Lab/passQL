# 질문관리 대량 JSON 코드리뷰 반영 및 가이드 모달 추가

## 개요
PR #62에 대한 CodeRabbit 코드리뷰 반영(topicCode 필터 풀림 버그 수정), 문제 편집 페이지의 개념형 문제 스키마 섹션 노출 버그 수정, 질문 관리 페이지에 대량 JSON 가이드 모달 추가, 그리고 누락되었던 가이드 MD 파일 생성을 수행했다.

## 변경 사항

### 버그 수정 (백엔드)
- `QuestionImportExportService.java`: `exportByFilter()`에서 존재하지 않는 topicCode가 들어올 때 topicUuid가 null이 되어 토픽 필터가 풀리는 문제 수정. `orElse(null)` → `orElseThrow(() -> new CustomException(ErrorCode.TOPIC_NOT_FOUND))`로 변경

### 버그 수정 (프론트엔드)
- `question-edit.html`: 개념형(CONCEPT_ONLY) 문제 편집 시 스키마 섹션(DDL, 샘플데이터, 정답SQL 등)이 숨겨지지 않고 그대로 노출되는 문제 수정. 페이지 로드 시 현재 선택된 실행 모드에 따라 `updateSchemaVisibility()`를 즉시 호출하도록 변경

### 가이드 모달 추가
- `questions.html`: 질문 관리 페이지 액션 바에 "가이드" 버튼 추가, 대량 JSON 내보내기/가져오기 가이드 모달 구현 (topicCode 목록, JSON 배열 스키마 예시, 주요 규칙, AI 변환 프롬프트 + 복사 기능, MD 파일 다운로드)

### 가이드 MD 파일 생성
- `static/docs/question-register-guide.md` (신규): 단건 문제 등록 가이드. 등록 페이지 모달에서 다운로드되는 파일이 누락되어 있었음
- `static/docs/question-bulk-guide.md` (신규): 대량 JSON 내보내기/가져오기 가이드. 질문 관리 페이지 모달에서 다운로드

## 주요 구현 내용

### topicCode 필터 풀림 방어
기존에는 `topicRepository.findByCode(topicCode)`가 빈 결과일 때 `orElse(null)`로 null을 반환하여 `findByFiltersAll()` 쿼리에서 토픽 필터가 무시되었다. 잘못된 topicCode로 의도보다 넓은 범위의 문제가 내보내기되는 보안 이슈였으며, `TOPIC_NOT_FOUND` 예외를 던지도록 수정했다.

### 개념형 문제 편집 시 스키마 섹션 숨김
`question-edit.html`의 JS에서 실행 모드 라디오 버튼의 `change` 이벤트 리스너만 등록하고, 페이지 로드 시 초기 상태 반영이 누락되어 있었다. CONCEPT_ONLY가 이미 checked인 상태로 로드되면 change 이벤트가 발생하지 않아 스키마 섹션이 그대로 노출되었다. `updateSchemaVisibility()` 함수를 추출하고 페이지 로드 시 즉시 호출하도록 수정했다.

### 가이드 MD 파일 분리 전략
단건 등록 가이드(`question-register-guide.md`)와 대량 가이드(`question-bulk-guide.md`)를 별도 파일로 분리했다. 단건은 단일 JSON 객체 스키마, 대량은 JSON 배열 스키마 + questionUuid/Sandbox 검증 규칙 + 건수 제한 등 추가 내용을 포함한다.

## 주의사항
- Entity/DB 스키마 변경 없음
- `question-register.html`의 스키마 섹션 숨김은 기본값이 EXECUTABLE(checked)이므로 동일 이슈가 발생하지 않아 수정하지 않음
- 가이드 MD 파일은 정적 파일로 관리. 토픽 추가 시 MD 파일도 함께 업데이트 필요
