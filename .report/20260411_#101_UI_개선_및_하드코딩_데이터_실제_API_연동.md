# UI 개선 및 하드코딩 분석 데이터 실제 API 연동

이슈: passQL-Lab/passQL#101

---

### 📌 작업 개요

연습 결과 화면의 하드코딩된 AI 코멘트를 실제 `fetchAiComment` API로 대체하고, 데일리 챌린지 전용 페이지를 신설하며, 홈 화면의 오늘의 문제 완료 UI를 개선. 버전 정보 하드코딩 제거 및 API 문서 최신화도 함께 진행.

---

### ✅ 구현 내용

#### 1. 연습 결과 화면 AI 코멘트 실제 API 연동

- **파일**: `src/pages/PracticeResult.tsx`
- **변경 내용**: 하드코딩된 분석 텍스트 제거 → `fetchAiComment()` + `useQuery`로 비동기 조회. 로딩 중 skeleton UI, 데이터 없을 경우 graceful 숨김 처리
- **이유**: 사용자 실제 학습 패턴을 기반으로 한 개인화 코멘트 표시. Redis 24h 캐시와 맞게 `staleTime: 1000 * 60 * 60 * 24` 설정

#### 2. 데일리 챌린지 전용 페이지 신설

- **파일**: `src/pages/DailyChallenge.tsx`, `src/App.tsx`
- **변경 내용**: `/daily-challenge` 라우트 추가. `useTodayQuestion()` 기반 조건부 렌더링 — 이미 풀었으면 홈으로 리디렉션, 문제 없으면 안내 화면. 제출 성공 시 `isDailyChallenge: true` 상태와 함께 결과 페이지로 이동
- **이유**: 홈의 "오늘의 문제" 카드와 연결되는 독립적인 챌린지 흐름 필요

#### 3. QuestionDetail 제출 콜백 확장

- **파일**: `src/pages/QuestionDetail.tsx`
- **변경 내용**: `onSubmitSuccess(result, questionUuid)` 선택적 prop 추가. 데일리 챌린지 등 호출자가 제출 후 네비게이션을 직접 제어할 수 있도록 콜백 우선 실행
- **이유**: 일반 문제풀기와 데일리 챌린지의 제출 후 동작 분리

#### 4. AnswerFeedback 데일리 챌린지 분기

- **파일**: `src/pages/AnswerFeedback.tsx`
- **변경 내용**: `isDailyChallenge` navigate state 플래그 추가. 정답 시 → 홈으로, 오답 시 → 데일리 챌린지 다시 풀기로 버튼 동작 분기

#### 5. 홈 오늘의 문제 완료 UI 개선

- **파일**: `src/pages/Home.tsx`
- **변경 내용**: `alreadySolvedToday` 플래그 활용. 완료된 문제 카드는 아이콘 제거 후 `opacity` 처리 + "오늘의 문제 (완료)" 라벨 표시. 완료 후 카드 클릭 비활성화(링크 제거)

#### 6. 버전 정보 하드코딩 제거

- **파일**: `vite.config.ts`, `src/vite-env.d.ts`, `src/pages/Settings.tsx`, `package.json`
- **변경 내용**: `package.json` 버전을 빌드 시 전역 상수 `__APP_VERSION__`으로 주입. `import.meta.url` 기반 절대 경로 해석으로 CWD 무관하게 동작. `package.json` 버전 `0.0.0 → 1.0.0`
- **이유**: Settings 화면에서 버전을 코드에 직접 박지 않고 단일 소스(`package.json`)로 관리

#### 7. API 문서 최신화

- **파일**: `docs/be-api-docs.json`, `.claude/rules/api-guide.md`
- **변경 내용**: `generateChoices` SSE 엔드포인트, `submitAnswer`의 `choiceSetId` 필드, `/progress/topic-analysis`, `/progress/ai-comment` 엔드포인트 추가. `SubmitResult`에 `selectedResult / correctResult / selectedSql / correctSql` 실행 비교 필드 문서화

### 🔧 주요 변경사항 상세

#### API 연동 패턴 (fetchAiComment)

`useEffect + cancelled` 플래그에서 `useQuery`로 교체. StrictMode에서 `useEffect`는 요청 자체를 2번 보내지만, `useQuery`는 `queryKey` 기준으로 중복 제거. `staleTime`을 Redis 캐시(24h)와 맞춰 불필요한 재요청 방지.

#### 데일리 챌린지 흐름

```
/daily-challenge 진입
  ├─ 이미 풀었으면 → / (홈) 리디렉션
  ├─ 문제 없으면 → 안내 화면
  └─ 문제 있으면 → QuestionDetail (onSubmitSuccess 주입)
       └─ 제출 성공 → /questions/{uuid}/result (isDailyChallenge: true)
            └─ 정답: 홈 / 오답: 데일리 챌린지 재시도
```

#### vite.config.ts 버전 주입

```typescript
const pkg = JSON.parse(
  readFileSync(
    fileURLToPath(new URL("./package.json", import.meta.url)),
    "utf-8",
  ),
);
define: {
  __APP_VERSION__: JSON.stringify(pkg.version);
}
```

`import.meta.url` 기반으로 저장소 루트가 아닌 설정 파일 위치 기준으로 `package.json` 탐색.

### 📌 참고사항

- `be-api-docs.json`은 실제 배포된 백엔드 서버 스펙이므로 프론트 단독으로 변경하지 않음
